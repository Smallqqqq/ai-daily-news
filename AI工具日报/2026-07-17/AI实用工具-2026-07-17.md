## n8n AI Assistant — 196K stars 的开源自动化平台迎来了自己的 AI agent：用自然语言搭工作流，400+ 集成开箱即用

  7 月 9 日，n8n 团队发布了 AI Assistant——一个内嵌在 n8n 编辑器里的对话式 agent，能根据自然语言描述自动规划、搭建、测试和调试工作流。n8n 是 GitHub 上最受欢迎的开源工作流自动化平台，196K stars，430 位贡献者，1500+ 集成，9000+ 社区模板。AI Assistant 直接操作 n8n 的可视化画布，创建标准 n8n 工作流——你能看到每一个节点、每一条连线，也可以手动调整任何一步。

  核心能力分四层。**创建**——描述"每天早上检查 Gmail 里的发票邮件，把 PDF 附件存到 Google Drive，在数据表里加一行记录"，AI Assistant 会自动规划节点链、配置触发器和参数。**编辑**——对已有工作流说"把失败订单的通知从邮件改成 Slack"，它只改相关节点不动其他地方。**测试与排查**——工作流跑失败了，让它诊断原因、建议修复方案，确认后再动手。**审查门控**——发布、删除等高影响操作必须先经过你确认；密钥和凭证永远不离开 n8n 的凭证管理系统，不会出现在发给 AI 的请求里。

  底层用的是 Claude Opus 4.8。n8n 团队在社区帖里解释了技术栈选择：AI Assistant 和 n8n MCP server 共享同一套底层架构，区别只在于交互界面——MCP server 是给外部 agent 用的 API，AI Assistant 是给人用的聊天面板。两者能力等价，但 AI Assistant 让你在一个界面里看到 AI 构建工作流的全过程，不用在 n8n 和终端之间来回切。

  自托管用户也能用。社区已经有人跑通了 Docker 部署方案，用 Sonnet 5 替换默认的 Opus 4.8 来降低成本。官方坦诚当前自托管部署流程还比较手动，涉及多个容器的网络配置，正在做简化。

```bash
# n8n Cloud 用户（v2.29.9+）
# 左侧边栏选择 AI Assistant → 选择项目 → 描述需求

# 自托管（Docker，预发布阶段）
docker volume create n8n_data
docker run -it --rm --name n8n -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  docker.n8n.io/n8nio/n8n
# 然后按照社区指南配置 AI Assistant 容器
```

## agentmanager — 93 个 AI CLI 工具的包管理器，一条命令管全部：检测、安装、更新、卸载

  2026 年了，开发者的终端里塞满了各种 AI CLI 工具。Claude Code 用 npm 装的，Codex 用 brew 装的，Gemini CLI 用原生安装器装的，Aider 用 pip 装的——每个工具有自己的安装方式、自己的更新命令、自己的版本检查方法。想知道哪些工具该更新了？一个个敲 `--version` 然后去官网查最新版。

  agentmanager 解决了这个碎片化问题。它是一个 Go 写的 CLI/TUI 应用，内置了 93 个 AI 开发 CLI 工具的目录——从 Claude Code、Codex、Gemini CLI 到 OpenCode、Aider、Cline、Qwen Code——覆盖 npm、pip、pipx、brew、go install、原生安装器等全部安装方式。一条 `agentmgr agent list` 列出所有已安装工具及版本状态（● 最新 / ⬆ 可更新），`agentmgr agent update --all` 一键全部更新。

  几个设计细节体现了作者对真实使用场景的理解。**并行版本检查**——几十个工具的版本查询并发执行，比串行快 10-20 倍。**离线优先目录**——工具目录编译进二进制文件里，`go install` 完立刻能用，不需要网络。**实时安装输出**——加 `-v` 参数后显示 brew/npm/pip 的原生安装日志，不是干等一个转圈动画。**TUI 模式**——`agentmgr tui` 启动 Bubble Tea 驱动的交互式终端界面，用键盘浏览和操作。

  它不是"AI 工具的 Homebrew"——Homebrew 只管通过 brew 安装的软件，而 agentmanager 理解每个工具的原始安装方式，用该工具原生的包管理器去安装和更新。你通过 brew 装的 Codex、通过 npm 装的 Claude Code、通过 pip 装的 Aider——agentmanager 知道它们各自该怎么升级，不需要你记住。

```bash
# 安装
brew install kevinelliott/tap/agentmanager
# 或
go install github.com/kevinelliott/agentmanager/cmd/agentmgr@latest

# 列出所有已安装的 AI 工具及版本状态
agentmgr agent list

# 一键更新全部
agentmgr agent update --all

# 安装新工具
agentmgr agent install aider

# 交互式 TUI
agentmgr tui
```

  agentmanager 和 Homebrew 解决的是不同层次的问题。Homebrew 管理的是"通过 brew 安装的软件包"——如果你用 npm 装了 Claude Code，brew 根本看不见它。agentmanager 管理的是"AI CLI 工具"这个特定类别——不管你用什么方式装的，它都能检测到并正确更新。对于每天都在用 3-5 个不同 AI CLI 工具的开发者，这个差异就是"一条命令更新全部"和"打开五个终端窗口各敲各的"的差异。
