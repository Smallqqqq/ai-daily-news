## GitHub Agentic Workflows — 用 Markdown 写 CI/CD，AI 在 Actions 里替你跑

  如果你写过 GitHub Actions 的 YAML，你一定经历过这种挫败：缩进差一格就炸、市场里几百个 Action 要逐个试、条件分支写得像意大利面。2026 年 6 月 11 日，GitHub 正式将 Agentic Workflows 推进到 Public Preview 阶段——现在你可以用自然语言 Markdown 描述自动化意图，由 AI Agent 在 GitHub Actions 的沙箱里替你执行。

  它的工作方式出奇简单：在 `.github/workflows/` 下写一个 Markdown 文件，用 YAML frontmatter 配置触发条件和权限，正文用自然语言写清楚"当新 Issue 打开时，分析内容、打标签、评估优先级，必要时 @ 相关负责人"。然后 `gh aw compile` 把它编译成标准 Actions YAML，推上去就跑。底层支持 GitHub Copilot、Claude Code、Codex 三种 AI 引擎，你可以按任务特点切换——比如代码审查用 Claude、Triage 用 Copilot。

  安全设计是它区别于"在 CI 里塞个 API key 让 AI 自己玩"的关键。Agent 默认只读权限，写操作必须通过 Safe Outputs 白名单——创建 Issue、添加评论、发起 PR 是预审批的动作，直接改代码不行。每次运行在沙箱容器里，后面还跟着一个威胁检测 Job 扫描所有输出。更关键的是 6 月 11 日同一天宣布支持 `GITHUB_TOKEN`，不再需要个人 PAT——这意味着自动化归组织所有，出问题能审计到人。

  实际能做什么？官方给了几个典型场景：Issue 自动分类与优先级评估、CI 失败根因分析、文档与代码一致性检查、依赖更新风险评估、测试覆盖率趋势报告。Carvana 和 Marks & Spencer 已经在用——M&S 的 CTO 说"以前需要几小时工程投入的任务，现在几分钟内自主完成"。

```bash
# 安装 CLI 扩展
gh extension install github/gh-aw

# 创建一个示例工作流
gh aw init

# 编译 Markdown → Actions YAML
gh aw compile

# 推送到仓库即可触发
git add .github/workflows/
git commit -m "add agentic workflow"
git push
```

  GitHub Agentic Workflows 不是要替代传统 CI/CD——它是在确定性流水线之上加了一层"需要推理和判断"的自动化。如果你的任务是"每次 PR 跑 lint + test + build"，传统 Actions 就够了。如果任务是"分析这个 Issue 说的是性能问题还是功能请求，查一下最近 3 个相关 PR 是谁改的，评估一下修复难度然后打标签"——这就是 Agentic Workflows 的战场。

## MiMo Code — Xiaomi 开源的终端 AI 编程 Agent，带跨会话记忆

  终端 AI 编程 Agent 这个赛道已经挤满了玩家：Claude Code、Codex CLI、Gemini CLI、Kimi Code CLI、OpenCode。2026 年 6 月 10 日，Xiaomi MiMo 团队以 MIT 协议开源的 MiMo Code 带来一个差异化破局点：**持久化跨会话记忆**。

  大多数终端 Agent 的问题是"金鱼记忆"——每次新开会话，Agent 都要重新读代码、重新理解架构、重新问你一遍"这个项目是做什么的"。MiMo Code 用 SQLite FTS5 全文搜索引擎在本地维护四层记忆：项目记忆（`MEMORY.md`，持久化的架构知识和编码规则）、会话检查点（`checkpoint.md`，由 checkpoint-writer 子 Agent 自动维护的状态快照）、便签（`notes.md`，Agent 临时工作区）、任务进度（按任务独立记录）。当会话恢复时，记忆自动注入上下文——Agent 不需要重新学习项目。

  MiMo Code 是 OpenCode 的 fork，保留了后者的全部能力（75+ 模型提供商、TUI、LSP、MCP、插件系统），在此基础上加了自己的东西。三个内置 Agent 各司其职：**build**（默认，全部工具权限，做开发）、**plan**（只读分析模式，设计方案）、**compose**（编排模式，走"需求→设计→编码→测试→部署"的完整工业流水线）。子 Agent 系统支持并行工作，`/goal` 命令设终止条件后有独立 judge 模型验证是否真正完成——防止 Agent 在长任务中"乐观早退"。

  Xiaomi 自报的 benchmark 显示，搭配 MiMo-V2.5-Pro 模型时，SWE-Bench Pro 62%（Claude Code + Sonnet 4.6 为 57%），Terminal Bench 2 73%（Claude Code 为 68%）。用同一模型跑两个 harness，差距大约 5 个百分点，归因于 Agent 系统本身而非模型。不过需要注意的是，这些数据是自报的，尚未出现在官方公开排行榜上；作为参考，Codex CLI + GPT-5.5 在 Terminal-Bench 2.1 公开榜上以 83.4% 排第一。

  MiMo Auto 通道内置免费模型（限时），零配置即可开始。也支持接入 DeepSeek、Kimi、GLM 等第三方 API，以及自定义 OpenAI 兼容端点。

```bash
# 一键安装（macOS / Linux）
curl -fsSL https://mimo.xiaomi.com/install | bash

# Windows / 通用
npm install -g @mimo-ai/cli

# 启动
cd your-project
mimo
```
