# AI 实用工具 · 2026-07-07

![[summary.png]]

  今天两个工具分别解决 AI 编码工作流中最枯燥的两件事：让 Agent 直接操作数据库而不是手写连接串，以及让 Agent 替你部署上线而不是手动 SSH 敲命令。

## FreePeak/db-mcp-server — 一个 MCP 服务器同时对接 MySQL、PostgreSQL、SQLite、Oracle 和 TimescaleDB，Agent 自动获得五套数据库工具

  目前 MCP 生态里大多数数据库服务器只连一种数据库。postgres-mcp 只能连 PostgreSQL，mysql-mcp 只能连 MySQL。你的项目里同时用了 PostgreSQL 做主库、Redis 做缓存、SQLite 做本地存储？那你要装三个 MCP 服务器、维护三份配置、Agent 的工具列表里塞满三十几个数据库工具。

  FreePeak 的 db-mcp-server 用一个二进制解决了这个问题。一份 JSON 配置文件声明所有数据库连接，服务启动后自动为每个连接生成一组工具——`query_mysql1`、`schema_postgres1`、`performance_oracle1`，命名规则一目了然。Agent 看到的不是抽象的"数据库工具"，而是明确知道哪个工具对应哪个库。目前完整支持 MySQL、PostgreSQL（v9.6-17）、SQLite（含加密和内存模式）、Oracle（10g-23c，含 RAC 和 Cloud Wallet）和 TimescaleDB（含超表、连续聚合、压缩策略）。

  一个容易被忽略但非常实用的设计是懒加载模式。配置文件里声明了 10 个数据库，但不一定全部都在用。加 `--lazy-loading` 参数后，服务只在 Agent 第一次调用某个数据库的工具时才建立连接。对于开发环境里经常切换项目、每个项目连不同数据库的场景，这个开关省掉了大量无谓的连接等待。

  三种部署模式覆盖不同使用场景。STDIO 模式直接嵌在 Claude Code 或 Cursor 的 MCP 配置里，随 Agent 启动。SSE 模式在服务器上常驻，多个 Agent 客户端共享同一套数据库连接。Docker 模式一条 `docker run` 加挂载配置文件即可，适合不想在本机装 Go 环境的用户。

  Go 编写，编译为单个二进制。6 位贡献者，17 个 release，最新 v1.9.0（2026 年 4 月），MIT 协议。

```bash
# 源码安装
git clone https://github.com/FreePeak/db-mcp-server.git
cd db-mcp-server && make build

# STDIO 模式（Claude Code / Cursor）
./bin/server -t stdio -c config.json

# Docker 模式
docker pull freepeak/db-mcp-server:latest
docker run -p 9092:9092 \
  -v $(pwd)/config.json:/app/my-config.json \
  -e TRANSPORT_MODE=sse \
  -e CONFIG_PATH=/app/my-config.json \
  freepeak/db-mcp-server
```

  Cursor 的 MCP 配置示例：

```json
{
  "mcpServers": {
    "stdio-db-mcp-server": {
      "command": "/path/to/db-mcp-server/server",
      "args": ["-t", "stdio", "-c", "/path/to/config.json"]
    }
  }
}
```

  FreePeak db-mcp-server 和 DBHub 走的是同一条路——一个网关接多种数据库。但设计取舍不同。DBHub 把工具数压到最少（只有 schema 查询和 SQL 执行两个），目的是省 token。FreePeak 则为每个连接生成 query/schema/execute/transaction/performance 五类工具，目的是功能深度。如果你只是偶尔让 Agent 查一下数据，DBHub 的两工具模型够用；如果你需要 Agent 帮你做性能分析、管理事务、读写分离，FreePeak 的工具粒度更实用。对于大多数日常开发场景——既要查数据又要看 schema 又要分析慢查询——FreePeak 是目前唯一一个同时做到「多数据库」「工具按连接自动生成」「有性能分析能力」「支持懒加载」四项全中的方案。

## Nixopus — 开源部署平台，AI Agent 替你分析代码、生成配置、部署上线、监控自愈

  部署这件事，从 Heroku 到 Vercel 到 Railway，体验一直在变好，但本质上还是你在操作。你写完代码→打开部署面板→配置环境变量→点部署→部署失败→读日志→修 bug→重新部署。Nixopus 把这个循环里「你在操作」的部分替换成了 AI Agent。

  核心流程很简单：连接你的 GitHub 仓库，告诉 Agent「帮我部署」，Agent 自动分析你的代码栈（Next.js / Django / Rails / Go / FastAPI 等任意容器化应用），生成对应的构建配置，构建镜像，部署上线，配置 SSL 证书和自定义域名。全程你只需要在仪表盘里或者 VS Code 侧边栏里说一句话。部署失败时 Agent 自动读日志、诊断问题、开 PR 修 bug、重新部署——目前这个自愈功能还在开发中，但部署和回滚已经是完整可用的。

  自托管版一条命令安装：`curl -fsSL install.nixopus.com | sudo bash`。装完之后你拥有一个完整的部署控制面板，支持多服务器管理、浏览器内终端 SSH、实时 CPU/内存监控、一键回滚到任意历史版本。也可以用 Nixopus Cloud 直接开始，不需要自己维护服务器。

  VS Code 和 Cursor 扩展把 Agent 嵌进了编辑器侧边栏。你在编辑器里写代码，Agent 在侧边栏里帮你部署。不需要切到浏览器、不需要开终端、不需要记命令。

  和同类的关键差异：Coolify（52K stars）更成熟、生态更大、模板更多，但它是传统 PaaS——你要自己在面板里操作。Nixopus 的差异化价值在于 Agent 接管了操作环节。你不是「用一个部署工具」，而是「告诉 Agent 帮你部署」。对个人开发者和小型团队来说，这个差异意味着你可以把注意力完全留在代码上。

  目前仍处于 alpha 阶段，许可证为 FSL（Functional Source License），自托管免费，生产环境建议等稳定版。

```bash
# 自托管一条命令安装
curl -fsSL install.nixopus.com | sudo bash

# 带自定义域名和管理员邮箱
curl -fsSL install.nixopus.com | sudo DOMAIN=panel.example.com \
  ADMIN_EMAIL=admin@example.com \
  OPENROUTER_API_KEY=sk-or-xxxxx bash

# VS Code 扩展
# 在 VS Code Marketplace 搜索 "Nixopus" 安装
```
