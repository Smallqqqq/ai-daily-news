# AI 实用工具 · 2026-06-08

  今天三个工具覆盖 AI 编码 Agent 的三个关键基础设施缺口：上下文成本控制、IDE 深度集成、以及终端 Agent 的编辑器体验。

---

## LeanCTX — 给 AI 编码 Agent 装上上下文压缩引擎，典型会话节省 88% token

  AI 编码 Agent 最大的隐性成本不是模型单价，而是上下文浪费。每次 `git status` 灌进去 800 token，重复读同一个文件又烧掉 2000 token，一次典型会话中超过 80% 的 token 消耗在非推理的 I/O 上。LeanCTX 是一个用 Rust 写的本地二进制文件，它拦截 Agent 和代码之间的每一次文件读取、Shell 命令和搜索查询，在输出到达模型之前对其压缩、缓存和验证。

  核心机制分四层。**智能 I/O**：10 种读取模式（full、map、signatures、diff、lines 等），tree-sitter 对 18 种语言做 AST 感知压缩，56 个模式模块自动精简 git/npm/cargo/docker 等命令输出。**会话记忆**：跨聊天持久化项目知识，带时序事实和多边依赖图（imports、calls、exports、type_ref），长会话不再"冷启动"。**Token 治理**：5 种角色（Admin、Coder、Debugger、Reviewer、Ops），按 Agent 配置预算和限流策略。**验证**：4 层验证引擎加 CI 漂移门禁，确保压缩不丢失关键信息。

  项目方公布的实际数据：缓存重读文件从 30,000 token 降到 195 token（99%），`git status/log/diff` 节省 70%，测试运行输出节省 90%，典型会话节省 88%。Active developers 每月节省 $30-100+ API 费用。

  三种集成模式自动适配不同工具。CLI-Redirect（推荐，Cursor、Claude Code、Codex）：Agent 通过 Shell 调用 `lean-ctx`，零 MCP 开销。Hybrid（Windsurf、Amp）：MCP 做缓存读取 + Shell hooks 做命令压缩。Full MCP（JetBrains、Copilot、VS Code）：全部 63 个 MCP 工具。v3.7.3 本周新增 `shell` MCP 工具，让 Codex Desktop/Cloud 也能享受压缩。

```bash
# macOS / Linux
brew install lean-ctx
# 或
cargo install lean-ctx

# 一键配置（自动注入 shell hooks + MCP）
lean-ctx setup

# 查看实时节省
lean-ctx gain --live
```

### 同类工具对比

| 工具 | 实现方式 | 核心特点 | 支持 Agent | 热度 |
|------|---------|---------|-----------|------|
| **LeanCTX**（今日推荐）| Rust 本地二进制 | 10 种读取模式、AST 压缩、会话记忆、预算治理 | 24+ | ⭐ 3,000+ |

## MCP Steroid — 给 AI Agent 完整的 IntelliJ IDE 能力：重构、调试、测试、截图全都有

🏪 JetBrains Marketplace · ⬇ 946 GitHub stars · 开源

  AI 编码 Agent 最大的能力盲区是什么？它看不到你的 IDE。Agent 可以用 grep 搜代码，但它不知道这个方法的调用链是什么、这个类的继承树长什么样、这个重构操作会影响多少文件——而这些恰好是 JetBrains IDE 每天都在做的事。MCP Steroid 是一个跑在 IntelliJ 平台内部的 MCP 服务器，通过标准 HTTP 端口把 IDE 的全部能力暴露给任何 MCP 兼容的 AI Agent。

  这是目前唯一一个同时覆盖以下能力的 MCP 服务器：**语义代码理解**（PSI 树、goto definition、find usages）、**高级重构**（安全重命名、代码生成）、**调试器集成**（断点、线程、变量检查）、**测试运行器**（执行测试、获取结果、导航测试树）、**视觉 IDE 理解**（截图 + OCR + 组件树），以及 **Kotlin 脚本运行时**——Agent 可以直接在 IDE 的 JVM 里执行 Kotlin 代码来调用任意 IntelliJ API。

  服务器跑在 IDE 的 JVM 进程内，不需要进程间通信。HTTP 端口默认 6315，启动后自动在项目 `.idea/mcp-steroid.md` 里写入连接 URL。支持三种人工审核模式（ALWAYS / TRUSTED / NEVER），敏感操作可以始终要求确认。

  9 个 MCP 工具覆盖 Execute Code、Vision Screenshot、Vision Input、Action Discovery、Capabilities Discovery、List Projects/Windows、Open Project 等。额外通过 LSP 暴露 22 个 IDE Power Operations（重构、代码生成、项目分析）、10 个 Debugger 工具、10 个 Test Runner 工具、3 个 VCS 工具。任何 MCP 兼容客户端都能用——Claude Code、Cursor、Codex、Gemini CLI、Junie 等。

  安装后在 JetBrains IDE 插件市场搜索 "MCP Steroid"，重启后在 Settings > Tools > MCP Server 中启用即可。

## Antigravity for VS Code — Google 的终端 AI 编码 Agent 终于有了 VS Code 皮肤

🏪 Visual Studio Marketplace · 2026-06-07 发布 · MIT 开源

  Google 的 Antigravity CLI（`agy`）是一个基于 Gemini 的终端编码 Agent，能力不输 Claude Code——多步推理、多文件编辑、工具调用、持久会话。但它在终端里裸跑：手动管理 session、凭记忆敲 slash command、编辑器和 Agent 之间来回切换。Antigravity for VS Code 是一个社区维护的 VS Code 扩展，它把 `agy` 的完整能力装进了一个 VS Code 侧边栏面板，架构思路和 Claude Code 的 VS Code 扩展一样——编辑器提供 UI 层，CLI 做实际工作。

  几个实用的设计细节。**Session 管理**：侧边栏列出所有已保存会话，可以恢复、删除或新建，新建时可选沙箱模式或绕过权限。**实时终端镜像**：标题栏的终端按钮一键切换出当前 `agy` 进程的真实终端——同一个进程，不是 fork。当你需要看 Agent 实际在跑什么，或者手动干预时，这个按钮省掉了 `ps` + `attach` 的流程。**Slash Command 导航器**：输入 `/` 自动弹出 35 个命令的补全菜单，命令列表从已安装的 CLI 动态拉取，永远匹配实际版本。

  **上下文感知**：选中代码后 `Ctrl/Cmd + Alt + K` 直接把选中内容和文件名、行号一起发给当前会话，不需要复制粘贴。安装要求只有两个：VS Code 1.90+ 和已安装并登录的 `agy` CLI。未安装时面板只显示 "Sign in with Google" 按钮和 "Install CLI" 操作，不会让新用户面对一个空白聊天框。

```bash
# 1. 安装 Antigravity CLI
# macOS / Linux
curl -fsSL https://antigravity.google/cli/install.sh | bash
# Windows PowerShell
irm https://antigravity.google/cli/install.ps1 | iex

# 2. 登录
agy

# 3. 在 VS Code 扩展面板搜索 "Antigravity for VS Code" 并安装
```
