# AI 实用工具 · 2026-06-19

![[summary.png]]

  今天两个工具，一个让 AI 生成的 UI 从"不可控的文本"变成"可约束的结构化界面"，另一个让 AI 编码助手真正学会用调试器而不是 println 排查问题——分别代表了 Generative UI 和 Agentic Debugging 两个方向在 2026 年 6 月的最新落点。

---

## json-render — Vercel 出品的 Generative UI 框架，用 JSON Schema 约束 AI 的界面输出

  AI 生成 UI 最大的问题不是"不好看"，而是"不可靠"——模型可能编造不存在的组件、输出格式不一致、无法跨平台复用。json-render 用一套 Zod Schema 定义的组件目录（Catalog）作为护栏，AI 只能从你预先定义的组件库中选择，输出结构化 JSON，再由对应平台的 Renderer 渲染成真实界面。15,000+ GitHub stars，Vercel 官方维护，Apache 2.0 开源。

  核心思路简单但有效：你定义 Catalog（有哪些组件、每个组件有什么 props、触发什么 action），AI 在你的约束内生成 JSON spec，Renderer 负责渲染。因为输出是纯 JSON，所以同一份 spec 可以在 React、Vue、Svelte、React Native、甚至 CLI 终端和 PDF 文档中分别渲染——跨平台不是事后适配，而是架构层面的设计决策。

  它内置了 36 个 shadcn/ui 组件（Radix UI + Tailwind CSS），开箱即用。同时支持流式渲染（SpecStream）——AI 边生成 JSON，界面边增量更新，用户体验上接近打字机效果。devtools 面板可以实时查看 spec 树、state 状态、action 日志，调试 Generative UI 不再靠猜。

```bash
# React 项目
npm install @json-render/core @json-render/react
# 带 36 个预置 shadcn/ui 组件
npm install @json-render/shadcn

# Vue / Svelte / Solid 项目
npm install @json-render/core @json-render/vue
npm install @json-render/core @json-render/svelte
npm install @json-render/core @json-render/solid

# Next.js 整站生成（路由、布局、SSR）
npm install @json-render/core @json-render/react @json-render/next
```

  实际场景：Dashboard 生成、表单引擎、AI 聊天中的富卡片、内部工具搭建——凡是"让 AI 出界面但不想失控"的场景都适用。特别适合需要同一套组件逻辑在 Web + Mobile + Email + PDF 多端输出的产品。

### 同类工具对比

| 工具 | 核心理念 | 跨平台 | 流式渲染 | 护栏机制 | 热度 |
|------|---------|:---:|:---:|---------|------|
| **json-render**（今日推荐）| JSON Schema → 多端 Renderer | ✅ Web/Mobile/CLI/PDF/3D | ✅ SpecStream | Zod Schema + Catalog | ⭐ 15.4K |
| Vercel AI SDK RSC | 服务端渲染 React 组件流 | ❌ Next.js only | ✅ createUIMessageStream | 无 | ⭐ 45K |
| CopilotKit | Copilot 模式组件嵌入 | ❌ React only | ✅ | 无 | ⭐ 22K |

  json-render 的独特价值在于它不是又一个 React-only 方案——它把"AI 生成 UI"拆成两层：Catalog 定义面 + Renderer 执行面。这意味着同一个 Catalog 约束下，AI 的输出可以同时渲染到 Web、移动端、邮件、PDF 甚至终端 TUI。Vercel AI SDK 和 CopilotKit 在 React/Next.js 生态内更成熟，但 json-render 是跨框架团队当下唯一现实的选择。

## DebugMCP — Microsoft 出品，让 AI 编码助手用真正的调试器排查 Bug

  AI 编码助手最大的能力盲区是什么？不是写代码，而是**调试**。当一个 agent 写完代码跑不通，它只能读错误日志、猜原因、改代码、再跑——循环往复。DebugMCP 改变了这个范式：它把你的 VS Code 调试器暴露为 MCP 工具，AI 助手可以像人类开发者一样设断点、单步执行、查看变量值、评估表达式。360+ GitHub stars，Microsoft 官方维护，VS Code Marketplace 近 4000 安装量，MIT 开源。

  原理很简单：DebugMCP 是 VS Code 扩展 + MCP Server 的组合体。安装后自动启动本地 MCP 服务，任何兼容 MCP 的 AI 助手（Claude Code、Copilot、Cursor、Codex、Windsurf、Cline 等 8+ 客户端）都能直接调用调试器。16 个 MCP 工具覆盖完整调试流程：启动/停止会话、设断点、单步执行（step over/into/out）、查看变量、评估表达式。

  支持 9 种语言：Python、JavaScript/TypeScript、Java、C#、C++、Go、Rust、PHP、Ruby——基本覆盖主流技术栈。无需额外配置，装好即用。v2.0.0 新增 `/debug` Agent Skill，AI 助手可以自动加载系统化调试流程。

```bash
# VS Code 中安装
# 搜索 "DebugMCP" 或直接访问：
# https://marketplace.visualstudio.com/items?itemName=ozzafar.debugmcpextension
```

  实际场景：你给 Claude Code 或 Cursor 一个任务，它写完代码运行失败。过去它只能读报错日志盲猜；现在它可以设断点在怀疑的函数入口、step into 进入调用、查看变量实际值、发现是某个字段为 null——然后精准修复，而不是改三处猜一处。

### 同类工具对比

| 工具 | 调试方式 | AI 客户端支持 | 语言覆盖 | 安装方式 | 热度 |
|------|---------|-------------|---------|---------|------|
| **DebugMCP**（今日推荐）| VS Code DAP 调试器 | 8+（Claude Code/Cursor/Copilot 等）| 9 种 | VS Code 扩展一键安装 | ⭐ 368 / 3,983 安装 |

  DebugMCP 的核心优势是 Microsoft 官方出品 + VS Code 原生集成 + 零配置。竞品 mcp-debugger 更灵活（可以脱离 IDE 用），但需要手动安装各语言的 debug adapter；DebugPilot 支持 Dart/Flutter 热重载调试，但语言覆盖较少。如果你已经在 VS Code 中写代码、用 AI 助手，DebugMCP 是路径最短的选择。
