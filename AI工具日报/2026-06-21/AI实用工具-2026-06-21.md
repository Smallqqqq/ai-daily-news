## VS Code ACP Client — 一个协议，11 个 Agent，从此不再切窗口

🏪 VS Code Marketplace · ⬇ 15,094 次安装 · v1.0+ · [安装](https://marketplace.visualstudio.com/items?itemName=formulahendry.acp-client)

  2026 年的 AI 编码工具已经陷入了碎片化地狱。Claude Code 在终端里，GitHub Copilot 在侧边栏，Gemini CLI 在另一个窗口，Codex 又在另一个——每切一次工具，上下文就断一次。更糟的是，不同团队、不同项目可能偏好不同的 Agent，你不得不同时维护四五套 CLI 和配置。VS Code ACP Client 用一招解决了这个问题：把 Agent Client Protocol（ACP）变成一个 VS Code 扩展，在一个面板里直接连接 11 个主流 AI Agent。

  ACP 是 Zed Industries 在 2025 年 8 月创建的开放协议，定位是「AI Agent 的 LSP」——就像 LSP 让任何语言服务器接入任何编辑器，ACP 让任何 AI Agent 接入任何支持协议的工具。协议本身是 JSON-RPC 2.0 over stdio，极简。VS Code ACP Client 是这个协议目前最成熟的 VS Code 实现，由 Jun Han（formulahendry）开发，MIT 开源。

  核心体验：打开 ACP 面板，点击一个 Agent 就连接上了。一次只激活一个 Agent，但可以在 11 个之间随时切换。每个 Agent 保留独立的会话历史——你可以在 Claude 那里讨论架构，切到 Copilot 写实现，再切到 Gemini 处理 Google Cloud 部署脚本，整个过程在同一面板、同一工作区，上下文不丢。

  它不只是一个「把终端塞进 VS Code」的薄封装。它做了几件终端做不到的事：动态会话配置（Agent 声明自己支持的模式、模型、推理等级，扩展自动渲染为工具栏选择器）、思考过程可视化（带流式动画和耗时的可折叠 Thinking 面板）、权限策略管理（`ask` / `allowAll` 两级）、完整的 JSON-RPC 协议流量日志（调试 MCP 连接问题时的救命稻草）。

  预配置的 11 个 Agent，涵盖当前所有主流选择：

| Agent | 启动方式 |
|-------|---------|
| GitHub Copilot | `npx @github/copilot-language-server@latest --acp` |
| Claude Code | `npx @agentclientprotocol/claude-agent-acp@latest` |
| Gemini CLI | `npx @google/gemini-cli@latest --experimental-acp` |
| Qwen Code | `npx @qwen-code/qwen-code@latest --acp` |
| Codex CLI | `npx @zed-industries/codex-acp@latest` |
| OpenCode | `npx opencode-ai@latest acp` |
| Kiro CLI | `kiro-cli acp` |
| Hermes Agent | `hermes acp` |
| Auggie CLI | `npx @augmentcode/auggie@latest --acp` |
| Qoder CLI | `npx @qoder-ai/qodercli@latest --acp` |
| OpenClaw | `npx openclaw acp` |

```bash
# 安装：VS Code 扩展面板搜索 "ACP Client"，或直接：
code --install-extension formulahendry.acp-client
```

  需要注意的是，ACP 和 MCP 是互补关系而非竞争关系。ACP 处理「编辑器 ↔ Agent」的通信，MCP 处理「Agent ↔ 外部工具」的通信。在典型工作流中：VS Code（ACP Client）→ Claude Code（ACP Agent）→ GitHub MCP Server（MCP）。两层协议各司其职。

## vscode-pi — 把 pi Agent 装进 VS Code，本地跑、不挑模型

🏪 VS Code Marketplace · Open VSX · ⬇ 73 ⭐ · v0.15.0 · [安装](https://marketplace.visualstudio.com/items?itemName=cdervis.vscode-pi)

  pi 是一个与众不同的 AI 编码 Agent——它由社区驱动，强调极简和本地运行。vscode-pi 是它的 VS Code 扩展，由社区开发者 cdervis 独立维护，把 pi CLI 的能力完整搬进了编辑器的侧边栏。

  这跟 ACP Client 的定位完全不同。ACP Client 是「多 Agent 的总控台」，vscode-pi 是「一个 Agent 的深度集成」。它的设计哲学是让 Agent 无限贴近你的代码——活动文件自动作为上下文、选中的代码直接发送、生成的文件变更在完成后统一展示。不需要切终端，不需要手动粘贴路径。

  几个值得关注的能力：支持 Mermaid 图表和数学公式的内联渲染（不仅是代码高亮）；内置基于 whisper.cpp 的本地麦克风听写（macOS/Windows 都支持）；上下文窗口实时跟踪和自动压缩；会话历史持久化并在 VS Code 重启后恢复；Post-run 文件变更摘要，让你在 Agent 完成后快速审阅改了什么。

  因为 pi 本身是本地 Agent CLI，模型选择取决于你本地的 pi 配置——可以是云端 API，也可以是通过 Ollama 运行的本地模型。这意味着它既可以做轻量级本地补全，也可以挂载远端强模型做复杂重构。

```bash
# 前置条件：先安装 pi CLI
# 参考 https://pi.dev 的安装指南

# 然后从 VS Code Marketplace 安装扩展
code --install-extension cdervis.vscode-pi
```
