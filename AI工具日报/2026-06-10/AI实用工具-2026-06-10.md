# AI 实用工具 · 2026-06-10

![[summary.png]]

  今天两个工具走向了终端 AI 编程 Agent 的两个极端：一个把 Agent 从 TypeScript 重写成 Go 单二进制，极致精简；另一个把 Agent 从终端里解放出来，放进一个像素风办公室，让它们自己开会、分工、干活。

---

## Reasonix CLI — DeepSeek 原生 Go 语言终端 AI 编程 Agent

  DeepSeek 模型用户长期以来的痛点：官方没有终端编程 Agent。Claude Code 绑定 Claude 模型，Codex CLI 绑定 GPT 模型，Kimi Code 绑定 Kimi 模型——你要用 DeepSeek 就只能靠开源社区的各种非官方桥接方案。Reasonix CLI（原 DeepSeek-Reasonix）在 2026 年 4 月以 1.0 版本用 Go 语言彻底重写，成了这个空白的最强填补者。

  v1.0 的重写是一次彻底的工程决策。原 0.x 版本是 TypeScript 写的，功能齐全但启动慢、打包大。Go 重写后变成单个静态二进制——`CGO_ENABLED=0` 编译，交叉编译到 Darwin/Linux/Windows × amd64/arm64 六个目标平台——零运行时依赖。整个项目围绕 DeepSeek 的 prefix cache 稳定性设计，长时间会话下 token 成本保持在低位。6 月 7 日发布的 v1.3.0 进一步稳定了这个新架构。

  配置驱动是 Reasonix 区别于同类工具的最大特征。模型、工具、权限、沙箱全部声明在一个 `reasonix.toml` 文件里，不要硬编码。任何 OpenAI-compatible 端点都是一个配置项而不是新代码。可选的双模型协作模式——executor + planner 跑在各自独立的 cache-stable 会话中——planner 负责只读研究和方案设计，executor 负责写文件和跑命令。MCP 客户端支持 stdio 和 Streamable HTTP 两种传输，还能直接读取已有的 `.mcp.json` 文件零配置迁移。权限系统是三层漏斗：`deny` > `ask` > `allow`，硬拦截规则（如 `bash(git push*)`）在所有模式下生效。沙箱把文件写入操作限制在项目目录内。

```bash
# 安装（npm 拉预编译 Go 二进制）
npm i -g reasonix

# 或 macOS Homebrew
brew install esengine/reasonix/reasonix

# 快速开始
reasonix setup               # 配置向导生成 reasonix.toml
export DEEPSEEK_API_KEY=sk-...
reasonix chat                 # 交互模式，输入 /init 生成 AGENTS.md
reasonix run "实现 main.go 里的 TODO"
```

  Reasonix 的选择理由非常具体：如果你主力用 DeepSeek 模型，它是目前唯一一个从底层为 DeepSeek prefix cache 优化的原生终端 Agent。Go 单二进制的分发方式意味着不需要 Node.js 运行时，在 CI 和 Docker 环境里特别干净。配置驱动的架构让团队可以通过 `reasonix.toml` 统一管理 Agent 行为，而不是每人手写 MCP JSON。不足是目前插件生态和社区 Skills 数量远少于 Claude Code。

## Munder Difflin — 把 Claude Code Agent 装进像素风办公室，让它们自己组队干活

  用过多 Agent 架构的开发者都知道一个尴尬的事实：你开了三个 Claude Code 终端窗口，Agent A 在修 bug，Agent B 在写测试，Agent C 在审查代码——但 A 不知道 B 写了什么，B 不知道 C 的审查结果，你得手动在三个窗口之间复制粘贴上下文。Munder Difflin 把它变成了一款 Electron 桌面应用：每个 Agent 是一个真实的 `claude` 进程（`node-pty` 伪终端），在屏幕上显示为一个像素风角色，坐在一张办公桌前，消息以信封动画在角色之间飞传。

  架构分两层。底层是 **Hive**（蜂巢）——一个本地 git 仓库承载的多 Agent 协作层：每个 Agent 有自己的身份和长期记忆（markdown-first，附带语义召回索引），原子文件邮箱系统（写 `outbox/`、路由投递到 `inbox/`），共享黑板和只追加事件日志。上层是 **GOD Agent**（Michael 的办公室）——一个常驻编排者 Agent，读取所有请求，自己能处理的直接处理，需要人工决策的升级到审批队列。

  这套设计解决了多 Agent 协作的几个关键问题。**内存隔离**：每个 Agent 的 `node-pty` 进程完全独立，一个崩溃不影响其他。**Git 隔离**：可选开关为每个 Agent 自动创建独立 git worktree，关闭时自动清理，Agent 永远不会在分支上冲突。**成本控制**：实时从 Claude Code 的 JSONL transcript 读取 token 用量和费用估算，每个 Agent 有 token 预算上限，超支触发断路器（steer → constrain → stop）。**可观测性**：OpenTelemetry collector 直播每个 Agent 的工具调用瀑布图和按模型归因的成本。

  这个项目只有 3 周历史（5 月 31 日创建），已经迭代了 11 个版本到 v0.2.1。v0.2.0 加入了完整的可观测性体系、SQLite 持久化、记忆压缩器（MemoryReflector）、任务看板、定时任务调度、GitHub issue 摄取和 CI 状态监控。桌面通知在 Agent 完成任务时弹出。视觉风格是 Animal Crossing × Earthbound × SNES 菜单 UI——像素对齐、矮胖、友好。

```bash
# 前提条件
# - Node.js 18+ 和 npm
# - C/C++ 工具链（node-pty 原生插件需要）
# - Claude Code 在 PATH 上

git clone https://github.com/chaitanyagiri/munder-difflin.git
cd munder-difflin
npm install          # postinstall 自动 rebuild node-pty
npm run dev          # 启动 Electron 应用（带热重载）
```

### 同类工具对比

| 工具 | 形式 | 核心特点 | Agent 隔离 | 可视化 | 热度 |
|------|------|---------|-----------|--------|------|
| **Munder Difflin**（今日推荐）| Electron 桌面应用 | GOD 编排、蜂巢协作、像素动画、git worktree 隔离 | ✅ 独立 PTY + worktree | ✅ 办公层 + 信封动画 | ⭐ 430 |
| Karajan | CLI | 本地多 Agent 编排、RAG、Story Board、审计 | ❌ 共享进程 | ❌ TUI only | npm 包 |
| AI-SDLC | CLI + TUI | 完整 SDLC 流水线、三审稿、DSSE 签名 | ✅ git worktree | ✅ TUI Dashboard | ⭐ 框架 |
| LangGraph | Python SDK | 有向图 Agent、持久状态、人工审批 | ❌ 框架级 | ❌ LangSmith 云 | ⭐ 24K |
| CrewAI | Python SDK | 角色扮演式多 Agent、最快原型搭建 | ❌ 框架级 | ❌ AMP 云 | ⭐ 46K |

  Munder Difflin 在这个赛道的独特性在于它根本不是框架——它是一个桌面应用。你不写 Python 代码定义 Agent 图，不配 YAML 声明工作流，你按一个按钮就 spawn 一个真实 Claude Code 终端。它的 GOD 编排模式意味着你不需要手动协调 Agent 之间的通信——这正好补齐了 LangGraph 和 CrewAI 做不到的事：把已有的终端 Agent CLI 进程直接可视化编排，而不是让你用 SDK 重新实现一套 Agent 逻辑。目前的状态仍然是早期原型，macOS 优先，Windows/Linux 测试有限，但核心概念已经在三周内被验证有效。
