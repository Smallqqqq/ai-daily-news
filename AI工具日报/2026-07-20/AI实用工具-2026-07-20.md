# AI 实用工具 · 2026-07-20

![[summary.png]]

  今天三个工具分别对应 AI 编码链条上三个正在被重新定义的环节：UI 如何生成（用约束替代自由发挥）、代码如何调试（让 Agent 自己能设断点）、AI 工具本身如何测试（用突变测试替代覆盖率幻觉）。

## json-render — Vercel 出品的生成式 UI 框架：AI 写 JSON，框架渲染成 React/Vue/Svelte 组件，不会乱编 CSS

  让 AI 生成 UI 的难题不在于"能不能生成"——GPT-4o 和 Claude 都能写出看起来像样的 React 组件。问题在于可靠性：模型会幻觉出不存在的组件名、编造不存在的 props、用一套谁也认不出来的 CSS。Vercel Labs 的 json-render 解决这个问题的方式很直接——把生成内容从"任意代码"降级为"受限的 JSON 树"，模型只能从你预先定义的组件目录里挑选。

  工作流程是：你定义 Catalog（可用组件清单 + props schema）→ AI 根据自然语言提示生成 JSON 规格 → Renderer 把 JSON 渲染成真实 UI。AI 生成的不是代码，是一份结构化数据——`{"component": "Card", "props": {"title": "用户列表"}, "children": [...]}`。框架确保这份 JSON 里的每个组件名都在 Catalog 里、每个 prop 都符合 schema。模型没有机会写 `<div style="color: #not-a-color">`，因为 Catalog 里根本没给他这个选项。

  实际体验比听起来更灵活。36 个预置的 shadcn/ui 组件开箱即用，覆盖了从 Button/Input 到 DataTable/Chart 的常规需求。Streaming 模式下，JSON 可以边生成边渲染——用户看到 UI 一块块"长出来"，而不是等 30 秒后刷新整个页面。框架支持 React、Vue、Svelte、Solid 四种 Web 框架，以及 React Native 和 React Three Fiber（是的，AI 可以生成 3D 场景的 JSON 描述然后渲染成 Gaussian Splat）。甚至还有 Next.js、Remotion（视频渲染）、React PDF、React Email 等特殊渲染器——同一个 JSON 树可以输出网页、视频、PDF 和邮件，因为数据层和渲染层是完全解耦的。

  一个值得注意的细节：json-render 也发布了 MCP Apps 集成包（`@json-render/mcp`），可以直接在 Claude、ChatGPT、Cursor 里用。模型在对话中生成的 UI JSON 通过 MCP 协议传给 Renderer，变成一个可交互的组件面板——不是截图，是活的 React 组件。

```bash
# React 项目
npm install @json-render/core @json-render/react

# 带 36 个预置 shadcn/ui 组件
npm install @json-render/shadcn

# Vue 项目
npm install @json-render/core @json-render/vue
```

## DebugMCP — 让 AI 编码 Agent 直接在 VS Code 里设断点、单步执行、检查变量、求值表达式

  AI 编码 Agent 处理 bug 的典型流程是：读错误日志 → 猜原因 → 改代码 → 跑测试 → 不行再猜。人类开发者不会这么干——我们会设断点、一步步看变量值、在调试上下文里求值表达式。DebugMCP 做的事就是把 VS Code 调试器变成一个 MCP Server，让任何 MCP 兼容的 AI Agent 都能直接操作调试器：设断点、启动调试会话、单步执行、检查变量、求值表达式——就像人在操作一样。

  安装后 VS Code 自动启动一个 MCP Server。Agent 通过 `start_debugging` 工具启动调试会话（指定文件和工作目录），用 `add_breakpoint` 在特定行设断点，用 `step_over` / `step_into` / `step_out` 控制执行流，用 `get_variables_values` 读取当前作用域内的变量（支持 local/global/all 过滤），用 `evaluate_expression` 在调试上下文中运行任意表达式。14 个工具覆盖了调试完整生命周期。

  附带一个 `/debug` Skill，安装后 Agent 自动获得系统性调试工作流知识——什么时候用断点而不是 grep、怎么从变量值推断根因、如何通过表达式求值验证假设。Skill 会在支持 Agent Skills 的客户端（Copilot、Codex、Cline 等）自动注册。

  安全模型值得注意。调试器是 VS Code 中最强大的功能之一——它可以在运行时暂停程序、读取内存中的任何值、甚至通过表达式求值执行代码。DebugMCP 100% 本地运行，不通过网络传输代码。但把调试器交给 AI Agent 意味着 Agent 理论上可以读取调试进程中的所有内存数据（包括密钥、token、用户数据）。官方建议在受信任的仓库中使用，生产环境调试需要人工审查。

```bash
# 在 VS Code 中安装
# 方式一：扩展商店搜索 "DebugMCP" 安装
# 方式二：命令行安装
code --install-extension ozzafar.debugmcpextension

# 安装后自动启动 MCP Server
# Agent 端自动检测并注册，无需手动配置
```

## testai — AI 原生代码的测试框架：像测普通代码一样测你的 Skills、MCP 服务器、Slash Commands 和 LLM 调用

  如果你在开发 Claude Code Skills、MCP 服务器或者任何依赖 LLM 调用的工具，你一定遇到过这个问题：改了一个 prompt，怎么知道它没有悄悄变差？传统的做法是——手动跑几遍，看看输出对不对，然后祈祷。testai 把这个问题变成了一个可重复的、有断言的测试流程。

  testai 的核心抽象是 `TestAI.createTestingEnv`——创建一个沙盒测试环境，里面有虚拟工作区（`createProject`）和可追踪的 Mock MCP 服务器（`createMcp`）。你的测试代码可以启动一个 Agent、给它一个 prompt、断言它调用了哪些 MCP 工具、产出了什么文件、甚至输出内容是否符合预期。框架无关——支持 Vitest、Jest、Node test runner 随便选。

  最有价值的是一组预置的 MCP 存根（`@testingai/mcp-*`）：Datadog、Slack、Figma、Linear 各有一个可追踪的 Mock 实现。Agent 调用 `mcp__slack__send_message` 时，Mock 记录下"被调用了，参数是这些"，但不真的发消息。测试结束后你可以断言 `expect(slackStub.sentMessages).toContainEqual({channel: "#alerts", text: "build failed"})`。这意味着你可以在不连接任何外部服务的情况下，验证 Agent 的行为是否正确。

  项目还在早期（5 月 12 日创建，98 stars），但 API 设计已经成熟——文档站 testingai.ai 有完整的使用指南，内置了 React、Nuxt、FastAPI 等预制项目夹具。testai 不测试 LLM 模型本身的质量（那是 benchmark 的事），它测试的是你用 LLM 构建的那层逻辑——Skill 的流程控制、MCP 工具的调用顺序、Agent 的决策路径——这些东西和普通代码一样需要回归测试。

```bash
# 安装
npm install -D @testingai/testai

# 基础用法示例
import { TestAI, createProject } from "@testingai/testai";
import { createMcpSlack } from "@testingai/mcp-slack";

const env = TestAI.createTestingEnv();
const project = await createProject("react"); // 预制 React 项目模板
const slack = createMcpSlack();

// 启动 Agent，给 prompt，断言 MCP 工具调用
const result = await env.runAgent({
  prompt: "Review src/App.tsx and post findings to #code-review",
  mcpServers: { slack },
  workspace: project,
});

expect(slack.sentMessages).toHaveLength(1);
expect(slack.sentMessages[0].channel).toBe("#code-review");
```
