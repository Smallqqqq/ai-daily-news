## GitHub MCP Server v1.1.0 — 搜索提交、CSV 输出、Issue 字段，三个小而美的实用升级

  GitHub MCP Server 是 GitHub 官方维护的 MCP 服务器，让 Claude Code、Cursor、Codex 等任何 MCP 客户端直接操作 GitHub——读仓库、管 Issue、查 PR、看 diff。30K stars，MIT 协议，GitHub 团队持续高频迭代。5 月 28 日到 29 日连发三个版本（v1.1.0 → v1.1.1 → v1.1.2），其中 v1.1.0 是功能性大更新。

  这次更新的四个新能力，每一个都精准打在日常使用摩擦点上：

  **`search_commits` 工具**填补了一个长期空白。之前 agent 只能列出某个仓库的提交历史，无法跨仓库搜索特定提交——调试回归问题时想知道"那个改了认证逻辑的 commit 在哪"，审计安全事件时想找"上周谁改了密钥配置"，都得手动翻。现在一句自然语言"搜索上周修改了认证中间件的提交"，agent 直接用 GitHub 搜索语法定位。

  **CSV 输出模式**是个 token 节省利器。默认的 JSON 格式，`list_issues` 返回 50 条 issue 动辄上万个 token，其中大半是重复的 schema 结构。CSV 模式扁平化输出，同样的数据量体积大幅缩小。目前在 Insiders Mode 下默认开启，不久后会全面放开。

  **Issue 字段支持**让 agent 能读写项目自定义字段（优先级、迭代、估算等）。之前 agent 只能操作 Issue 的标题、正文、标签和状态，项目管理的核心字段碰不到。现在 `list_issue_fields`、按字段值过滤、读写字段值，一套齐全。

  **精简的项目条目响应**显著减少了 `list_project_items` 等工具的返回体积。之前返回的是完整的 Issue/PR payload，现在只返回紧凑的项目条目摘要，去掉不必要的上下文。

```bash
# Docker 方式（推荐）
claude mcp add github --transport stdio -- docker run -i --rm -e GITHUB_PERSONAL_ACCESS_TOKEN ghcr.io/github/github-mcp-server

# 或 npx 方式
claude mcp add github --transport stdio -- npx @anthropic-ai/github-mcp-server
```

## trace-mcp — 用依赖图谱替代 80 次 Grep 和 190 次文件读取

  AI 编码 agent 在代码库中的工作方式有个根本问题：它在"盲目搜索"。你问 agent"改这个 Model 会影响哪些地方"，它的做法是疯狂 Grep 类名、Read 文件、Grep 方法名、Read 更多文件——每轮对话可能有 80 次 Grep 调用和 190 次文件读取，大部分 token 烧在无意义的探索上。trace-mcp 的答案是：先建好跨语言的依赖图谱，一次工具调用替代几十次盲目搜索。

  trace-mcp 的工作原理分三层。底层扫描源代码，通过 tree-sitter 和 LSP 解析 81 种语言、58 种框架的符号、导入、调用关系、DI 注入、ORM 映射、路由注册，构建一张跨语言、跨框架的依赖图。中间层通过 MCP 暴露 138 个工具——`search_symbols`、`get_change_impact`、`find_callers`、`get_execution_path` 等等。顶层可选安装 PreToolUse hook，拦截 agent 的 Grep/Read 调用并重定向到 trace-mcp 工具，实现硬执行。

  最实用的功能是 `get_change_impact`。改一个 Eloquent Model 的字段，trace-mcp 能追踪影响链：Model → Controller → Service → Vue 组件 → 数据库迁移 → 测试用例，跨 PHP、TypeScript、SQL 三个语言一次性给出完整爆炸半径。官方数据：一次调用替代约 80 次 Grep 和 190 次文件读取，最高节省 99% token。

  另一个亮点是**决策记忆**。Agent 会话中的架构决策、技术选型、bug 根因，通常随着 `/compact` 或会话结束就消失了。trace-mcp 从 Claude Code 的 JSONL 日志中自动挖掘决策，链接到对应的代码符号。下次有人问"为什么这个连接池大小设成 20"，agent 能直接调出当时的决策记录。

  v1.40.0 附带的 Electron 桌面应用内置了 GPU 加速的依赖图浏览器——基于 cosmos.gl 渲染，数万个节点/边在交互帧率下流畅操作，支持按文件/符号过滤、社区检测、深度步进。

```bash
# 安装
npm install -g trace-mcp

# 初始化（配置 MCP 客户端、安装 hook、添加 CLAUDE.md 路由规则）
trace-mcp init

# 注册当前项目
trace-mcp add
```

### 同类工具对比

| 工具 | 核心机制 | 语言覆盖 | 跨语言边 | MCP 工具数 | 热度 |
|------|---------|---------|---------|-----------|------|
| **trace-mcp**（今日推荐）| tree-sitter + LSP 依赖图谱 | 81 语言 + 58 框架 | ✅ 框架感知 | 138 | ⭐ 80+ |
| codedev-mcp | ripgrep + TF-IDF + tree-sitter | 40+ 语言 | ❌ 单语言 | 33 | ⭐ 较新 |
| codeaware-mcp | Rust tree-sitter + 语义索引 | 多语言 | ❌ 基础 | ~10 | ⭐ 较新 |
| Continuum | 常驻 daemon + 混合搜索 | 5 语言 | ❌ 基础 | 12 | ⭐ 较新 |
| codescout | LSP + ONNX 语义搜索 | 9 语言 | ❌ 单语言 | ~20 | ⭐ 较新 |

  trace-mcp 在这个赛道上的核心差异有三：第一，跨语言框架边——它能理解 Laravel Controller → Eloquent Model → Vue 组件之间的连接，而不是把每种语言当成独立岛屿；第二，138 个工具覆盖了从符号导航到变更影响分析到会话记忆的完整链路，用不着再装其他代码智能 MCP；第三，三层递进的执行策略（从软提示到 CLAUDE.md 路由到硬 hook）让你可以按项目需求调整控制粒度。
