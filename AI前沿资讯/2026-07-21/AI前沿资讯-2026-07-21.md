## 1. Claude Fable 5 推翻 Jacobian 猜想——87 年数学难题在世界杯决赛夜被 AI 破解

**评分: 9.5/10 | 可操作性: 低（科学突破） | 热度: X 120 万浏览 + HN 679 points / 440 评论 + YouTube 多频道报道**

**是什么：**
数学家 Levent Alpöge 使用 Claude Fable 5，在 2026 世界杯决赛期间找到了 Jacobian 猜想（1939 年提出，Smale 21 世纪数学难题之一）的反例。猜想断言：若多项式映射的 Jacobi 行列式为非零常数，则映射必有多项式逆映射。Fable 5 给出的反例是一个从 ℂ³ → ℂ³ 的 7 次多项式映射——Jacobi 行列式恒为 -2，却将三个不同点映射到同一输出，证明猜想为假。验证只需 8 行 Sage 代码。这是继 5 月 GPT-5.6 推翻 Erdős 单位距离猜想、7 月 Codex 并行解决 20 个 Erdős 问题后，AI 数学发现的又一里程碑。

**怎么用：**
```bash
# 验证反例（8 行 Sage/Python）
# 来源：jacobianfun.org
uv run --with sympy python verify.py

# 完整验证仓库
git clone https://github.com/algalg/jacobianfun.org
cd jacobianfun.org
uv run --with sympy python verify_page_facts.py

# 论文级解析
# https://jacobianfun.org/jacobian-explained
```

**为什么重要：**
这是 2026 年 AI 数学发现的第三块里程碑，三个月内三连击：GPT-5.6 推翻单位距离猜想（5/20）→ Codex 并行解决 20 个 Erdős 问题（7 月）→ Fable 5 推翻 Jacobian 猜想（7/19）。反例的验证只需 8 行代码——"验证是平凡的，推导是不可审计的"，这种不对称性正在重塑数学研究的范式。此前 AlphaProof Nexus、Claude Mythos 和 AxiProver 分别在 Erdős 问题、单位距离猜想和代数几何问题上取得突破，Lean 形式化验证已成为 AI 数学输出的标准校验层。

**讨论现场：**

- HN 另一热评："任何傻瓜都能做这个，不过是高中微积分而已。随机鹦鹉，spicy autocomplete。等我看到 AI 做了真正的事情再说" —— 社区戏仿 AI 怀疑论者的经典反应

## 7. kvcache-ai/ktransformers——LLM 异构推理优化框架，458 stars/day

**评分: 7.5/10 | 可操作性: 高 | 热度: GitHub 458 stars/day，18,754 total**

**是什么：**
ktransformers 是一个灵活的 LLM 推理/微调优化框架，核心卖点是异构硬件加速——支持 CPU+GPU 混合推理，让消费级硬件也能跑大模型。基于 PyTorch，提供一键式模型部署、多种量化方案（GPTQ/AWQ/GGUF）、vLLM 兼容 API。特别适合在 RTX 4090 等消费级 GPU 上跑 70B+ 模型。

**怎么用：**
```bash
# 安装
pip install ktransformers

# 启动推理服务（异构模式：GPU + CPU 混合）
python -m ktransformers.server \
  --model-path Qwen/Qwen3.8-2.4T \
  --quantization awq \
  --port 8000

# API 调用（OpenAI 兼容）
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "qwen3.8", "messages": [{"role": "user", "content": "写一个快速排序"}]}'
```

**为什么重要：**
开源大模型的参数规模已经突破万亿（Kimi K3 2.8T、Qwen 3.8 2.4T），但绝大多数开发者没有 A100/H100 集群。ktransformers 的 CPU+GPU 异构推理让"用消费级硬件跑前沿模型"变得可行——通过将 attention 层放 GPU、FFN 层放 CPU + 量化，在 RTX 4090 + 64GB RAM 上就能跑百亿参数模型。对比 vLLM 纯 GPU 方案，ktransformers 的门槛低一个数量级。

**讨论现场：**
- 社区关注 K3 权重 7/27 发布后 ktransformers 的适配速度
- 开发者实测：DeepSeek V4 在 RTX 4090 + 64GB RAM 上通过 ktransformers 可达 5-8 token/s
- 与 llama.cpp 互补：llama.cpp 侧重 CPU 推理，ktransformers 侧重 CPU+GPU 混合

## 9. topoteretes/cognee——开源 AI Agent 记忆平台，知识图谱驱动的持久化记忆

**评分: 7.0/10 | 可操作性: 高 | 热度: GitHub 234 stars/day，28,819 total**

**是什么：**
cognee 是面向 AI Agent 的开源记忆平台，使用知识图谱（而非简单向量检索）构建 Agent 的持久化长期记忆。支持多种 LLM 后端（OpenAI/Anthropic/本地模型）、多种向量数据库（ChromaDB/Qdrant/Weaviate/PGVector）、Neo4j 图数据库。解决 Agent 的"失忆"问题——跨会话记住用户偏好、项目上下文和历史决策。

**怎么用：**
```bash
# 安装
pip install cognee

# 快速启动（使用内存中的向量存储 + 网络图）
cognee init

# Python API
from cognee import Cognee

agent = Cognee()
await agent.add_memory("用户偏好Python异步编程风格")
await agent.add_memory("项目使用FastAPI + SQLAlchemy + Redis")

# 检索相关记忆
memories = await agent.search("用户的编程偏好是什么？")
print(memories)

# MCP Server 集成
cognee mcp install  # 注册到 Claude Code
```

**为什么重要：**
Agent 记忆是 2026 年 AI 工程的核心瓶颈之一。向量检索（RAG）解决"找相关文档"，但解决不了"记住用户过去说了什么、偏好是什么、上次的决策逻辑是什么"。cognee 用知识图谱替代向量检索，存储的不是文档片段而是实体-关系-属性三元组，这让记忆可推理、可追溯、可更新。对比 MemPalace(57.5K stars)，cognee 的知识图谱方法更适合需要复杂关系建模的场景。

**讨论现场：**
- 社区关注 cognee 与 LangChain Memory/OpenAI Memory 的对比——知识图谱方案在关系推理上明显优于向量方案
- 有开发者将 cognee 与 code-review-graph 组合使用，构建"代码知识图谱 + Agent 记忆图谱"的双图架构

## 10. Cursor 官方博文：Agent Swarms 与新模型经济学

**评分: 6.5/10 | 可操作性: 中（方法论） | 热度: HN 125 points / 51 评论**

**是什么：**
Cursor 官方博客 7 月 20 日发布深度文章，探讨 Agent Swarm（Agent 集群）如何改变 AI 编码的经济学模型。核心论点：当模型 API 成本持续下降（Kimi K3 $3/$15 vs Fable 5 $10/$50），"多用便宜模型并行 + 只在关键时刻用贵模型"的策略正在成为最优解。文章给出了具体的工作流设计建议：99% 的编辑用快速模型，1% 的关键决策用前沿模型。

**怎么用：**
```bash
# Cursor 的 Agent Swarm 模式已在产品中可用
# 在 Cursor 设置中启用 Agent Mode → 选择 Parallel 策略

# 方法论通用（适用于任何编码 Agent）：
# 1. 日常编辑/格式化/简单重构 → Sonnet/Kimi K3 Fast
# 2. 复杂架构决策/代码审查 → Fable 5/Opus
# 3. 大量并行搜索/测试 → DeepSeek V4（最便宜）
```

**为什么重要：**
