## 1. Meituan LongCat-2.0（美团 1.6T 全华芯训练开源模型）

**评分: 9.0/10 | 可操作性: 高 | 热度: VentureBeat/SCMP/TechCrunch 头条 + OpenRouter 使用量 Top 3**

**是什么：**
美团于 6 月 30 日正式开源 LongCat-2.0，一个 1.6 万亿参数 Mixture-of-Experts 模型，MIT 许可证。核心爆点：这是全球首个从预训练到推理全流程运行在国产 AI ASIC 集群（约 5 万张卡，非 NVIDIA）上的万亿参数模型。此前它以 "Owl Alpha" 匿名代号在 OpenRouter 上排名使用量前三，跟 GPT-5.5、Gemini 3.1 Pro 同台竞技。架构采用 LongCat Sparse Attention（LSA），原生 1M token 上下文，每 token 动态激活 33B-56B 参数（均值约 48B），预训练数据 30T+ tokens。

**怎么用：**
```bash
# Hugging Face 下载权重（MIT 许可证）
huggingface-cli download meituan-longcat/LongCat-2.0

# OpenRouter API 调用
curl https://openrouter.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $OPENROUTER_API_KEY" \
  -d '{"model": "meituan-longcat/longcat-2.0", "messages": [{"role": "user", "content": "Write a Python function to..."}]}'

# 在线体验
# https://longcat.ai
```

**为什么重要：**
SWE-bench Pro 得分 59.5，Terminal-Bench 70.8，与 GPT-5.5（58.6%）和 Gemini 3.1 Pro 处于同一梯队。API 定价 $0.75/$2.95 每百万 token（促销价 $0.30/$1.20），比 GPT-5.5 便宜 70% 以上。最关键的是——一家外卖公司用纯国产芯片训练出了准前沿模型，证明 NVIDIA 出口管制无法阻挡中国 AI 迭代。同时以 MIT 许可证开源 1.6T 模型，给"前沿模型必须闭源"的说法一记重拳。

**讨论现场：**
- VentureBeat："这标志着全球技术基础设施的决定性拐点——如果中国能用自研 ASIC 持续迭代万亿参数架构，NVIDIA 在 AI 训练领域的统治地位将受到根本性威胁"
- X 上多位开发者实测反馈：通过 OpenRouter API 调用时 Agent 编码能力接近 Claude Opus 4.6，但免费 Chat 站点体验较差（可能是量化或限流原因）
- HN 相关讨论聚焦：美团从外卖切入 AI 的跨界冲击——"如果送外卖的公司都能训出 1.6T 模型，那 OpenAI 烧了几百亿究竟花在哪了"

**来源：** https://www.longcatai.org/models/longcat-2.html | https://venturebeat.com/technology/meituan-open-sources-longcat-2-0 | https://huggingface.co/meituan-longcat/LongCat-2.0

## 2. ChromeDevTools MCP（Chrome DevTools  for  AI  Agent 官方  MCP  Server）

**评分: 8.5/10 | 可操作性: 高 | 热度: GitHub Trending 今日 | Chrome DevTools 团队官方维护**

**是什么：**
Chrome DevTools 团队发布的官方 MCP Server，让 Claude Code、Codex、Cursor 等 AI 编码 Agent 能直接操控 Chrome 浏览器——包括页面导航、元素点击、表单填写、截图、性能分析、网络请求检查等完整 DevTools 能力。本质上是把 Chrome 变成了 AI Agent 的原生运行时环境，无需 Playwright 或 Puppeteer 等第三方中间层。

**怎么用：**
```bash
# 安装
npm install -g chrome-devtools-mcp

# 启动 MCP Server（连接本地 Chrome）
npx chrome-devtools-mcp --browserUrl=http://127.0.0.1:9222

# Claude Code 配置（.claude/.mcp.json）
{
  "mcpServers": {
    "chrome-devtools": {
      "command": "npx",
      "args": ["-y", "chrome-devtools-mcp"]
    }
  }
}

# 启动 Chrome 时开启远程调试
chrome --remote-debugging-port=9222
```

**为什么重要：**
之前 AI Agent 操控浏览器需要依赖 Playwright MCP 等第三方方案，兼容性和性能参差不齐。Chrome DevTools 团队亲自下场意味着：直接访问 Chrome DevTools Protocol 底层 API，零中间损耗；支持完整的性能分析（Core Web Vitals、Lighthouse）、网络请求拦截和修改；由 Chrome 团队持续维护，不会像社区方案那样断更。这对做 Web 自动化测试、爬虫、前端调试的开发者是基础设施级升级。

**讨论现场：**
- HN 相关讨论："这是 MCP 生态成熟的标志——浏览器厂商亲自为 AI Agent 提供一等公民接口，而不是让 Agent 去模拟人类点击"
- X 上多位前端开发者实测：截图和性能分析功能比 Playwright MCP 快 3-5 倍，因为直接走 CDP 而非模拟用户交互
- GitHub issues 中社区请求最多的是支持移动端模拟和多标签页管理

**来源：** https://github.com/ChromeDevTools/chrome-devtools-mcp

## 3. NanoBanana 2 Lite（Google 超快廉价 AI 图像生成）

**评分: 8.0/10 | 可操作性: 高 | 热度: Google 官方博客 + ThursdAI 覆盖 + 多平台转推**

**是什么：**
Google DeepMind 于 7 月 2 日发布 NanoBanana 2 Lite（Gemini 3.1 Flash Image），是目前最快、最便宜的 Gemini 图像生成模型。生成一张图不到 4 秒，价格为 $0.034/千张，画质超过初代 NanoBanana。同周 Gemini Omni Flash 也进入 GA 阶段，支持原生多模态视频工作流。

**怎么用：**
```bash
# 通过 Gemini API 调用
curl https://generativelanguage.googleapis.com/v1beta/models/gemini-3.1-flash-image:generateContent \
  -H "Content-Type: application/json" \
  -H "x-goog-api-key: $GEMINI_API_KEY" \
  -d '{"contents":[{"parts":[{"text":"A cat wearing a spacesuit on Mars, photorealistic"}]}]}'

# Python SDK
import google.generativeai as genai
genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("gemini-3.1-flash-image")
response = model.generate_content("A futuristic city skyline at sunset")
```

**为什么重要：**
$0.034/千张意味着生成一张图只需 0.0034 美分，比 Midjourney（约 $0.01/张订阅摊销）便宜约 3 倍，比 DALL-E 3（$0.04/张）便宜 10 倍以上。速度不到 4 秒，适合批量生成场景（电商产品图、社交媒体素材、游戏资产）。Gemini Omni Flash GA 同时补齐了视频模态，开发者现在能用单一 API 覆盖文/图/视频全模态。

**讨论现场：**
- X 上 @OfficialLoganK 实测对比：NanoBanana 2 Lite vs Midjourney V7，同 prompt 下画质差距不大，但 Lite 版在文字渲染上明显优于 MJ
- 社区关注焦点：价格+速度组合对于需要批量生产的商业场景（数千张产品图）是降维打击

**来源：** https://blog.google/innovation-and-ai/technology/ai/google-ai-updates-june-2026

## 4. meetily（纯本地 AI 会议助手，4 倍速 Whisper 转录）

**评分: 8.0/10 | 可操作性: 高 | 热度: GitHub Trending 今日 + Trendshift #3 | 100% 本地处理**

**是什么：**
Zackriya-Solutions/meetily 是一个开源的隐私优先 AI 会议助手，100% 本地处理，无需云端。基于 Rust 构建，使用 Parakeet/Whisper 实现 4 倍速实时转录，支持说话人分离（speaker diarization），用 Ollama 做本地摘要。同时提供 macOS 和 Windows 原生客户端。不录视频、不传云端、不存服务器——所有数据留在本地。

**怎么用：**
```bash
git clone https://github.com/Zackriya-Solutions/meetily.git
cd meetily

# 安装依赖
cargo build --release

# 启动（需先安装 Ollama 并拉取模型）
ollama pull llama3.2
./target/release/meetily
```

**为什么重要：**
Zoom AI Companion、Fireflies.ai、Otter.ai 等会议助手都需要将音频上传到云端处理——这对于讨论敏感信息的内部会议是不可接受的风险。meetily 用 Rust + Whisper + Ollama 全本地栈解决了这个痛点。4 倍速转录意味着 1 小时会议只需 15 分钟处理完毕。在企业合规要求越来越严格（尤其是金融、医疗、法律行业）的背景下，纯本地处理是不妥协的方案。

**讨论现场：**
- Trendshift 评论区："用 Rust 做 AI 应用是个值得关注的方向——性能比 Python 方案高出数倍，但生态还需要补课"
- GitHub issues 中用户最关心的是支持更多语言（目前已支持英文，中文/日文在路上）

**来源：** https://github.com/Zackriya-Solutions/meetily

**讨论现场：**

## 6. terax-ai（7MB 超轻量终端 AI 开发工作区）

**评分: 7.5/10 | 可操作性: 高 | 热度: GitHub Trending 今日 | 仅 7MB 体积**

**是什么：**
crynta/terax-ai 是一个超轻量的终端优先 AI 原生开发工作区，整个应用仅 7MB。它直接在终端内提供代码编辑、AI Agent 交互、项目管理等功能，不依赖 Electron 或 Web 技术栈。定位是 VSCode + Claude Code 的极简融合版，专注于速度和低资源占用。

**怎么用：**
```bash
# 安装
npm install -g terax-ai

# 启动工作区
terax

# 在终端内切分面板、调 AI agent、编辑代码
```

**为什么重要：**
主流 AI 编码工具越来越重——VSCode + Copilot 插件动辄 500MB+ 内存，Cursor 基于 Electron 同样笨重。terax-ai 的 7MB 体积意味着可以在树莓派、低配 VPS、甚至 Android Termux 上流畅运行。它的设计哲学与 2026 年的"终端复兴"趋势一致——Claude Code、Codex CLI、OpenCode 等终端 Agent 证明了终端是 AI 编码的最佳界面，terax-ai 把这一理念推到了极致。

**讨论现场：**
- GitHub 讨论区："7MB 的 AI IDE 听起来像是个噱头，但实际跑起来确实快——冷启动不到 200ms"

## 7. Termi Protocol（AI Agent 3D 实时可视化）

**评分: 7.5/10 | 可操作性: 高 | 热度: Product Hunt #5（7月4日）+ PH CEO 推荐**

**是什么：**
Termi Protocol 将 AI 编码 Agent 的工作流可视化为 3D 场景——你的 Agent 有一个机器人形象，坐在虚拟办公桌前阅读文件、写代码、跑命令。当 Agent 遇到 bug 时，机器人会抖动并发出警告。目标不只是"好看"——当多个 Agent 并行工作时，3D 可视化比一墙滚动的终端日志更容易发现问题和理解 Agent 的行为意图。

**怎么用：**
```bash
# 下载桌面应用
# https://termiprotocol.com

# 目前支持 Claude Code / Codex / Cursor
# 启动后自动检测本地 Agent 进程并可视化
```

**为什么重要：**
AI Agent 的"可观察性"是 2026 年最大的未解决问题之一。当 Agent 自主运行数小时并修改数百个文件时，开发者需要一种比 `git diff` 和终端日志更直观的理解方式。Termi Protocol 把 DevOps 的 observability 理念带入了 Agent 开发——不是替代日志，而是给日志增加空间维度和异常可视化。

**讨论现场：**
- Product Hunt 评论："让 Agent 在 3D 世界里工作是真心有趣。我看了几年 Agent 日志，它们从来不展示意图。这个视角能提示'为什么'而不只是'做了什么'吗？"
- Termi 作者回复："当 Agent 遇到 bug 时，机器人会抖动、发出提示音、头顶显示红色警告——问题被加入历史记录，你可以看到发生了什么以及为什么"

**来源：** https://www.producthunt.com/products/termi-protocol
