## 1. bradautomates / claude-video

**评分: 9.5/10 | 可操作性: 高 | 热度: GitHub 951 stars/day**

**是什么：**
让 Claude 具备"看视频"能力的开源 CLI 工具。用户只需执行 `/watch` 命令，工具自动下载视频、提取关键帧、调用 Whisper 转录音频，一次性将完整上下文交给 Claude 分析。由独立开发者 bradautomates 用 Python 构建，一经发布即登上 GitHub Trending 榜首。

**怎么用：**
```bash
git clone https://github.com/bradautomates/claude-video.git
cd claude-video
pip install -r requirements.txt
# 在 Claude Code 中使用 /watch 命令即可分析视频
```

**为什么重要：**
视频理解是目前 AI Agent 的薄弱环节——多数 Agent 只能处理文本，最多看图。claude-video 以最轻量的方式（不依赖第三方 API）打通了"视频→帧→文本→Agent 分析"这条链路，单日 951 stars 说明开发者对这个模式有真实需求。对比同类方案（如 yt-dlp + 手动传帧），它将整个流程压缩为一行命令。

**讨论现场：**
- HN 用户："这是我见过的最实用的 Claude Code 插件之一，已经用它分析了 3 个会议录像。"
- X 上 @bradautomates 的首发推文获 200+ 转发，多位 AI 开发者表示已集成到自己的工作流。

## 3. TencentCloud / CubeSandbox

**评分: 8.5/10 | 可操作性: 高 | 热度: GitHub 564 stars/day**

**是什么：**
腾讯云开源的 AI Agent 安全沙箱，用 Rust 语言构建。提供即时启动、高并发、轻量级隔离环境，让 AI Agent 可以在受控环境中执行任意代码而不会威胁宿主机。支持一键创建和销毁沙箱实例，毫秒级冷启动。

**怎么用：**

## 6. TencentCloud / TencentDB-Agent-Memory

**评分: 7.8/10 | 可操作性: 高 | 热度: GitHub 318 stars/day**

**是什么：**
腾讯云发布的 AI Agent 本地长期记忆系统，采用 4 级渐进式管道（Progressive Pipeline）架构：短期缓存→语义摘要→结构化知识图谱→长期持久化存储。全程本地运行，零外部 API 依赖，TypeScript 实现。

**怎么用：**
```bash
git clone https://github.com/TencentCloud/TencentDB-Agent-Memory.git
cd TencentDB-Agent-Memory
npm install
npm run build
# 启动记忆服务
npm start
```

**为什么重要：**
Agent 记忆是 2026 年 AI Agent 领域最明确的痛点之一——没有记忆的 Agent 每次对话都从零开始。TencentDB-Agent-Memory 的创新在于 4 级管道设计：不把所有对话都存进一个向量库，而是按重要性自动分级——闲聊过滤掉、决策类对话摘要化、关键事实结构化。对比 Mem0（52K stars）的单一向量存储方案，这种分层策略在检索精度和存储效率上有明显优势。

**讨论现场：**
- GitHub 讨论区有用户测试后反馈：在 1000 轮对话后，关键信息召回率比纯向量方案高 34%。
- 社区关注点：TypeScript 选型是否限制了 Python 生态的集成。

**来源：** https://github.com/TencentCloud/TencentDB-Agent-Memory

## 10. AgentLens——编码 Agent 评估新基准

**评分: 6.5/10 | 可操作性: 中 | 热度: arXiv 新提交 + 开源**

**是什么：**
来自学术界的新论文，提出了 AgentLens——一个结合形式化验证与 LLM 轨迹评审的编码 Agent 评估基准。不仅看 Agent 生成的代码对不对，还评估 Agent 在执行过程中的决策质量（是否走了弯路、是否有不必要的操作等）。已开源基准数据集和评估框架。

**怎么用：**
```bash
git clone https://github.com/[作者]/AgentLens.git
pip install -e .
python evaluate.py --agent <your_agent> --dataset agentlens-full
```

**为什么重要：**
现有编码 Agent 评估（如 SWE-Bench）只看最终结果——代码能不能通过测试。AgentLens 的创新在于评估"过程质量"：Agent 是否花了不必要的 token？是否做了多余的文件读取？这类评估对于优化 Agent 的 token 经济有直接价值。论文中测试了 Claude Opus 4.7 和 GPT-5.5，发现两者在"过程效率"维度上的差距（23%）远大于最终结果差距（3%），说明只看最终分数的评估方法在漏掉重要信息。

**讨论现场：**
- arXiv 评论区："Process-level evaluation is the missing piece. SWE-Bench tells you if the code works, AgentLens tells you if the agent was smart about it."
- Hugging Face Papers 页面有 15 个讨论，关注如何将此基准集成到现有 CI 流程中。

**来源：** https://arxiv.org/abs/2607.06624
