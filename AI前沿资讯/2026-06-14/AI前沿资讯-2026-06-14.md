## 6. LMCache：LLM 推理加速 KV Cache 层，单日 238 star

**评分: 7.3/10 | 可操作性: 高 | 热度: GitHub Trending #4，238 stars/day，累计 8,889 stars**

**是什么：**
LMCache 是一个专注于 LLM 推理加速的 KV Cache 优化层，通过在内存/SSD 之间智能缓存和预取注意力机制的 Key-Value 状态，显著降低长上下文推理的延迟和成本。支持 vLLM、SGLang 等主流推理框架。核心原理是在多个请求间共享 KV Cache，避免重复计算——尤其适合多轮对话、RAG 检索和 Agent 工作流等场景。

**怎么用：**
```bash
pip install lmcache
# 在 vLLM 中启用 LMCache
vllm serve meta-llama/Llama-4-8B \
  --enable-lmcache \
  --lmcache-config '{"chunk_size": 256, "backend": "cpu"}'
```

**为什么重要：**
长上下文推理的成本是阻碍 Agent 和 RAG 系统规模化部署的主要瓶颈。LMCache 声称可将多轮对话场景的推理延迟降低 3-5x，TTFT（Time to First Token）缩短 70% 以上。对于运行本地模型的开发者，这意味着用相同硬件可以支撑更多并发用户。8,889 star 且持续高速增长说明缓存优化是当前 LLM 基础设施最热门的投资方向之一。

**讨论现场：**

## 10. Google Gemini-SQL2：Text-to-SQL 新王者，BIRD 榜单 80.04%

**评分: 6.5/10 | 可操作性: 高（API 已可用）| 热度: Reddit r/accelerate 日报收录，专业社区热议**

**是什么：**
Google 于 6 月 13 日发布 Gemini-SQL2，基于 Gemini 3.1 Pro 微调的专用 Text-to-SQL 模型，在 BIRD（Big Bench for Large-Scale Database Grounded Text-to-SQL Evaluation）单模型排行榜上达到 80.04% 的执行准确率，刷新了单模型（非 ensemble）最佳成绩。BIRD 是 Text-to-SQL 领域最严格的大规模基准测试，覆盖 95 个真实数据库、跨越 37 个专业领域。

**怎么用：**
```bash
# 通过 Google Gemini API 调用

  -H "Content-Type: application/json" \
  -H "x-goog-api-key: $GEMINI_API_KEY" \
  -d '{
    "contents": [{"parts": [{"text": "Database schema:\nCREATE TABLE orders(...)\n\nQuery: 上月销售额最高的前10个产品"}]}],
```
