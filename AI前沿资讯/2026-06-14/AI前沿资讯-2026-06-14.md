## 6. LMCache — LLM 推理加速 KV Cache 层，单日 238 star

**评分: 7.3/10 | 可操作性: 高 | 热度: GitHub Trending #4 · 238 stars/day · 累计 8,902 stars**

**是什么：**
LMCache 是一个专注于 LLM 推理加速的 KV Cache 优化层，通过在 GPU/CPU/磁盘之间智能缓存和预取 KV Cache，显著降低长上下文推理的延迟和成本。核心思路是减少重复计算——当多个请求共享相同前缀（如 system prompt、few-shot examples）时自动复用已计算的 KV Cache。支持 vLLM、SGLang 等主流推理框架的即插即用集成。

**怎么用：**
```bash
pip install lmcache
# 在 vLLM 中启用 LMCache
vllm serve meta-llama/Llama-4-8B \
  --enable-lmcache \
  --lmcache-config '{"chunk_size": 256, "backend": "cpu"}'
```

**为什么重要：**
在 Agent 场景中，system prompt 和工具定义通常占 2K-10K token，每次调用都重复推理这些前缀是巨大的浪费。LMCache 实测可将前缀重复推理延迟降低 50-70%，成本节省 30-50%。与上周大火的 Headroom（token 压缩，+14,266 星）形成互补——一个压缩上下文、一个复用缓存。

**讨论现场：**
- GitHub Issues 用户反馈：在 50+ 轮次的 Agent 会话中，LMCache 将端到端延迟从 12 秒降到 4 秒
- Reddit r/LocalLLaMA："Combined with Qwen 3.6 on RTX 5080, LMCache brings 1M context inference to consumer GPUs"

**来源：** https://github.com/LMCache/LMCache
