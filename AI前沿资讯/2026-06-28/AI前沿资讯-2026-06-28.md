## 1. design.md：Google 开源视觉标识规范格式，让 AI Coding Agent 看懂设计系统

**评分: 9.2/10 | 可操作性: 极高 | 热度: GitHub 22.4K★ | +1,541 stars/day | #1 GitHub Trending**

**是什么：**
Google Labs 发布的 design.md 是一套面向 AI Coding Agent 的视觉标识描述格式（Apache 2.0 开源）。它用 YAML 前语（Front Matter）+ Markdown 正文的双层结构，把设计系统的颜色、字体、圆角、间距、组件样式变成 Agent 可解析的机器可读 Token。内置 9 条 Lint 规则（断引用检测、WCAG AA 对比度校验、孤儿 Token 警告等），附带 CLI 工具支持 lint/diff/export 三种操作，可导出为 Tailwind v3/v4 配置和 W3C DTCG 标准 Token。仓库 TypeScript 95.7%，版本 alpha 0.3.0。

**怎么用：**
```bash
npm install "@google/design.md"

# 校验 DESIGN.md 文件
npx @google/design.md lint DESIGN.md

# 导出为 Tailwind CSS 变量（v4）
npx @google/design.md export --format css-tailwind DESIGN.md > theme.css

# 导出为 Tailwind v3 JSON 配置
npx @google/design.md export --format json-tailwind DESIGN.md > tailwind.theme.json

# 对比两个版本检测回归
npx @google/design.md diff DESIGN.md DESIGN-v2.md
```

**为什么重要：**
design.md 解决了 AI 编程中最尴尬的断层：Agent 能写代码，但生成的前端永远对不齐设计稿，因为没有结构化的设计 Token 输入。1,541 stars/day 的爆发说明这是一个普遍痛点。与 W3C Design Token Format 的区别在于：design.md 面向的是 Agent 上下文窗口，强调可读性+机器可解析的平衡。0.3.0 已经支持 CLI lint/diff/export 全套工具链，可直接插入 Claude Code/Codex 的 system prompt 中作为设计约束。

**讨论现场：**
- GitHub 社区大量 PR 围绕新增 Lint 规则展开，"contrast-ratio"规则（校验组件背景/文字颜色对比度 ≥4.5:1）被认为是 Agent 生成 UI 时防止可访问性事故的第一道防线
- TypeScript 社区关注点：`@google/design.md/linter` 可作为库导入，意味着 CI 管道可以自动校验 design.md 变更——设计 Token 第一次有了"单元测试"

**来源：** https://github.com/google-labs-code/design.md

## 2. DSpark / DeepSpec：DeepSeek 联合北大发布推测性解码框架，推理速度提升 60-85%

**评分: 9.0/10 | 可操作性: 高 | 热度: HN 726 points / 299 comments / GitHub 已开源**

**是什么：**
DeepSeek 与北京大学联合发布论文《DSpark: Confidence-Scheduled Speculative Decoding with Semi-Autoregressive Generation》，梁文锋署名。DSpark 框架融合两大创新：(1) 半自回归生成，保留并行草稿模型高吞吐，叠加轻量 Markov Head 建模块内 Token 依赖，解决后续位置接受率衰减；(2) 硬件感知置信度调度，置信度头实时评估每个 Token 的存活概率，根据系统负载动态调整验证长度，算力只分配给预期回报最高的 Token。随论文开源的 DeepSpec 是全栈推测性解码训练代码库（MIT），内置 DSpark、DFlash、Eagle3 三种草稿模型，支持 Qwen3 和 Gemma 系列，覆盖数据准备→训练→评估全流程。

**怎么用：**
```bash
git clone https://github.com/deepseek-ai/DeepSpec
cd DeepSpec
pip install -e .

# 使用 DSpark 草稿模型加速 Qwen3-8B 推理
python -m deepspec.serve \
  --target-model Qwen/Qwen3-8B \
  --draft-model deepseek-ai/dspark-qwen3-8b-draft \
  --scheduling confidence

# 训练自定义草稿模型（需 8 卡环境）
python -m deepspec.train \
  --target-model Qwen/Qwen3-8B \
  --output-dir ./my-draft-model \
  --num-gpus 8
```

**为什么重要：**
在 DeepSeek-V4 线上实测中，DSpark 相比 MTP-1 基线单用户生成速度提升 60-85%（Flash）/ 57-78%（Pro）。在 Qwen3-14B 上对比 Eagle3（自回归草稿），接受长度 +30.0%；对比 DFlash（并行草稿），接受长度 +18.3%。DSpark 把推测性解码从"学术论文"推进到"生产就绪"，38TB 目标缓存面向单节点 8 卡环境，中小团队也可部署。HN 726 points 直接登顶首页，成为当天技术社区讨论度最高的单一技术发布。

**讨论现场：**
- HN 热评（+156）："半自回归生成+置信度调度这个组合很优雅——之前 Eagle/DFlash 两派各自在吞吐和延迟上做权衡，DSpark 证明了不需要二选一"
- HN 另一热评（+98）指出"DeepSeek 这一波开源策略非常聪明：论文+代码+预训练草稿模型一起放出来，OpenAI 和 Anthropic 的推理加速方案至今只发论文不发代码"

**来源：** https://github.com/deepseek-ai/DeepSpec | https://arxiv.org/abs/2606.xxxxx

## 3. cognee：开源 AI Agent 统一记忆平台，单 PostgreSQL 替代图+向量+缓存三件套

**评分: 8.8/10 | 可操作性: 极高 | 热度: GitHub 24K★ | +780 stars/day | #2 GitHub Trending**

**是什么：**
cognee 是一个面向 AI Agent 的开源持久记忆平台（Apache 2.0），核心创新是把传统 Agent 记忆方案需要的三套数据库（图数据库 Neo4j、向量数据库 Qdrant、会话缓存 Redis）压缩到单个 PostgreSQL 实例中。通过自研 Postgres 图后端 + pgvector + SQL 会话缓存，实现 graph/vector/hybrid 三种检索的自适应路由。提供 `remember`/`recall`/`forget`/`improve` 四 API，支持 Python/TypeScript/Rust SDK，有 Claude Code 插件可自动注入会话上下文。BEAM 长上下文基准测试中，100K Token 得分 0.79，10M Token 得分 0.67。

**怎么用：**
```bash
# 安装
uv pip install cognee
# 或使用 PostgreSQL 后端
pip install "cognee[postgres]"

# 启动（含 Web UI + MCP Server）
cognee-cli -ui

# Python API
python -c "
import asyncio, cognee
async def main():
    await cognee.remember('Cognee 把文档变成 AI 记忆。')
    results = await cognee.recall('Cognee 做了什么？')
    print(results)
asyncio.run(main())
"

# Docker 一键部署
docker run --env-file ./.env -p 8000:8000 --rm -it cognee/cognee:main
```

**为什么重要：**
Agent 记忆是 2026 年 AI 工程化的核心瓶颈。每个 Agent 框架都在造自己的记忆方案，cognee 是首个把"图+向量+会话"三层统一到一个 Postgres 的开源方案。CI 基准显示 Postgres 后端比分离式图+向量方案快约 10%。780 stars/day 的爆发说明社区对"简化 Agent 基础设施栈"有强烈需求。Claude Code 插件已上架，覆盖会话开始/提示提交/工具调用/上下文压缩/会话结束全生命周期。

**讨论现场：**
- GitHub Issues 中，有开发者测试了 10M Token 场景后反馈："之前用 Neo4j+Qdrant+Redis 方案在 500K Token 时就开始 OOM，cognee 的 Postgres 统一方案到 10M 还能跑"
- 社区关注点：cognee-rs（Rust SDK）刚发布，Rust Agent 生态正在期待原生记忆层

**来源：** https://github.com/topoteretes/cognee

Princeton 大学 Kaushik Sengupta 教授团队用 AI 攻克了射频集成电路（RFIC）设计——这一长期被视为需要数十年经验的"玄学"领域。方法分两阶段：(1) 强化学习从零搜索最优电路拓扑和器件参数，类似 AlphaGo Zero 自对弈；(2) 逆向设计+CNN 电磁仿真器，把传统数小时的电磁场求解器替换为毫秒级神经网络。生成的电路版图看起来像 QR 码而非传统对称布局——AI 发现了人类工程师不会尝试的结构。论文发表于 ISSCC 2026 和 CICC 2026。

**为什么重要：**
RFIC 设计是半导体行业最后的"手工工艺"堡垒之一——全球能独立设计高性能 RF 芯片的工程师不超过几百人。AI 介入后，设计周期从数周缩短到数分钟，且产出的电路性能（带宽/功率/效率）超过了已知最优硅基 PA 方案。Falcomm 公司已基于此技术推出商用 EDA 工具 GaNdalph.ai（6 月 5 日发布），面向 GlobalFoundries RFGaN1 工艺。HN 191 points 说明硬件+AI 交叉话题在开发者社区有大量关注。
