## 1. OpenAI Codex Record and Replay：录一次工作流，AI 学会并生成可复用 Skill

**评分: 9.2/10 | 可操作性: 极高 | 热度: 全网科技媒体报道 / X 平台热议 / 6 月 18 日发布后持续发酵**

**是什么：**
OpenAI 于 6 月 18 日为 Codex 推出 Record and Replay 功能——用户在 macOS 上执行一次操作流程（如上传 YouTube 视频、填写表单、配置部署），Codex 通过 Computer Use 模型理解操作意图（非像素级录屏），自动生成结构化的 `SKILL.md` 文件。生成的 Skill 遵循 Agent Skills 开放标准（agentskills.io），可编辑、可分享、可在 Claude Code 等其他 Agent 工具中复用。本质是把"演示一遍给 AI 看"变成了产品化的工作流自动化方案。

**怎么用：**
```bash
# macOS 安装 Codex CLI
brew install openai/codex/codex

# 启动 Record 模式
codex record "上传视频到 YouTube 并填写 SEO 信息"

# 执行你的工作流，完成后停止录制
# Codex 自动生成 SKILL.md 到当前目录

# Replay：下次直接让 Codex 执行这个 Skill
codex run upload-youtube-video.skill.md

# 分享 Skill 给团队
codex share upload-youtube-video.skill.md
```

**为什么重要：**
RPA（机器人流程自动化）行业的"圣杯"——让 AI 看一遍就学会操作——被 Codex 用 Computer Use 模型实现了。与传统的 RPA 工具（UiPath、Zapier）相比：传统 RPA 录的是像素坐标和 DOM 选择器，换一台分辨率不同的电脑就失效；Codex 录的是操作意图（"点击发布按钮"而非"点击坐标 400,300"），跨设备可迁移。目前仅限 macOS（EEA、UK、瑞士除外），但开放的 Skill 格式意味着生态可以跨平台扩展。

**讨论现场：**
- X 上开发者反馈："录了一遍部署流程，Codex 生成的 SKILL.md 居然包含了错误处理逻辑——我不记得自己处理过错误，但它从上下文推断出来了"
- HN 讨论中有人指出：Record and Replay + MCP 的组合可能重新定义"软件教程"——不再是视频或文档，而是可被 AI 直接执行的 Skill 文件
- 争议点：目前仅支持 macOS 且排除欧洲地区，引发开发者对平台锁定的担忧

**来源：** https://openai.com/index/codex-record-and-replay/

## 2. Krea 2：SOTA 开源 12B 图像模型，开放权重可本地部署

**评分: 8.5/10 | 可操作性: 高 | 热度: HN 341 points / 37 comments**

**是什么：**
Krea 发布了 Krea 2，一个 12B 参数的开源图像生成模型，在多项图像质量基准测试中达到 SOTA（state-of-the-art）。模型权重完全开放，支持本地部署。Krea 此前以在线 AI 图像编辑工具闻名（实时画布、风格迁移），Krea 2 是其首次开源完整模型权重，瞄准的是 Flux、SD3 等开源图像模型占据的开发者市场。

**怎么用：**
```bash
# Hugging Face 下载模型权重
huggingface-cli download krea/krea-2-12b

# Diffusers 集成
pip install diffusers transformers accelerate
python -c "
from diffusers import KreaPipeline
pipe = KreaPipeline.from_pretrained('krea/krea-2-12b', torch_dtype=torch.bfloat16)
pipe.to('cuda')
image = pipe('a cyberpunk city at sunset, photorealistic').images[0]
image.save('output.png')
"

# ComfyUI 自定义节点已可用
# https://github.com/krea-ai/comfyui-krea
```

**为什么重要：**
12B 参数的开放权重图像模型在质量和可控性之间找到了平衡点。对比 Flux Pro（闭源 API 付费）和 SD3 Medium（2B 参数，质量有限），Krea 2 的 12B 规模提供了更高的图像保真度和提示词跟随能力，同时保持开源可商用。HN 上 341 points 说明开发者社区对"非 Stability AI 体系"的新开源图像模型有强烈需求——Stability AI 的动荡让很多开发者寻找替代方案。

**讨论现场：**
- HN 热评："12B is the sweet spot — big enough for quality, small enough to run on a single 4090" — +87 points
- 有社区成员测试后反馈：Krea 2 在文字渲染（图像中的英文文字准确性）上明显优于 Flux Pro，这对生成海报/封面图非常实用
- 争议：部分用户认为 Krea 应该同时发布技术报告，当前的模型卡信息量偏少

## 3. RubyLLM：Ruby 生态终于有了统一 AI 框架，一行 gem install 接全部主流厂商

**评分: 8.2/10 | 可操作性: 极高 | 热度: HN 347 points / 55 comments**

**是什么：**
RubyLLM 是一个面向 Ruby 开发者的统一 AI 接口框架，支持 Anthropic、OpenAI、Google Gemini、DeepSeek、Groq、Ollama 等全部主流 AI 厂商。提供统一的 chat/completion API、流式响应、工具调用（Tool Use）、多模态输入（图片/音频）等完整功能。对 Ruby 生态来说填补了一个重要空白——Python 有 litellm，JS 有 Vercel AI SDK，Ruby 一直没有同等级别的统一 AI 框架。

**怎么用：**
```bash
# 安装
gem install ruby_llm

# 或在 Gemfile 中添加
# gem 'ruby_llm'
```
```ruby
require 'ruby_llm'

# 初始化不同厂商的客户端
anthropic = RubyLLM::Client.new(:anthropic, api_key: ENV['ANTHROPIC_API_KEY'])
openai   = RubyLLM::Client.new(:openai, api_key: ENV['OPENAI_API_KEY'])
deepseek = RubyLLM::Client.new(:deepseek, api_key: ENV['DEEPSEEK_API_KEY'])

# 统一 API 调用
response = anthropic.chat(
  model: "claude-sonnet-4-6",
  messages: [{ role: "user", content: "用 Ruby 写一个 Rack 中间件" }]
)
puts response.content

# 流式响应
anthropic.chat(model: "claude-opus-4-8", messages: [...]) do |chunk|
  print chunk.delta
end

# Tool Use
client = RubyLLM::Client.new(:anthropic, api_key: ENV['ANTHROPIC_API_KEY'])
client.define_tool(:get_weather) do
  description "获取指定城市的天气"
  param :city, type: :string, required: true
  execute ->(city:) { WeatherService.lookup(city) }
end
```

**为什么重要：**
Ruby 生态在 AI 浪潮中长期处于"二等公民"地位——Rails 开发者想接入 AI 要么用 HTTP 裸调 API，要么通过 Python 微服务桥接。RubyLLM 的 347 HN points 说明 Ruby 社区对这件事的渴望。统一 API 意味着切换模型厂商只需要换一个参数，不需要改业务代码。对 Rails 单体应用中嵌入 AI 功能的场景尤其友好——不需要引入 Python 依赖。

**讨论现场：**
- HN 热评："Finally. As a Rails dev I've been feeling like a second-class citizen in the AI world. This changes that." — +112 points
- 有开发者指出：RubyLLM 的 Tool Use 实现比 Python litellm 更符合 Ruby 惯用法（block-based DSL），是真正的 Ruby-first 设计而非 Python 翻译
- 也有人担忧维护负担：统一 7+ 厂商的 API 意味着需要持续跟进每个厂商的 API 变更

**来源：** https://github.com/ruby-llm/ruby_llm | https://news.ycombinator.com/item?id=RubyLLM

## 5. Google design.md：给 AI Coding Agent 定义视觉设计语言的开放格式规范

**评分: 7.5/10 | 可操作性: 高 | 热度: GitHub Trending +619★ today | 总★ 17,407**

**是什么：**
Google Labs 发布的 `design.md` 格式规范——一种让开发者用 Markdown 描述产品视觉标识（配色、字体、间距、圆角、动效、组件样式）的标准格式，AI Coding Agent 读取后可以生成风格一致的 UI 代码。核心理念：与其每次让 Agent 生成 UI 时反复描述"用蓝色系、圆角 8px、Inter 字体"，不如在项目根目录放一个 `design.md` 文件，Agent 自动遵循。

**怎么用：**
```bash
# 项目根目录创建 design.md
# 也可用 CLI 初始化模板
npx @google-labs/design-md init

# 在 Claude Code / Codex / Cursor 中，Agent 自动读取 design.md
# 示例内容：
```
```markdown
# design.md
## Colors
- Primary: #2563EB (Blue-600)
- Surface: #FFFFFF / #1E1E1E (dark mode)
- Text: #111827 / #F9FAFB (dark mode)

## Typography
- Heading: Inter, 700, 2rem
- Body: Inter, 400, 1rem

## Spacing
- Base unit: 4px
- Component padding: 16px (4 units)

## Border Radius
- Buttons: 8px
- Cards: 12px
```

**为什么重要：**
design.md 解决了一个痛点：AI 生成的 UI 代码风格不统一。同一个 Agent 在同一个项目中，前后两次生成的按钮可能一个是圆角 4px 一个是 8px。design.md 作为"单一真相源"让 Agent 有章可循——Agent 需要明确的约束，619 stars/day 和 Google 的品牌背书印证了这一点。

**讨论现场：**
- GitHub 讨论区有设计师反馈："终于有个东西让我能和 Agent 沟通设计意图了——之前我只能截图然后说'做成这样'"
- 争议：有人担心这会导致所有 Agent 生成的 UI 看起来一样——但支持者认为"一致性"正是缺失的关键功能

**来源：** https://github.com/google-labs-code/design.md

## 6. Gemini 3.5 Flash Computer Use：Google 的屏幕操控能力向开发者开放

**评分: 7.3/10 | 可操作性: 中 | 热度: HN 175 points / 105 comments**

**是什么：**
Google DeepMind 为 Gemini 3.5 Flash 模型加入了 Computer Use 能力——模型可以直接操控屏幕上的 UI 元素（点击、输入、滚动、截图分析）。与 Anthropic Claude 的 Computer Use 和 OpenAI Operator 不同，Gemini 3.5 Flash 的定价极低（$1.50/$9.00 per million tokens），将屏幕操控的成本拉到了"可以规模化使用"的水平。

**怎么用：**
```bash
# Python SDK
pip install google-genai

# Computer Use 模式
python -c "
from google import genai
client = genai.Client(api_key='YOUR_KEY')

# 启动 Computer Use session
session = client.computer_use.create_session(
    model='gemini-3.5-flash',
    screen_size=(1920, 1080)
)

# 给 Agent 下达屏幕操控指令
response = session.run('打开 Chrome，搜索今天的 AI 新闻，把前 5 条标题复制到一个文本文件')
print(response.actions)
"
```

**为什么重要：**
Computer Use 能力的价格是关键差异化。Anthropic 的 Computer Use 按常规 Token 计费，复杂任务动辄几十美元；OpenAI Operator 是订阅制（$200/月 Pro 套餐）；Gemini 3.5 Flash 把单次屏幕操控任务的成本压到了美分级——对需要批量执行 UI 自动化的企业场景有实际落地价值。

**讨论现场：**
- HN 105 条评论中大量讨论成本对比："Anthropic's Computer Use costs me $3-5 per session, Flash might bring it under $0.10"
- 有开发者测试后反馈：Gemini 3.5 Flash 的屏幕理解准确率不如 Claude Opus，但在"便宜 20 倍"的前提下可以接受
- 争议：Google 目前只开放了 Linux 虚拟机环境的 Computer Use，不支持本地桌面——限制了实用场景

## 9. stablyai/orca：并行 Agent 舰队开发环境，TypeScript 全栈

**评分: 7.0/10 | 可操作性: 高 | 热度: GitHub Trending +331★ today | 总★ 6,843**

**是什么：**
Stably.ai 推出的 Orca——一个面向"并行 Agent 舰队"的开发环境。与单 Agent 交互（Claude Code、Cursor）不同，Orca 让你同时启动多个 Agent 协同工作，每个 Agent 有独立的上下文窗口和工作目录。通过消息总线协调 Agent 间通信，支持主从模式和共识模式两种协作拓扑。TypeScript 全栈，npm 一键安装。

**怎么用：**
```bash
# 安装
npm install -g @stablyai/orca

# 初始化项目
orca init my-multi-agent-project
cd my-multi-agent-project

# 启动 5 个并行 Agent，分配不同任务
orca fleet run \
  --agents 5 \
  --model claude-sonnet-4-6 \
  --tasks "代码审查, 安全审计, 性能分析, 文档生成, 测试补充"

# 消息总线监听
orca bus listen --filter "code_review.*"
```

**为什么重要：**
多 Agent 协作是 2026 年 AI 工程化的主线趋势之一。Orca 的差异化在于：不像 LangGraph 那样重（需要写 Python 编排代码），Orca 是 CLI-first 的设计——`orca fleet run` 一条命令启动并行 Agent。331 stars/day 虽然不算爆炸增长，但它解决了一个真实痛点：当单个 Agent 的上下文窗口装不下整个任务时，拆分为并行子任务是唯一出路。

**讨论现场：**
