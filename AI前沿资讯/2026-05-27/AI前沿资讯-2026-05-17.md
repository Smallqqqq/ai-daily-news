## 1. codegraph — 预索引代码知识图谱

**评分: 8.9/10 | 可操作性: 高 | 热度: +857 stars/day · Trendshift 本周 #2**

**是什么：** 一个预索引的代码知识图谱工具，为 Claude Code、Codex、Cursor 和 OpenCode 提供极低 token 消耗的代码上下文查询。100% 本地运行，不依赖云端 API。

**怎么用：**
```bash
# 安装后直接对接 Claude Code 等 AI 编程助手
# 源码位于 GitHub，TypeScript 编写
git clone https://github.com/colbymchenry/codegraph
```

**为什么重要：** 代码知识图谱将代码库结构预计算为图结构，相比传统的 grep/全文搜索，token 消耗大幅降低（与 Semble 的策略互补），直接减少编码 Agent 的上下文窗口压力。+857 stars/day 说明社区对上下文效率优化的需求极其迫切。

**来源：** https://github.com/colbymchenry/codegraph

## 3. Semble — 面向 AI Agent 的代码搜索（98% token 节省）

**评分: 8.3/10 | 可操作性: 高 | 热度: HN 首页 247 pts · 78 comments**

**是什么：** 一个专为 AI 编程 Agent 设计的代码搜索工具，声称比 grep 节省 98% token。Show HN 发布当日即登上 HN 首页，获得 247 分。

**怎么用：**
```bash
# 直接集成到 Claude Code（MCP 方式）
claude mcp add semble -s user -- uvx --from "semble[mcp]" semble
```

**为什么重要：** 代码搜索是 AI 编码 Agent 最频繁的操作之一。将搜索 token 消耗降低 98% 直接转化为更低的 API 成本、更快的响应速度和更大的有效上下文窗口。这是解决 Agent 上下文效率问题的一种实用工程方案。

**来源：** https://github.com/minishlab/semble (HN: https://news.ycombinator.com/item?id=48169874)

## 7. Meta AI SP-KV — KV 缓存内存优化突破

**评分: 7.1/10 | 可操作性: 低 | 热度: Reddit r/LocalLLaMA 热议 · LLM Daily 头条报道**

**是什么：** Meta AI 提出的一种训练时注意力机制，让 LLM 学会预测未来需要哪些 KV 键值对，从而大幅削减 KV 缓存的内存开销。

**怎么用：**
```bash
# 当前为研究阶段，尚未有可直接使用的开源实现
# 论文核心思路：在训练时注入 "未来 KV 需求预测" 信号
```

**为什么重要：** KV 缓存是长上下文推理的内存瓶颈——上下文越长，缓存越大。SP-KV 从训练阶段解决这一问题，对 Agent 长时间运行、超长文档处理、多轮对话等场景的部署成本有根本性影响。

**来源：** Meta AI 研究论文（arXiv 待确认）
