## 1. CodeGraph — Pre-indexed Code Knowledge Graph

**评分: 9.0/10 | 可操作性: 高 | stars: 3,508 (+857/天)**

**是什么：** 为 Claude Code、Codex、Cursor、OpenCode 预构建的代码知识图谱，将代码库索引为语义图谱，减少 AI 编码时的 token 消耗和 tool call 次数。

**怎么用：**
```bash
git clone https://github.com/colbymchenry/codegraph
cd codegraph && npm install
codegraph index /path/to/your/repo
```

**为什么重要：** 传统 AI 编码工具每次对话都要重新读取代码文件，消耗大量 token。CodeGraph 提前索引生成知识图谱，**100% 本地运行**，代码不出本机。大代码库场景下效果显著。

**来源：** https://github.com/colbymchenry/codegraph

## 2. Claude Code 五月更新 — Fast Mode Opus 4.7 + Hook 增强 + MCP 热重载

**评分: 8.8/10 | 可操作性: 高 | 直接影响所有 Claude Code 用户**

**是什么：** Claude Code 五月累积更新，涵盖模型升级、Hook 系统增强、MCP 体验优化。

**怎么用：**
```bash
# Fast Mode 已默认使用 Opus 4.7（无需操作）
# 如需固定 Opus 4.6，设置环境变量：
# CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1

# MCP 配置热重载（无需重启 Claude Code）
claude mcp reconnect

# 清理孤儿插件依赖
claude plugin prune
```

**为什么重要：**
- **Fast Mode 升级 Opus 4.7**：默认更快更强的推理，无需额外付费
- **MCP 热重载**：改 `.mcp.json` 后 `/mcp Reconnect` 即可生效，无需重启
- **Hook `continueOnBlock`**：PostToolUse Hook 拒绝原因可反馈给 Claude 继续对话
- **Compaction 保留敏感指令**：上下文压缩时不再丢失用户设定的关键规则
- **MCP 服务器新增 `alwaysLoad` 选项**：关键工具跳过延迟加载，始终可用

**来源：** https://code.claude.com/docs/en/changelog

**评分: 8.0/10 | 可操作性: 中 | 影响所有 LangChain 生态用户**

**怎么用：**
```bash

```

**为什么重要：** 三项关键更新使 LangGraph 从"能做 demo"升级到"能跑生产"：

- **Content-block-centric streaming API v3**：流式输出更细粒度控制
- 预置 **Supervisor、Swarm、LangMem、Trustcall** 四种 Agent 架构开箱即用

**来源：** https://github.com/langchain-ai/langgraph/releases
