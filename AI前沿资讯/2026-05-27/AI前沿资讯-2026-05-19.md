## 3. Forge — 护栏让 8B 模型 Agent 成功率从 53% 飙到 99%

**评分: 9.0/10 | 可操作性: 高 | 热度: HN 285 pts · 105 comments**

**是什么：**
Forge 是一个轻量级 Agent 护栏框架，在 LLM 输出层叠加结构化约束验证和自动纠错，将 8B 小模型的 Agent 任务成功率从 53% 提升到 99%。核心思路不是训练更强的模型，而是在推理输出层做逻辑验证。

**怎么用：**
```bash
pip install forge-guardrails
# 在 Agent 代码中包裹 LLM 调用
from forge import Guardrail
guard = Guardrail(model="agent-rules.json")
result = guard.run(llm_output, context)
```

**为什么重要：**
53% → 99% 意味着小模型做 Agent 任务成为可行的生产方案，不再必须依赖 Opus/GPT-5.5 级别的大模型。8B 模型推理成本远低于 200B+ 模型，这对 Agent 部署的经济可行性是根本性改变。同一天 HN 同时热议"Guardrails"和"Agent 生产化"，Forge 恰好踩中最大痛点。

**讨论现场：**
- HN 热评："This is the most important paper of the day. 53%→99% on agentic tasks with an 8B model means you don't need GPT-5.5 to ship production agents." — +95 points
- 讨论焦点：护栏规则编写成本 vs 模型成本节省的权衡、对开放式任务的泛化能力

**来源：** HN: Forge 官方仓库

## 4. Google I/O: Gemini Omni — 原生 Omni-Modal 模型

**评分: 8.8/10 | 可操作性: 中 | 热度: HN 276 pts · 112 comments**

**是什么：**
Gemini Omni 是 Google 首个从预训练阶段就将文本、图像、音频、视频作为统一 token 空间处理的原生 omni-modal 模型。不同于传统"文本为主 + 视觉附加"架构，Omni 在所有模态上的表现更均衡。

**怎么用：**
通过 Google AI Studio 或 Gemini API 调用，详见 https://ai.google.dev

**为什么重要：**
Omni-modal 是下一代模型架构的共识方向。GPT-5.5、Claude Opus 4.6 都支持多模态但本质仍是"文本为主+视觉附加"。Gemini Omni 声称从预训练层就是真正的统一模态——如果 benchmark 兑现，这将是架构层面的突破。Google I/O 直播峰值 8.5M 同时在线，Omni 是讨论度最高的发布之一。

**讨论现场：**
- HN 讨论焦点：Omni 是否真的"原生" vs 营销包装；与 GPT-5.5 多模态能力的直接对比
- Google I/O 直播中 Omni demo 环节弹幕反应最热烈

**来源：** https://deepmind.google
