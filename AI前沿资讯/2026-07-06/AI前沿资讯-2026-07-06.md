## 1. planning-with-files（给 AI Agent 加持久化文件级工作记忆）

**评分: 8.8/10 | 可操作性: 高 | 热度: GitHub Trending 今日 + 兼容 60+ Agent 工具 + SKILL.md 标准**

**是什么：**
OthmanAdi 发布的 planning-with-files 是一套基于 Markdown 文件的持久化任务规划方案，专治 AI Agent 在长任务中"忘记自己在干什么"的顽疾。核心思路：用 `PLANNING.md`、`TASK_LOG.md`、`COMPLETION_GATE.md` 三个纯文本文件作为 Agent 的共享状态层，崩溃恢复或 `/clear` 后 Agent 重新读取文件即可无缝接续。支持 Claude Code、Codex CLI、Cursor、Kiro、OpenCode 等 60+ 种 Agent 工具。灵感来自 Manus 的多 Agent 协作文件系统，但以 SKILL.md 标准打包，零依赖安装。

**怎么用：**
```bash
# 安装为 Claude Code 技能
cd ~/.claude/skills
git clone https://github.com/OthmanAdi/planning-with-files.git

# 或手动创建三个核心文件
touch PLANNING.md TASK_LOG.md COMPLETION_GATE.md

# Claude Code 中引用 SKILL.md 即可自动激活
# AI Agent 会自动将任务拆解写入 PLANNING.md，完成项标记在 TASK_LOG.md
```

**为什么重要：**
AI Agent 最大的可靠性问题不是"不够聪明"，而是"记忆断片"。上下文窗口满了、用户清了、进程崩了，之前做到哪一步全忘了。planning-with-files 把 Agent 的 working memory 外化到文件系统，相当于给了 Agent 一个外置硬盘。对需要 >100 步的复杂编码任务、多 Agent 协作、跨天执行的长任务，这是基础设施级的补丁。Manus 风格的文件化规划正在成为 Agent 工程的标准模式。

**讨论现场：**
- GitHub Trending 页面上榜不到 24 小时已获大量关注，标签页显示 `@claude`、`@kaichen`、`@ebrevdo` 等 Claude Code 团队成员已关注
- X 上多位 Agent 开发者："终于有人把 Manus 的工作记忆模式标准化了。这是在填补 AI Agent 工程最大的空白——持久化状态管理"
- 社区讨论聚焦：这种文件化方案 vs 数据库方案（如 mem0）的取舍——文件方案零依赖、人类可读、Git 友好，数据库方案更适合高频读写

**来源：** https://github.com/OthmanAdi/planning-with-files

## 2. Nano Banana 2 Lite（Google 最快最便宜的图像生成 API）

**评分: 8.5/10 | 可操作性: 高 | 热度: Google 官方博客 + 多篇技术评测 + X 广泛讨论**

**是什么：**
Google DeepMind 于 7 月 2 日发布 Nano Banana 2 Lite（Gemini 3.1 Flash Image），是迄今最快的 Gemini 图像模型——单张图片生成时间不到 4 秒，千张成本 $0.034（约人民币 2.4 分钱）。图像质量高于第一代 Nano Banana，支持文本渲染、多轮迭代编辑、风格保持一致。Gemini 图像交互式 API 同步正式 GA，开发者通过自然语言对话即可持续修改图像。

**怎么用：**
```bash
# 通过 Gemini API 调用（Python SDK）
pip install google-genai
```
```python
from google import genai

client = genai.Client()
response = client.models.generate_content(
    model="gemini-3.1-flash-image",
    contents="A futuristic city skyline at sunset, cyberpunk style, 16:9 ratio"
)
# 保存图片
with open("city.png", "wb") as f:
    f.write(response.generated_images[0].image.image_bytes)
```

**为什么重要：**
$0.034/千张意味着生成一张图的成本几乎为零。批量生成 1000 张图只要 3.4 美分，而 Midjourney 按量付费最低也要 $10/200 张（5 美分/张）。速度上 4 秒出图碾压多数竞品（Midjourney ~30s, Imagen 3 ~8s）。对需要批量生成缩略图、商品图、教育插图的场景，这是成本断崖式下降。Google 在图像生成赛道从追赶者变成了价格屠夫。

**讨论现场：**

## 3. TryCase（AI 编码 Agent 的 disposable 测试沙箱）

**评分: 8.3/10 | 可操作性: 高 | 热度: Product Hunt #4 / 143 upvotes（7 月 5 日）**

**是什么：**
TryCase 为 AI 编码 Agent 提供 disposable（一次性）测试环境。AI Agent 写完代码后，在一个全新隔离的云环境中跑测试、验证功能、截图对比，然后自动销毁环境。解决了"Agent 写的代码在我机器上能跑，但不知道在干净环境行不行"的问题。支持一键为 Agent 创建沙箱、设定超时自动销毁、保存测试结果截图。

**怎么用：**
```bash
# 安装 CLI
npm install -g trycase

# 为当前项目创建测试沙箱
trycase create --project . --timeout 30m

# AI Agent 可以在沙箱内自由运行命令
trycase exec "npm install && npm test"

# 用完后销毁
trycase destroy
```

**为什么重要：**
AI Agent 写代码最大的风险不是"写不对"，而是"破坏本地环境"——装错了依赖、改了系统配置、跑了恶意代码。TryCase 等于给每个 Agent 任务配了一个 condom。对比同类方案 Daytona（更偏向持久化开发环境）、E2B（更重型的沙箱），TryCase 的 disposable 理念更契合 AI Agent 的一次性任务模式——用完即弃，不计成本。这可能是 Agent 安全性的一个关键补丁。

**讨论现场：**
- Product Hunt 评论区："这正是我需要的——让 Claude Code 放胆去跑测试，不用担心它把我的 npm 搞坏"
- 创始人回应：已支持 GitHub Actions CI 集成，下一个版本支持 Docker Compose 多容器场景
- X 上 Agent 开发者："TryCase + planning-with-files = Agent 终于有了完整的'写→测→提交'闭环"

**来源：** https://www.producthunt.com/products/trycase | https://trycase.dev

    "unity-mcp": {
      "command": "node",
      "args": ["path/to/unity-mcp/index.js"]

  }

# 然后在 Claude Code 中直接对话操作 Unity

**怎么用：**
```bash
# 安装 SDK
npm install @context-dev/sdk
```
```javascript
import { Context } from '@context-dev/sdk';

const ctx = new Context({ apiKey: 'YOUR_KEY' });

// 搜索 + 抓取 + 提取，一步完成
const result = await ctx.search('AI agent frameworks 2026', {
  scrapePageContent: true,
  maxResults: 10
});
console.log(result.items.map(i => ({ title: i.title, text: i.content }));
```

**为什么重要：**
Firecrawl（130K+ stars）已经证明"AI Agent 需要专门的网络数据基础设施"是一个真实且巨大的市场。Context.dev 在 Firecrawl 的基础上进一步抽象和简化，把 Search + Scrape + Parse + Crawl + Map + Interact 打包在一个更轻量的 API 里。893 个 Product Hunt upvotes 说明开发者对这个方向的强烈需求。

## 9. Gemini Omni Flash（原生多模态 API 公测）

**是什么：**
Google DeepMind 将 Gemini Omni Flash 推向 API 公测，这是首个面向开发者的原生多模态模型 API——同时处理文本、图像、音频和视频输入，支持视频理解、视频描述生成、多模态对话等。不同于此前需要分别调用不同模型的方案，Omni Flash 在一个模型内原生融合所有模态。

**怎么用：**
```bash
# Python SDK
pip install google-genai
```
```python
from google import genai

client = genai.Client()

# 上传视频并分析
video = client.files.upload(file="meeting.mp4")
response = client.models.generate_content(
    model="gemini-omni-flash",

```
