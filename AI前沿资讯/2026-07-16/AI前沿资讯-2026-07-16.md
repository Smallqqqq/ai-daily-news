## 1. Inkling（Thinking Machines Lab）——Mira Murati 交出的 975B 开源权重旗舰模型

![[inkling-bench.png]]

**评分: 9.2/10 | 可操作性: 高 | 热度: HN 710 points / 183 评论，Reuters 头版报道**

**是什么：**
Mira Murati（OpenAI 前 CTO）创立的 Thinking Machines Lab 于 7 月 15 日发布 Inkling，一个 975B 参数的 MoE 开源权重模型。架构采用 256 个路由专家 + 2 个共享专家，每次前向传播激活 41B 参数。支持文本、图像、音频多模态输入，100 万 token 上下文窗口。滑窗注意力和全局注意力以 5:1 比例交错。训练采用 Muon+Adam 混合优化器，RL 阶段用了 3000 万+ rollout。还包含一个较小的 Inkling-Small 变体。

**怎么用：**
```bash
# Hugging Face 下载模型
huggingface-cli download thinkingmachines/inkling --local-dir ./inkling

# Tinker Playground 在线体验（免安装）
# 浏览器打开：https://tinker.thinkingmachines.ai/playground

# OpenAI 兼容 API（通过 vLLM 部署）
pip install vllm
vllm serve thinkingmachines/inkling --tensor-parallel-size 8 --max-model-len 1000000

# API 调用示例
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"inkling","messages":[{"role":"user","content":"解释 MoE 架构的优势"}]}'
```

**为什么重要：**
Meta 放弃 Llama 4 开源路线后，西方开源权重生态出现真空。Inkling 是首个填补这个空白的美国旗舰模型。Benchmark 对比：HLE text-only 30.0%（vs DeepSeek V4 Pro 35.9%、GPT 5.6 Sol 47.2%），HLE with tools 46.0%，AIME 2026 达 97.1%，SWE-Bench Verified 77.6%。音频能力在所有开源权重模型中排名第一（Audio MC 56.6% vs Qwen3-Omni 24.3%）。训练仅用竞品约 1/3 的 token 量，效率极高。

**讨论现场：**
- HN 热评："This is the first serious US answer to the Chinese open-weights dominance we've seen over the past year. Mira Murati's team delivered." — 置顶评论
- Reuters 报道指出：该模型填补了西方开源生态 "Meta 留下的真空"，是继 Meta 转为闭源策略后最重要的美国开源发布
- 社区关注点：RL 阶段 3000 万 rollout 规模空前，显示了 Thinking Machines 在后训练阶段的投入远超一般开源项目

**来源：** https://thinkingmachines.ai/news/introducing-inkling | https://huggingface.co/thinkingmachines/inkling

## 2. Grok Build 全面开源——xAI 编码 Agent 源码公开，可本地运行

![[grokbuild-oss.png]]

**评分: 8.8/10 | 可操作性: 极高 | 热度: HN 276 points / 322 评论，X 全平台刷屏**

**是什么：**
xAI 于 7 月 15 日将 Grok Build（原为闭源的编码 Agent CLI/TUI）完整开源。开源范围包括从上下文组装到工具调用分发的全部源码。关键变化：现在可以本地编译、对接自有推理服务、完全离线运行。支持 skills、plugins、hooks、MCP servers、subagents 的完整扩展体系。此前 Grok Build 在 7 月 12 日爆出 "上传完整仓库到服务器" 的安全丑闻（0.2.93 版本），开源是对信任危机的直接回应。xAI 同时清除了此前收集的用户数据。

**怎么用：**
```bash
# 克隆源码
git clone https://github.com/xai-org/grok-build.git
cd grok-build

# 编译（需要 Rust 工具链）
cargo build --release

# 本地运行（对接自有推理）
./target/release/grok-build --model local --api-base http://localhost:11434/v1

# 通过 config.toml 配置本地模型
cat > config.toml << 'EOF'
[model]
provider = "local"
api_base = "http://localhost:11434/v1"
model_name = "llama3.3"
EOF

# Headless 模式（CI/CD）
grok-build -p "重构 src/auth 模块，拆分出 token 验证逻辑"
```

**为什么重要：**
Grok Build 的 plan→review→approve 工作流 + 8 个并行子 Agent（独立 Git worktree 隔离）是目前编码 Agent 中最成熟的并行架构之一。开源后开发者可以：1）完全离线使用，消除数据隐私顾虑；2）接入任意本地模型降低成本；3）自定义扩展体系。这是继 Claude Code 和 OpenClaw 之后第三个开源的头部编码 Agent。

**讨论现场：**
- HN 高赞评论："Open-sourcing isn't just about trust repair — it's about making the harness the platform. This is the right long-term move." 276 points
- X 上 @grok："Grok Build is now fully open source. Build it yourself, point it at local inference, own your toolchain." 获 5,800+ 转发
- 争议：部分开发者认为开源无法完全消除 7 月 12 日隐私事件的影响，"信任需要时间重建"

**来源：** https://x.ai/news/grok-build-open-source | https://github.com/xai-org/grok-build

## 4. iOfficeAI/OfficeCLI——AI Agent 原生 Office 套件，1,224 stars/day

![[officecli-demo.png]]

**评分: 8.4/10 | 可操作性: 极高 | 热度: GitHub 1,224 stars/day，累计 17.8K stars**

**是什么：**
一个专为 AI Agent 设计的 Office 命令行工具。单个二进制文件即可读写和创建 Word (.docx)、Excel (.xlsx)、PowerPoint (.pptx) 文件，无需安装 Microsoft Office。支持跨平台（Windows/Mac/Linux）。内置渲染引擎，Agent 操作完文档后可以直接 "看到" 输出效果，而非猜测原始 XML。最新版 v1.0.136（7 月 14 日发布）。

**怎么用：**
```bash
# 下载预编译二进制（Windows/Mac/Linux 均可用）
# Windows:
curl -L -o officecli.exe https://github.com/iOfficeAI/OfficeCLI/releases/latest/download/OfficeCLI-win-x64.exe

# macOS:
curl -L -o officecli https://github.com/iOfficeAI/OfficeCLI/releases/latest/download/OfficeCLI-osx-x64
chmod +x officecli

# AI Agent 使用示例：读取 Excel 并分析数据
officecli read --format json report.xlsx | jq '.sheets[0].data'

# 创建 Word 文档
officecli create --type word --output proposal.docx --content "项目提案内容"

# 修改 PowerPoint
officecli modify --input deck.pptx --slide 3 --replace-title "新标题"
```

**为什么重要：**
AI 编码 Agent 在 2026 年的瓶颈已从 "写代码" 扩展到 "产出完整工作交付物"——实现计划、预算表格、客户提案、管理汇报 PPT。OfficeCLI 填补了这个关键空白，让 Agent 能将工作成果输出为人类可读的专业文档格式。与同类方案（python-docx/openpyxl/python-pptx）相比：1）单二进制无依赖；2）内置渲染可预览；3）支持三种格式统一操作。HN 社区评价："这才是 Agent 真正需要的最后一公里基础设施。"

**讨论现场：**
- HN 讨论："The most useful AI agents in 2026 won't stop at code. They'll create the surrounding artifacts that make work usable: proposals, spreadsheets, decks."
- 开发者实测：Agent 可全自动生成包含图表的季报 PPT，"以前需要 2 小时的手动排版现在 30 秒完成"
- 社区关注点：v1.0.136 已支持公式计算和图表嵌入，图表渲染质量持续改进中

**来源：** https://github.com/iOfficeAI/OfficeCLI | https://officecli.ai

**是什么：**
一套基于 shadcn/ui 风格的 React 组件库，专门模拟主流 AI 编码工具的终端 UI 风格——Claude Code 的暗色极简风、Grok Build 的蓝紫渐变风、Codex CLI 的亮色终端风。AI Agent 生成前端界面时可直接引用这些组件，输出 "看起来像专业开发者工具" 的 UI。支持主题切换和自定义配色。

**怎么用：**
```bash
# 安装组件库
npx shadcn@latest add brainless

# 在 React 项目中使用
# 导入 Claude Code 风格终端组件
import { CCTerminal, GrokTerminal, CodexTerminal } from 'brainless'

# 使用示例
<CCTerminal>
  <CCTerminal.Line type="command">$ claude "fix the auth bug"</CCTerminal.Line>
  <CCTerminal.Line type="output">Analyzing codebase... Found issue in auth.ts:42</CCTerminal.Line>
</CCTerminal>
```

**为什么重要：**
"Agent 生成的 UI 没有风格" 是 2026 年前端开发者的普遍抱怨。Brainless 解决了一个具体但高频的痛点：让 Agent 生成的开发者工具界面 "看起来对"。95 个 HN points 虽然不算爆炸，但评论区高度集中——几乎都是实际使用过的正面反馈。

**讨论现场：**
- HN 评论："Finally someone thought of this. Every AI-generated dashboard looks like Bootstrap 5 default theme. These components make agent output feel intentional." — 95 points
- 开发者反馈：结合 hallmark（#5 条目）使用效果更佳——先 hallmark 审查设计质量，再 brainless 统一视觉风格
