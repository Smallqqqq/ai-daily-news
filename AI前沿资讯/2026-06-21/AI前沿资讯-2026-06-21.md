## 1. SubQ：全球首个亚二次方稀疏注意力（SSA）架构模型，1200 万上下文成本仅 Claude Opus 的 5%（Subquadratic）

**评分: 9.3/10 | 可操作性: 高 | 热度: 36Kr / AINews / HN 热议 + 全渠道刷屏**

**是什么：**
SubQ 是迈阿密 13 人初创公司 Subquadratic 推出的全球首款基于完全亚二次方稀疏注意力（SSA）架构的 AI 模型，上下文窗口高达 1200 万 Token。核心原理是：对每个 query，基于内容动态选择真正值得关注的 Token 位置，只在这些位置精确计算注意力，而非盲目计算所有 Token 对之间的关联。相比 Transformer 的密集注意力，计算量暴减 1000 倍（1200 万 Token 场景）。公司 2024 年成立，拿下 2900 万美元种子轮，估值 5 亿美元。

**怎么用：**
```bash
# SubQ API 调用（12M token 全量上下文）
curl https://api.subq.ai/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $SUBQ_API_KEY" \
  -d '{"model": "subq-1.0", "messages": [{"role": "user", "content": "分析整个代码库的架构"}]}'

# SubQ Code CLI（整个代码库一次性塞入）
npm install -g @subq/code
subq code --repo /path/to/your/project "重构所有 API 路由"
```

**为什么重要：**
成本差距是结构性的——RULER 128K 基准测试：SubQ 花费 $8，Claude Opus 花费 $2600，300 倍成本差距。100 万 Token 推理场景，SSA 比 FlashAttention-2 快 52.5 倍。SWE-bench 得分 81%，在长上下文检索 MRCR v2 上得分 65.9%，虽低于 Opus 4.6（78%）但远超 GPT 5.4（39%）和 Gemini 3.1 Pro（23%）。13 人种子轮公司的产品在多项核心基准上打平甚至超过 Anthropic 和 OpenAI 的旗舰模型——这不是渐进式改进，而是架构层面的一次范式转移。

- OpenAI 前研究员 Will Depue 分析："SubQ 几乎可以确定是基于 Kimi 或 DeepSeek 的稀疏注意力微调"——暗示现有开源模型 + 稀疏注意力可能是一条被低估的工程路线
- HN 社区反应："这是 2026 年最疯狂的 AI 发布之一。如果 SSA 能保持这个效率优势并持续提升能力，Transformer 的统治地位确实悬了"
- 36Kr 热评："13 人干翻 Transformer，算力暴减千倍"——中国科技圈对架构创新的关注度极高

## 2. Claude Code Artifacts：Claude Code 工作成果一键转化为交互式网页，6 月 18 日上线（Anthropic）

**评分: 8.8/10 | 可操作性: 极高 | 热度: YouTube 多频道视频覆盖 + Anthropic 官方发布**

**是什么：**
Anthropic 于 6 月 18 日推出 Claude Code Artifacts 功能——将 Claude Code 会话中的工作成果转化为实时交互式网页。页面基于对话全文上下文、代码库、关联工具自动构建，不是截图或静态报告。支持的输出类型包括：PR 走查（代码变更 + 行级注释）、可交互的数据看板（筛选/排序）、系统架构图、自动填充的检查清单。页面是实时的——Claude 更新 artifact 后，所有打开该页面的人立即看到新版本。支持控件（滑块、开关）、一键导出选择结果回对话继续迭代。

**怎么用：**

## 3. aitm 1.0：AI 是参与者的终端，而不是驾驶员——Tauri + Rust 构建的安全优先终端（kanfu-panda）

**评分: 8.5/10 | 可操作性: 极高 | 热度: Dev.to 首页 6 月 20 日**

**是什么：**
aitm 是一个全新设计理念的 AI 终端——AI 在里面是"参与者"而非"驾驶员"。与传统 AI 终端（AI 自动执行命令）不同，aitm 的核心设计是：终端操作完全由人掌控，AI 只能读取终端输出并提供建议，不能主动执行任何命令。技术栈：Tauri 2 + Rust 后端（处理 PTY、IPC、安全门控）+ React 19 前端 + CodeMirror 编辑器。安全门控在 Rust 层实现，JavaScript 层无法绕过。支持 6 个 LLM 提供商（OpenAI、Anthropic、DeepSeek、Qwen、Zhipu、Moonshot），所有数据存本地 SQLite，无需账号。

**怎么用：**
```bash
# macOS（已签名的 DMG）
# 从 GitHub Release 下载：https://github.com/kanfu-panda/aitm/releases

# Windows x86_64 / ARM64
# 同样从 Release 页面下载对应安装包

# 源码构建
git clone https://github.com/kanfu-panda/aitm
cd aitm
npm install && npm run tauri build
```

**为什么重要：**
aitm 的安全理念在当前的 AI 终端赛道中是反主流的。大多数工具（Claude Code Terminal 模式、Warp、Cursor Terminal）在追求 AI 自主执行命令，而 aitm 则强调"AI 可以看、可以建议、不能碰"——这在处理生产环境或敏感系统时是巨大的差异化优势。Tauri 2 选型也很有意思：对比 Electron 的 ~150MB 空闲内存，Tauri 可以控制在几十 MB。Apache 2.0 协议，6 个 Provider 覆盖中西主流 API，中英文双语 UI。

**讨论现场：**
- Dev.to 热评："我喜欢这种安全模型。在当前所有 AI 终端都在追求自主性的趋势下，有人认真思考安全边界是难得的"
- 社区反馈："Rust 层的安全门控是正确做法——理论上 JS 层即使被攻击也不能绕过"

**来源：** https://github.com/kanfu-panda/aitm | https://dev.to/kanfu-panda/aitm-10-a-terminal-where-the-ai-is-a-participant-not-the-driver-2nb

## 4. SLIME：Zhipu 开源 RL 后训练框架，GLM-5.2 仅用 ~2 天完成完整 OPD 训练（THUDM）

**评分: 8.5/10 | 可操作性: 高 | 热度: 知乎热榜 + 36Kr + 鉅亨號 + X 平台热议**

**是什么：**
SLIME 是 Zhipu AI（智谱/清华）开源的内部强化学习后训练框架。最震撼的数据：刚刚发布的 GLM-5.2，整个 OPD（Off-Policy Distillation）后训练流程在 SLIME 上仅用了大约 2 天完成。SLIME 已在 Qwen 系列（3.6/3.5/3Next/3MoE/3/2.5）、Miles（面向企业的 RL 框架）、P1（物理推理模型）、TritonForge（GPU Kernel 自动生成训练）、ART（AgentCore RL Toolkit）等多个项目中被实际验证。GLM-5.2 在 CodeArena 前端开发盲测中取得全球可用模型第一。

**怎么用：**
```bash
# 克隆仓库
git clone https://github.com/THUDM/slime
cd slime

# 安装依赖
pip install -e .

# 使用 SLIME 进行 GRPO 训练（示例配置）
python -m slime.train \
  --model_path /path/to/base_model \
  --dataset_path /path/to/rl_dataset \
  --algorithm grpo \
  --output_dir ./output
```

**为什么重要：**
RL 后训练是当前大模型能力提升的最关键环节，但此前开源社区缺乏经过大规模实战验证的 RL 框架。SLIME 的价值在于：A) 它不是在实验室玩具上测试的，而是在 GLM-5.2、Qwen 系列等顶级生产模型上实际跑通的；B) 仅用 ~2 天完成 GLM-5.2 的 OPD 训练，说明效率极高；C) 与 Miles、ART、P1 等项目的上下游联动形成了一个完整生态。开源意味着任何团队都可以复现 GLM-5.2 级别的后训练流程。

## 6. NVIDIA SpatialClaw：免训练空间推理框架，将代码作为 VLM 的动作接口（NVIDIA Research）

**评分: 8.0/10 | 可操作性: 高 | 热度: MarkTechPost + AI-trends.today + 知乎 + GitHub Trending（6 月 20 日发布）**

**是什么：**
SpatialClaw 是 NVIDIA Research 发布的一个免训练（training-free）空间推理 Agent 框架。核心创新是将代码作为动作接口——让 VLM 调用感知工具（Depth Anything 3 生成深度图、SAM 3 生成语义掩码），然后把感知结果组合输出为可执行的 Python 代码，最后让 Agent 在代码层面做空间推理。全程不需要重新训练 VLM。与 SpaceTools（此前最佳空间 Agent）相比提升了 11.2 个百分点。

**怎么用：**
```bash
git clone --recursive https://github.com/NVlabs/SpatialClaw.git
cd SpatialClaw
bash spatial_agent/scripts/setup.sh
# 编辑 .env 填入 API Key（支持 OpenAI / Anthropic / Gemini / vLLM）
python -m spatial_agent.entrypoints.run \
  --dataset spatial_agent/config/dataset/erqa.json \
  --model spatial_agent/config/model/gemini-3-pro.json \
  --concurrency 4
```

**为什么重要：**
VLM 的 3D 空间判断一直是公认的短板——它们能识别物体但说不清"左边还是右边"、"距离多远"、"会不会撞到"。SpatialClaw 的方案很优雅：不给 VLM 加训练负担，而是给它一套感知 API 让它自己写代码分析。这种"code as action"范式正在成为 AI Agent 领域的新趋势（Webwright 也是类似的思路）。20 个基准平均 59.9% 的准确率看起来不高，但考虑到是 zero-shot + training-free，反映了当前 VLM 空间推理的真实能力天花板。

**讨论现场：**
- MarkTechPost："免训练意味着任何 VLM 都可以接入——不用收集空间标注数据、不用微调。这极大降低了空间 AI 的落地门槛"
- 知乎分析："Code as action 是 2026 年 Agent 架构的最大趋势——Webwright（浏览器操作）、SpatialClaw（空间推理）都在走这条路"

**来源：** https://github.com/NVlabs/SpatialClaw | https://ai-trends.today/nvidia-ai-introduce-spatialclaw

## 10. VibeThinker-3B：3B 参数在 AIME26 数学测试追平 671B 模型，证明"可验证推理"可以被压缩（Spectrum-to-Signal Pipeline）

**评分: 7.3/10 | 可操作性: 中 | 热度: 知乎 + 36Kr + arXiv 论文（6 月 17 日提交）**

**是什么：**
VibeThinker-3B 是一个仅 3B 参数的密集推理模型，基于 Qwen2.5 架构，使用 spectrum-to-signal 后训练流水线。关键结果：AIME26 数学竞赛得分 94.3，追平 671B 参数的 DeepSeek V3.2 和 1T 参数的 Kimi K2.5；LeetCode 周赛通过率 96.1%，与 GPT-5.2 持平。其理论核心是：将 LLM 能力解耦为"知识存储"和"可验证推理"两个维度——推理能力可以被压缩到小模型中，知识存储才需要大参数量。引入 CLR（声明级可靠性评估）后进一步提升至 AIME26 97.1。

**怎么用：**
```bash
# Hugging Face 下载模型权重
huggingface-cli download vibe-research/VibeThinker-3B

# 使用 vLLM 部署
pip install vllm
vllm serve vibe-research/VibeThinker-3B --max-model-len 32768

# 直接使用 transformers 推理
python -c "
from transformers import AutoModelForCausalLM, AutoTokenizer
model = AutoModelForCausalLM.from_pretrained('vibe-research/VibeThinker-3B', device_map='auto')
tokenizer = AutoTokenizer.from_pretrained('vibe-research/VibeThinker-3B')
# 输入数学问题，模型自动进入推理模式
"
```

**为什么重要：**
3B vs 671B，参数差距 223 倍但性能持平——这不只是"小模型又进步了"，它验证了一个关于模型能力的深层假设：推理能力和知识存储可以被解耦。如果这个假说成立，未来的 AI 架构可能是"小推理模型 + 大知识库"的混合形态——本地跑 3B 的推理引擎，需要知识时查询外部检索。CLR（声明级可靠性评估）机制也有实用价值：模型在推理过程中自我评估每个步骤的可靠性，低可靠性步骤触发重新推理——这是提升小模型输出质量的重要技术方向。

**讨论现场：**
- 知乎热议："3B 参数打平 671B——这不是小模型的胜利，是'能力解耦'路线的胜利"
- 技术质疑："论文未报告对话、创意写作等多语言任务的性能，这恰恰是知识密集型任务——3B 的开放域能力可能远不如 671B"

**来源：** https://arxiv.org/abs/2606.16140 | https://github.com/vibe-research/VibeThinker-3B
