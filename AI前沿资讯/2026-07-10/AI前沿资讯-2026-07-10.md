## 1. OpenAI GPT-5.6

**评分: 9.5/10 | 可操作性: 高 | 热度: HN 1,080 points + 788 comments**

**是什么：**
OpenAI 在 7 月 9 日正式发布 GPT-5.6 系列，采用三档分层策略——Sol（旗舰）、Terra（均衡）、Luna（经济）。Sol 在 Terminal-Bench 2.1 编码基准上以 91.9%（Ultra 模式）和 88.8%（基础模式）双料登顶；Terra 以 Sol 一半的价格（$2.50/$15 per 1M tokens）提供接近 GPT-5.5 的性能；Luna 首次将前沿模型拉入 $1/$6 经济区间。三个型号共用同一模型架构，通过推理深度区分能力。

**怎么用：**
```bash
# API 接入（limited preview，需申请）
# Base URL: https://api.openai.com/v1
# Model ID: gpt-5.6-sol / gpt-5.6-terra / gpt-5.6-luna
# 价格：Sol $5/$30 · Terra $2.50/$15 · Luna $1/$6 (per 1M tokens)
curl https://api.openai.com/v1/chat/completions \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -d '{"model": "gpt-5.6-terra", "messages": [{"role": "user", "content": "..."}]}'
```

**为什么重要：**
这是 OpenAI 首次在旗舰代际同时推出三个价格层级，而不是等几个月再出"mini"版。Terra 是关键变量——GPT-5.5 质量但价格砍半，直接改变生产环境的成本结构。HealthBench Professional 临床推理测试中，Luna（$1/$6）居然超过 GPT-5.5（$5/$30），说明小模型推理能力正在追赶大模型。Sol Ultra 的 Terminal-Bench 91.9% 超过 Claude Mythos 5（88.0%）和 GPT-5.5（88.0%），重新夺回编码榜首。

**讨论现场：**
- HN 热评："Terra is the real story here. GPT-5.5 quality at half the price changes the unit economics for every production pipeline." — +156 points
- HN 另一高赞："The fact that Luna beats GPT-5.5 on clinical reasoning benchmarks should make everyone rethink their model routing strategy."
- 社区争议：GPT-5.6 目前是 limited preview，API 需要申请等待，不是立即可用——这点被多人吐槽。

**来源：** https://openai.com/index/gpt-5-6

## 4. iOfficeAI / OfficeCLI

**评分: 8.5/10 | 可操作性: 高 | 热度: GitHub 1,929 stars/day + 13,541 total**

**是什么：**
专为 AI Agent 设计的 Office 命令行套件。单个二进制文件，零依赖，不需要安装 Microsoft Office。Agent 可以直接通过 CLI 读取、编辑、自动化 Word（.docx）、Excel（.xlsx）和 PowerPoint（.pptx）文件。内置 HTML 渲染引擎可将文档转为 HTML/PNG 供 Agent 视觉检查——这就是 Agent 的"眼睛"，实现 render → look → fix 闭环。C# 实现，MIT 协议。

**怎么用：**
```bash
# 一行安装（Windows/Mac/Linux）
curl -fsSL https://officecli.ai/install.sh | bash
# Agent 通过 CLI 操作文档
officecli word render report.docx --format png
officecli excel update data.xlsx --cell A1 "Q3 Revenue"
officecli pptx create presentation.pptx --template quarterly_report
```

**为什么重要：**
企业和政府场景中，大量数据和决策流程仍然围绕 Office 文档运转。AI Agent 如果只能处理纯文本和代码，在这些场景中就是"半残"的。OfficeCLI 用最轻量的方式（单二进制、无 Office 依赖）打通了 Agent ↔ Office 文档的最后一公里。内置渲染引擎的设计让 Agent 能看到自己生成的文档长什么样——这是 Agent 自主迭代的核心能力。

**讨论现场：**
- 日本科技媒体 Gigazine 专题报道："OfficeCLI allows any AI agent to fully control Word, Excel, and PowerPoint for free."
- GitHub 讨论：有用户测试了 Agent 自主创建包含图表和公式的 Excel 财报，效果"超出预期"。

**来源：** https://github.com/iOfficeAI/OfficeCLI

## 6. DeepSeek-Reasonix

**评分: 8.0/10 | 可操作性: 高 | 热度: GitHub 1,654 stars/day + 26,473 total**

**是什么：**
DeepSeek 生态的原生终端编码 Agent，完全围绕 DeepSeek API 的前缀缓存特性设计。不经过 Anthropic/OpenAI 翻译层，直连 api.deepseek.com。核心设计哲学：利用 DeepSeek V4 的 byte-stable prefix-cache 机制，让缓存保持热状态，从而大幅降低长时间会话的 token 成本。实测单日工作负载下缓存命中率 99.82%，$12 完成等价 $61 的工作量。默认使用 DeepSeek-V4-Flash，可按次（`/pro`）或按会话（`/preset max`）升级到 V4-Pro。

**怎么用：**
```bash
git clone https://github.com/esengine/DeepSeek-Reasonix.git
cd DeepSeek-Reasonix
go build -o reasonix .
# 配置 DeepSeek API Key
export DEEPSEEK_API_KEY="sk-..."
./reasonix
# 内置命令：/pro（升级到 V4-Pro）、/plan（计划模式）、MCP 原生支持
```

**为什么重要：**
Reasonix 解决了一个被多数编码 Agent 忽略的经济学问题：前缀缓存稳定性。Claude Code 和 Codex 等通用 Agent 在多次 API 调用间很难保持缓存命中，导致实际成本远高于理论最优。Reasonix 的 99.82% 缓存命中率意味着长期运行成本下降 80%，对于把 Agent 当日常工具的开发者来说，每月可节省数百美元。MIT 协议 + DeepSeek 官方 awesome-deepseek-agent 目录收录。

**讨论现场：**
- Oosmetrics 数据显示 Reasonix 在 LLM 开发速度排名 Top 3。
- Discord 社区活跃讨论"leave it running"工作模式——Agent 开着不关，缓存一直热着。
- 社区关注：仅支持 DeepSeek API，不支持切换其他模型。

**来源：** https://github.com/esengine/DeepSeek-Reasonix

## 10. kyutai-labs / pocket-tts

**评分: 7.0/10 | 可操作性: 高 | 热度: GitHub 235 stars/day + 7,011 total**

**是什么：**
法国 AI 实验室 Kyutai 开源的一款极轻量文本转语音模型——100M 参数，纯 CPU 推理，MIT 协议。不需要 GPU，不需要云端 API，可以直接在笔记本甚至树莓派上运行。支持声音克隆和流式输出，延迟极低（首字延迟 < 100ms）。Python 实现，已被多个语音助手项目集成。

**怎么用：**
```bash
pip install pocket-tts
# 命令行使用
pocket-tts synthesize "Hello, this is pocket TTS running on CPU" --output hello.wav
# Python API
from pocket_tts import PocketTTS
tts = PocketTTS()
tts.synthesize("你好，我正在用 CPU 运行", output="output.wav")
```

**为什么重要：**
100M 参数 + CPU-only 意味着 TTS 不再需要云端服务和 GPU 集群。对于嵌入式设备、离线应用、隐私敏感场景，这是巨大的突破。对比 ElevenLabs 等商业 TTS API 按字符计费的模式，pocket-tts 的 MIT 协议意味着零边际成本。235 stars/day 不算爆炸但持续稳定增长，说明这是有真实需求的基础设施类项目。

**讨论现场：**
