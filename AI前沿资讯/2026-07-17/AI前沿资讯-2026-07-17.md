## 1. Kimi K3（Moonshot AI）——2.8 万亿参数开源权重旗舰，多项 Agentic 基准登顶

**评分: 9.5/10 | 可操作性: 高 | 热度: HN 1190 points / 742 评论，全平台刷屏**

**是什么：**
月之暗面（Moonshot AI）于 7 月 17 日正式发布 Kimi K3，2.8 万亿参数 MoE 模型，896 个专家中每 token 激活 16 个，原生多模态（文本+图像+视频），100 万 token 上下文窗口。两项核心架构创新：Kimi Delta Attention（KDA）在百万 token 上下文中解码速度提升最高 6.3 倍；Attention Residuals（AttnRes）训练效率提升约 25%，额外成本低于 2%。权重格式 MXFP4，激活值 MXFP8。同步上线 Kimi.com、Kimi Work、Kimi Code 和 API。最关键的是：**开源权重承诺 7 月 27 日前发布**。

**怎么用：**
```bash
# Kimi API（OpenAI SDK 兼容）
pip install openai
export KIMI_API_KEY="your-key"

# 调用示例
curl https://api.moonshot.cn/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $KIMI_API_KEY" \
  -d '{
    "model": "kimi-k3",
    "messages": [{"role": "user", "content": "分析这份财报的核心风险"}],
    "max_tokens": 131072
  }'

# 7月27日开源后本地部署（vLLM，推荐64+加速器超节点）
# huggingface-cli download moonshot-ai/Kimi-K3 --local-dir ./kimi-k3
# vllm serve ./kimi-k3 --tensor-parallel-size 8 --max-model-len 1000000
```

**为什么重要：**
API 定价 $3/$15 每百万 token（cache-hit 输入 $0.30），在多项 Agentic 基准上超越或逼近闭源旗舰：Terminal Bench 2.1 达 88.3（vs Fable 5 84.6）、Program Bench 77.8（榜首）、SWE Marathon 42.0（榜首）、BrowseComp 91.2（榜首）。在 DeepSWE（67.5 vs GPT-5.6 Sol 73.0）和 FrontierSWE（81.2 vs Fable 5 86.6）仍有差距。Moonshot 坦诚承认与 Fable 5/GPT-5.6 Sol 之间存在 "用户体验上的明显差距"，但对开源生态而言，这是一个与闭源旗舰正面竞争的开源权重模型，7 月 27 日后任何人都能下载 2.8T 参数的完整权重。

**讨论现场：**
- HN 热评："This is the open-weights moment we've been waiting for since DeepSeek V4. 2.8T parameters with open weights in 10 days — the economics of the API market are about to shift again." — 置顶评论
- 社区关注 Moonshot 坦诚的局限性声明，认为这种透明度在模型发布中罕见且值得赞赏
- 开发者担忧实际推理成本：K3 仅支持 `reasoning_effort: max`，每任务 token 消耗可能偏高

**来源：** https://www.digitalapplied.com/blog/kimi-k3-open-frontier-model-release-2026 | https://platform.moonshot.cn/docs

## 2. Gemini 3.5 Pro 正式上线——Google 的 Agentic 旗舰，API 即日可用

**评分: 8.8/10 | 可操作性: 高 | 热度: 跨渠道综合热度极高，多日前即为社区焦点**

**是什么：**
Google DeepMind 于 7 月 17 日按预定计划上线 Gemini 3.5 Pro，定位为面向长时间 Agentic 任务的旗舰模型。经过从 6 月推迟到 7 月的额外测试和优化，重点加强了长程 Agent 任务中的稳定性和可靠性。同期，Google 将 NotebookLM 更名为 Gemini Notebook，整合进 Gemini 品牌体系。Google Search 结果也已全面由 Gemini 3.5 Flash 生成的 AI 摘要取代传统 "十条蓝链"。

**怎么用：**
```bash
# Gemini API
pip install google-genai

# Python SDK 调用
import google.generativeai as genai
genai.configure(api_key="your-api-key")
model = genai.GenerativeModel("gemini-3.5-pro")
response = model.generate_content("分析这份合同的风险条款")

# 或通过 OpenAI 兼容接口
curl https://generativelanguage.googleapis.com/v1beta/openai/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $GEMINI_API_KEY" \
  -d '{"model":"gemini-3.5-pro","messages":[{"role":"user","content":"Hello"}]}'

# Gemini CLI 也已支持 3.5 Pro
gemini-cli --model gemini-3.5-pro
```

**为什么重要：**
预期 API 定价 $1.25 输入 / $10 输出每百万 token，输入成本仅为 GPT-5.6 Sol 的 1/4。Google 将 NotebookLM 整合进 Gemini 品牌，加上 Google Search 全面 AI 化，展现了 "模型+产品+搜索" 三位一体的生态策略。对发布商而言，传统 SEO 流量模式被 AI 摘要彻底颠覆——不再靠排名获取点击，而是竞争被 AI 答案引用的资格。

**讨论现场：**
- HN 热评："Google is eating its own ecosystem. The ten blue links were Google's core product for 25 years. Now they're gone, replaced by a model that may or may not cite you." — +200 points
- 开发者关注 $1.25 输入价格，认为这将对 OpenAI 和 Anthropic 的定价策略产生压力
- NotebookLM 老用户对品牌整合态度分裂：一部分认为更方便，另一部分怀念独立产品

**来源：** https://blog.google/technology/developers/gemini-3-5-pro/ | https://blog.google/products/gemini/

## 5. OfficeCLI ——首款专为 AI Agent 设计的 Office 套件，单日 1,224 stars

**评分: 8.0/10 | 可操作性: 极高 | 热度: GitHub 1,224 stars/day**

**是什么：**
iOfficeAI 团队发布的 OfficeCLI 是一个专门给 AI Agent 用的 Office 工具集，支持读写 Word、Excel、PowerPoint 文件。单个二进制文件，无需安装 Microsoft Office。Agent 可以直接通过命令行操作 Office 文档——读取表格数据、生成报告、修改演示文稿。对自动化办公场景（周报生成、数据分析、合同填充）是一站式解决方案。

**怎么用：**
```bash
# 下载单个二进制（Windows/Mac/Linux）
# 从 GitHub Releases 获取：https://github.com/iOfficeAI/OfficeCLI/releases

# 读取 Excel 文件
officecli read --format json --input report.xlsx

# Agent 生成 Word 文档
officecli write --template template.docx --data '{"name":"张三","date":"2026-07-17"}' --output result.docx

# 在 Claude Code / Codex 中作为 MCP tool 使用
# 配置 .mcp.json 指向 officecli 二进制即可
```

**为什么重要：**
AI Agent 的核心瓶颈在于无法操作实际工作中的文件格式——Office 文档是企业的通用语言。OfficeCLI 填补了 Agent 能力与办公现实之间的这个缺口。它以 C# 编写，MIT 许可，单二进制零依赖，可直接嵌入任何 Agent 工具链。

**怎么用：**
```bash

cp -r graphify ~/.claude/skills/graphify

# "@graphify index this project"
```

**为什么重要：**

## 9. Manta AI ——自主 Web 应用测试 Agent，Product Hunt 日榜第一

**评分: 7.0/10 | 可操作性: 高 | 热度: Product Hunt daily #1（7月16日）**

**是什么：**
Manta AI 是一个自主测试 Agent，给它一个 URL，它像真实用户一样探索你的 Web 应用——自动发现页面流程、识别 bug、生成自愈测试用例。用自然语言描述测试流程（如 "测试登录后修改密码的完整路径"），Agent 自动执行并生成报告。当 UI 变化时测试用例自动适配。可本地或服务器运行，测试内网/本地应用无限制。提供免费 tier。

**怎么用：**
```bash
# 注册并安装 CLI
npm install -g manta-ai
manta login

# 启动自主测试（无需写脚本，自然语言描述即可）
manta test "https://your-app.com" \
  --flow "用户注册 → 邮箱验证 → 首次登录 → 修改个人信息 → 登出"

# 或直接让 Agent 自主探索
manta explore "https://your-app.com" --depth 3
```

**为什么重要：**
E2E 测试的维护成本是 QA 团队的长期痛点——UI 一改，测试脚本全废。Manta AI 的 "自愈" 测试用例直接解决了这个问题：Agent 理解页面的语义结构而非 DOM 位置，UI 变动时自动重新定位元素。AI 时代的测试范式正在从 "写脚本" 转向 "描述意图"。

**讨论现场：**
- Product Hunt 评论："Finally, testing that doesn't feel like paying technical debt interest."
- 参与者关注与 Browser-use、Playwright MCP 的差异化：Manta 专注于测试场景而非通用浏览器自动化
