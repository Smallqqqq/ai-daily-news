# Claudeception — 让 Claude Code 拥有"记忆力"，越用越聪明

  每次打开 Claude Code 新会话，Agent 都像失忆一样从零开始。你花一小时踩坑调出来的诡异 bug，下次碰到还得再来一遍。Claudeception 解决了这个 AI Agent 最根本的痛点——它能在你调试过程中自动识别"值得记住"的知识，写成 SKILL.md 存下来，下次遇到同类问题自动加载。2317 stars，MIT 协议，可能是你装过回报率最高的一个技能。

### 它解决什么问题

  用 AI Agent 写代码有一个结构性缺陷：**每次会话都是全新的大脑**。你花了半小时排查出一个 Prisma 连接池耗尽的问题，Agent 学会了。会话结束。下周一你换了个分支，Agent 又忘光了。

  Claudeception 把 Claude Code 原生的技能系统从"只读"变成了"可写"。它监控你的调试过程，识别那些经过实际验证的、非显而易见的解决方案，然后自动生成带精准触发条件的 SKILL.md 文件。下次你再碰到 `PrismaClientKnownRequestError: Too many database connections`，Agent 不再从头排查——它直接加载你上次总结的经验。

### 核心原理

  Claude Code 的技能系统在启动时加载所有技能的 `name` + `description`（约 100 tokens/个），然后用当前上下文去匹配描述，匹配上了就加载完整内容。这个匹配-加载机制原本只是"读"——技能由人提前写好。

  Claudeception 的关键洞察是：**这个检索系统也可以被写入**。它通过一个 `UserPromptSubmit` 钩子，在每次你发消息时注入一段提醒，让 Agent 评估当前任务是否产生了"可提取知识"：

  - 是否经过实际调试（而非查文档就能解决）？
  - 对未来任务是否有复用价值？
  - 触发条件是否足够明确（能写出精准的描述）？
  - 解决方案是否已验证有效？

  四个条件全部满足，Agent 才会用标准的 YAML 头 + Markdown 体格式写一个新技能文件。写出来的技能文件描述字段高度精准，比如「修复 PrismaClientKnownRequestError：serverless 环境下数据库连接数超限」，而不是模糊的「帮助解决数据库问题」。

  这个设计思路有扎实的学术基础——Voyager（2023）证明了游戏 Agent 可以通过积累技能库避免重复学习，CASCADE（2024）提出了"元技能"（获取技能的技能）概念，SEAgent（2025）展示了 Agent 通过试错学习新软件环境的能力。

### 安装配置

  两步完成，全程不超过两分钟：

**第一步：克隆技能文件**

```bash
# 全局安装（推荐，所有项目共享）
git clone https://github.com/blader/Claudeception.git ~/.claude/skills/claudeception

# 或项目级安装
git clone https://github.com/blader/Claudeception.git .claude/skills/claudeception
```

**第二步：配置激活钩子**

```bash
# 创建 hooks 目录并复制激活脚本
mkdir -p ~/.claude/hooks
cp ~/.claude/skills/claudeception/scripts/claudeception-activator.sh ~/.claude/hooks/
```

  然后在 `~/.claude/settings.json` 中添加钩子配置：

```json
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/claudeception-activator.sh"
          }
        ]
      }
    ]
  }
}
```

  如果已有 `settings.json`，把 `hooks` 部分合并进去即可。钩子的作用是在每次你发消息时注入评估提醒，比纯靠语义匹配的激活率高得多。

### 使用指南

#### 场景一：调试完成后自动沉淀经验

  你正在排查一个诡异的报错——Next.js 服务端渲染时 `fetch` 返回了浏览器的 User-Agent 字符串而不是 API 数据。经过二十分钟的 `console.log` 和抓包，你发现是因为 middleware 里的 `fetch` 默认不携带原始请求头。

  Claude 帮你定位到根因并修复后，什么都不用做。钩子会在下一条消息时触发 Claudeception，让它评估刚才的排查过程是否满足提取条件。一旦满足，Agent 自动生成一个技能文件，描述为「Next.js middleware 中 fetch 请求丢失原始 headers 的问题及解决方案」。

  下次你在任何项目中碰到类似现象，Agent 直接匹配到这个技能，省掉二十分钟的排查。

#### 场景二：主动触发学习回顾

  你刚完成了一个复杂的功能开发，想把项目特有的几个约定沉淀下来。直接输入：

```
/claudeception
```

  或说：

```
把我们刚才学的保存成技能
```

  Agent 会回顾本次会话中的关键发现，逐个评估是否值得提取。你可以在它生成的技能列表中选择保留哪些。

#### 场景三：新人入职的知识传递

  你的团队在用一套非标准的部署流程——先跑 `db:migrate:dry-run` 检查迁移安全性，再手动执行 `deploy:staging`，最后等监控面板的绿色指示灯亮了才能合并到 main。新人每次都要问一遍。

  把 Claudeception 装到团队共享的项目级 `.claude/skills/` 里，老员工日常工作中发现的约定和踩坑经验会被自动写进去。新人的 Agent 启动时加载这些技能，等于直接继承了团队的集体经验，不会再把 `deploy:staging` 直接对着生产环境跑。

### 进阶技巧

  **和 Claudeception 搭配 `memory` 系统使用效果翻倍。** Claude Code 的 memory 系统（`~/.claude/projects/*/memory/`）记录的是关于用户偏好和项目背景的元信息，而 Claudeception 记录的是技术解决方案。两者互补——memory 告诉 Agent「这个项目用 Prisma + PostgreSQL」，Claudeception 告诉 Agent「这个项目在 serverless 上跑 Prisma 时连接池设成 `connection_limit=3` 否则会炸」。

  **手动审查生成的质量。** 虽然 Claudeception 自带质量门控（必须是实际调试过的、有明确触发条件的、验证有效的），但建议前几次使用后手动检查 `~/.claude/skills/` 下生成的文件——确认描述是否精准、解决方案是否完整。一旦你对生成质量建立了信任，后续就可以全自动了。

### 常见踩坑

  **钩子没生效**：最常见的遗漏是忘了把激活脚本设为可执行。Windows 用户（Git Bash / WSL）不需要 `chmod`，但 macOS / Linux 用户一定要跑 `chmod +x ~/.claude/hooks/claudeception-activator.sh`。

  **生成频率太高**：如果 Agent 把每次文档查询也当成"发现"来保存，说明 SKILL.md 中的提取标准需要收紧。可以在 `~/.claude/skills/claudeception/SKILL.md` 中加强「只有当解决方案需要实际调试和推理时才提取」的约束。

  **全局 vs 项目级的选择**：全局安装适合个人积累通用经验（如 Prisma 连接池、TypeScript 循环引用等跨项目的坑）。项目级安装适合沉淀团队特定约定（部署流程、内部库用法、业务规则）。建议两者都装——全局共享通用技能，项目级沉淀专属知识。

### 同类技能对比

| 技能 | 安装方式 | 核心特点 | 适用场景 | 热度 |
|------|---------|---------|---------|------|
| **Claudeception**（今日推荐）| `git clone` + hook 配置 | 自动从调试中提取经验，生成可复用技能 | 所有需要持续积累项目经验的开发者 | ⭐ 2317 |
| SkillAnything | 插件市场安装 | 手动描述需求，Agent 帮你写技能文件 | 一次性创建特定领域的技能 | ⭐ ~500 |
| Claude memory 系统 | Claude Code 内置 | 自动记录用户偏好和项目元信息 | 记录"人"相关的偏好，不是技术方案 | 内置 |

  Claudeception 的独特之处在于**全自动 + 精准触发**。SkillAnything 是你想到了才去创建一个技能，Claudeception 是你踩完坑它就自动帮你记住。它不是"技能创建工具"，而是"技能自动生长系统"。
