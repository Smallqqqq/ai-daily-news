# Cloudflare Skills — 让你的 AI Agent 原生操控 Cloudflare 全家桶

  你在 Claude Code 里说"帮我部署一个 Worker 到 Cloudflare"，Agent 开始瞎编 API、发明不存在的 Wrangler 命令、把 KV 的 `put` 写成 `set`。Cloudflare 官方出品的这套 Agent Skills 直接解决了这个问题——10 个技能 + 2 个命令 + 5 个 MCP 服务器，一次性把 Cloudflare 开发者平台的全部知识塞进你的 AI Agent，让它像读官方文档一样准确。

  1648 stars，20 位 Cloudflare 员工贡献，Apache 2.0 协议。这不是社区玩具，是 Cloudflare 官方维护的第一方技能包。

### 它解决什么问题

  用 AI Agent 做 Cloudflare 开发时，最常见的问题是 Agent 对 Cloudflare API 的知识停留在训练数据截止日期——它不知道新版本 Wrangler 的命令变化、不了解 D1 的 SQLite 限制、不清楚 Workers AI 的模型列表。结果就是反复试错、来回纠正，效率极低。

  Cloudflare Skills 把 Workers、KV、D1、R2、Workers AI、Durable Objects、WAF、Tunnel 等全部产品的权威知识，打包成 Agent 可以按需加载的技能文件。Agent 读到的不再是"猜测"，而是直接从 Cloudflare 官方文档提取的最新信息。

### 核心原理

  Cloudflare Skills 采用三层架构：

  **第一层：技能文件（Skills）**。每个技能是一个 `SKILL.md` 文件，采用 Agent Skills 开放标准（agentskills.io）。Agent 启动时只加载技能的 `name` + `description`（约 100 tokens），当你的任务匹配到某个技能的描述时，Agent 才加载完整内容——这叫渐进式披露（Progressive Disclosure），不影响上下文窗口。

  **第二层：斜杠命令（Commands）**。`/cloudflare:build-agent` 和 `/cloudflare:build-mcp` 两个手动调用的命令，帮你一键搭建 AI Agent 或 MCP Server 的脚手架。

  **第三层：MCP 服务器**。5 个远程 MCP 服务器提供实时能力——`cloudflare-api` 操作账户资源、`cloudflare-docs` 拉取最新文档、`cloudflare-bindings` 辅助开发、`cloudflare-builds` 管理构建、`cloudflare-observability` 查日志和监控。

### 安装配置

  最推荐的方式是通过 Claude Code 插件市场一键安装：

```
/plugin marketplace add cloudflare/skills
/plugin install cloudflare@cloudflare
```

  安装完成后，10 个技能自动注册。无需额外配置，下次对话中涉及 Cloudflare 相关话题时，Agent 会自动加载对应技能。

  其他 Agent 的安装方式：

```bash
# 通过 npx（适用于 Codex、Claude Code、Cursor 等所有支持的 Agent）
npx skills add https://github.com/cloudflare/skills

# Cursor 用户
# Settings > Rules > Add Rule > Remote Rule (Github) → 填入 cloudflare/skills

# 手动克隆安装
git clone https://github.com/cloudflare/skills.git
cp -r skills/cloudflare ~/.claude/skills/
```

### 使用指南

#### 场景一：部署一个 Cloudflare Worker

  直接对 Agent 说："帮我在 Cloudflare 上部署一个 Worker，功能是把请求里的 JSON body 存到 KV，然后返回一个短链接。"

  Agent 会：
  1. 自动加载 `cloudflare` 和 `wrangler` 技能
  2. 创建 Worker 代码（使用正确的 `c.env.KV_NAMESPACE.put()` API）
  3. 通过 MCP 服务器查询 KV namespace 是否存在
  4. 配置 `wrangler.toml`，绑定 KV
  5. 执行 `wrangler deploy` 完成部署

  整个过程 Agent 用的是真实 API，不会发明不存在的参数。

#### 场景二：创建 D1 数据库并写查询

  对 Agent 说："在 Cloudflare D1 里创建一个 users 表，包含 id、name、email、created_at 字段，然后写一个 Worker 提供 REST API 查询用户列表。"

  Agent 加载 `cloudflare` 技能后，会：
  1. 调用 MCP 服务器的 `cloudflare-bindings` 创建 D1 数据库
  2. 生成正确的 SQL（D1 基于 SQLite，语法不同于 PostgreSQL/MySQL）
  3. 在 `wrangler.toml` 中绑定数据库
  4. 编写 Worker 代码，用 `c.env.DB.prepare("SELECT * FROM users").all()` 查询
  5. 自动处理 D1 的 JSON 序列化限制

#### 场景三：搭建一个带状态的 AI Agent

  直接调用斜杠命令：

```
/cloudflare:build-agent
```

  Agent 会加载 `agents-sdk` 和 `building-ai-agent-on-cloudflare` 技能，引导你：
  1. 选择 Agent 类型（聊天 / Slack / 邮件 / 语音）
  2. 生成基于 Agents SDK 的项目脚手架
  3. 配置状态管理（Durable Objects + SQLite）
  4. 接入 WebSocket 或 HTTP 通信通道
  5. 设置调度任务和 RPC 通信

  完成后你得到一个可以直接 `npm run dev` 跑起来的 Agent 项目。

#### 场景四：审计网站性能

  对 Agent 说："检查一下 example.com 的 Core Web Vitals。"

  Agent 加载 `web-perf` 技能，调用 MCP 服务器的 `cloudflare-observability`，输出：
  - FCP（First Contentful Paint）
  - LCP（Largest Contentful Paint）
  - TBT（Total Blocking Time）
  - CLS（Cumulative Layout Shift）
  - 渲染阻塞资源列表
  - 优化建议（按优先级排列）

### 进阶技巧

  **组合使用多个技能**：部署一个完整的 AI 应用时，Agent 会同时加载 `cloudflare`（平台总览）、`wrangler`（部署工具）、`agents-sdk`（Agent 框架）、`durable-objects`（状态管理）四个技能，互不冲突，层层递进。

  **MCP 服务器提供实时数据**：技能文件里的知识是静态的（写于发布时），但 MCP 服务器提供的是实时的。比如 `cloudflare-docs` MCP 会实时拉取最新的 API 文档，所以即使 Cloudflare 发布了新功能，Agent 也能在技能基础上查到最新用法。

### 常见踩坑

  **Wrangler 版本兼容性**：技能假设你使用 Wrangler 3.x。如果你本机装的是 Wrangler 2.x，部分命令（如 `wrangler deployments list`）不存在。升级方法是 `npm install -g wrangler@latest`。

  **D1 的 SQLite 限制**：D1 底层是 SQLite，不支持 PostgreSQL 的 `RETURNING` 子句、MySQL 的 `ON DUPLICATE KEY UPDATE` 等语法。Agent 加载技能后会遵守这些限制，但如果你手动写了不兼容的 SQL，Agent 会根据技能的约束自动修正。

  **MCP 服务器需要认证**：`cloudflare-api` 和 `cloudflare-builds` MCP 需要有效的 Cloudflare API Token。如果 Agent 报"未授权"错误，需要在 Cloudflare Dashboard 创建一个 API Token（My Profile → API Tokens → Create Token），然后配置到 MCP 服务器中。

## 今日小结

  Cloudflare Skills 适合所有在 Cloudflare 平台上做开发的工程师。花 30 秒安装，之后你的 AI Agent 就再也不需要你去翻 Wrangler 文档、查 API 参数、试错 SQL 语法了。如果你日常用到 Workers、KV、D1、R2 或 Workers AI 中的任何一项，这套技能几乎是必装。如果你用 Cloudflare One（Access、Gateway、WARP）做企业安全，还有专属的 `cloudflare-one` 和 `cloudflare-one-migrations` 技能，覆盖了从配置到迁移的完整流程。
