# EvoSkill — 让 AI Agent 的技能从失败中自动进化

  你给 Claude Code 装了五六个 skill，起初跑得挺好。两个 sprint 之后，代码库变了、API 升级了、你的需求也变了，那些 skill 开始给出过时的答案，触发越来越不稳定，但你哪有时间一个个去重写？EvoSkill 把这件事自动化了：给它一组测试用例，它反复让 agent 做题、看哪里错了、自动生成或修改 skill 来修复。跑完 20 轮迭代后，准确率能从 42% 提到 67%。960 stars，Apache 2.0 协议，2026 年 6 月刚发 v1.3.0。

### 它解决什么问题

  手工维护 agent skill 有三个死结：

  1. **写完就过时**。skill 是在某个时间点、针对某个代码库状态写的。代码库在变，skill 不会自动跟着变。
  2. **调参靠直觉**。system prompt 加一句话、skill 改一段描述——改了之后真的变好了吗？没有量化反馈，全靠感觉。
  3. **无法跨任务复用**。你为项目 A 调试出来的 skill 经验，没法自动迁移到项目 B。

  EvoSkill 的核心思路是：把 agent 的配置（system prompt + 技能文件）当作可进化的基因组。给定一组带标准答案的测试用例，它反复运行 agent → 收集失败样本 → 分析失败原因 → 生成新的 skill 或改写 prompt → 再跑一遍验证。这个循环自动跑 20 轮，每轮保留表现最好的 3 个版本（frontier），确保只进化不退步。

### 核心原理

  EvoSkill 的进化循环包含五个阶段，跑在 git 分支上，每一轮迭代就是一个新分支：

  ```
  训练集样本
      ↓
  [运行 agent] → 收集通过/失败结果
      ↓
  [分析失败]  → 提取失败模式（哪类问题总是答错？）
      ↓
  [生成 skill] → 针对失败模式生成新的 SKILL.md 或修改 system prompt
      ↓
  [验证]      → 用新配置重新跑训练集，计算准确率变化
      ↓
  保留 frontier（Top 3）→ 下一轮迭代
  ```

  **它不是在改代码**——EvoSkill 操作的是 agent 的 system prompt 和 skill 文件，生成的产物是标准 `SKILL.md` 格式，可以直接复制到 Claude Code 的 `~/.claude/skills/` 目录使用。

  关键设计决策：
  - **Frontier 机制**：每轮保留最好的 3 个版本。新变体必须超过 frontier 中最差的那个才会被保留，确保只进化不退化。
  - **Git 原生**：每一轮迭代对应一个 `program/iter-skill-N` 分支，`evoskill diff` 可以对比任意两轮之间的 skill 变化。
  - **跨模型迁移**：论文实验表明，用 Claude 进化出来的 skill，迁移到 GPT 上同样有效——因为进化的是 skill 的文本内容，不是模型特定的 prompt 技巧。

### 安装配置

  一行命令安装：

```bash
curl -fsSL https://raw.githubusercontent.com/sentient-agi/EvoSkill/main/install.sh | bash
```

  前置依赖：Python 3.12+。安装脚本会自动处理 `uv` 包管理器和 Python 依赖。然后配置 API key：

```bash
# Claude Code 作为被进化的 agent
export ANTHROPIC_API_KEY=your-key-here

# 或用 OpenRouter（可调用任意模型）
export OPENROUTER_API_KEY=your-key-here
```

  在你的项目根目录初始化：

```bash
evoskill init
```

  交互式问答会引导你选择 agent 运行时（Claude Code / Codex CLI / OpenCode / Goose / OpenHands）、数据集来源（CSV 文件或 Harbor 基准）、执行模式（本地 / Docker / Daytona 远程）。

### 使用指南

#### 场景一：优化客服问答 agent 的准确率

  你有一个自动回答用户问题的 agent，但发现某些类型的问题总是答错。准备一个 CSV 文件，每行一个问题和一个标准答案：

```csv
question,answer
"订单超过 48 小时未发货怎么处理","自动触发加急流程并补偿 10 元优惠券"
"VIP 用户的退货期限是多久","30 天，普通用户为 7 天"
```

  运行 `evoskill init` 选择 CSV 数据源，指向这个文件。然后 `evoskill run`。EvoSkill 会让 agent 逐题作答，比对标准答案，把答错的题目归因到具体的能力缺失（比如"不理解 VIP 退货政策"），自动生成一个 `vip-return-policy` skill。

```bash
  Iter  Accuracy  Δ          Skills  Frontier  Status
  1     42.0%     —          0       [1]       baseline
  2     51.3%     +9.3%      1       [1, 2]    ★ new best
  3     49.7%     -1.6%      1       [1, 2]    discarded
  4     58.2%     +6.9%      2       [1, 4]    ★ new best
  ...
```

#### 场景二：用 Harbor 基准测试进化编码 agent

  如果你的 agent 做的是代码生成/修复类任务，可以用 Harbor 数据集替代 CSV。Harbor 提供容器化的编程任务，每个任务自带验证器（verifier）：

```bash
evoskill init
# 选择 Harbor → swe-bench/swe-bench-verified
evoskill run
```

  Harbor 模式会在 Docker 容器中运行 agent，自动判断任务是否通过。适合 SWE-bench 这类需要实际运行代码验证的场景。

#### 场景三：跨任务复用进化后的 skill

  EvoSkill 论文中的核心发现：为 SealQA（搜索增强问答）进化出来的 skill，迁移到 BrowseComp（网页浏览问答）上，零样本准确率提升了 5.3%。操作方式很简单：`evoskill run` 跑完后，`git checkout program/iter-skill-N` 切到最佳分支，把 `.claude/skills/` 目录复制到另一个项目中即可。

### 进阶技巧

  **混合 provider 降成本**。进化循环的计算开销不小——每轮要跑 N 个样本，每个样本都是一次完整的 agent 调用。EvoSkill v1.3.0 支持用 Fireworks AI 上的开源模型（如 Llama 70B）替代 Claude API 做评分，大幅降低成本。配置方式：

```toml
[harness]
name = "openhands"
model = "fireworks_ai/accounts/fireworks/models/llama-v3p1-70b-instruct"
```

  **自定义评分器**。默认的 `multi_tolerance` 评分器做精确字符串匹配，如果你的任务需要语义匹配（比如"答案意思对但措辞不同"应该算对），可以换用 LLM 评分器：

```toml
[scorer]
type = "llm"
model = "anthropic/claude-sonnet-4-6"
```

### 常见踩坑

  **API 费用**。进化循环会大量调用 API——假设 20 轮、每轮 50 个样本、frontier=3，每次调用约 2000 token，用 Claude Sonnet 的话一轮大约 $2-3。建议先用小样本（train_ratio=0.1）跑通流程，确认有效后再扩大。

  **进化卡住**。如果连续 5 轮没有提升（`no_improvement_limit = 5`），循环自动停止。这种情况通常是任务描述不够清晰——编辑 `.evoskill/task.md` 补充更多示例和约束后 `evoskill run --continue` 继续。

### 同类技能对比

| 技能 | 安装方式 | 核心特点 | 适用场景 | 热度 |
|------|---------|---------|---------|------|
| **EvoSkill**（今日推荐）| `curl \| bash` 一键安装 | 从失败中自动进化 skill，支持 6 种 agent | 有测试集的 agent 优化 | ⭐ 960 |
| SkillAnything | `git clone` 到 skills 目录 | 7 阶段流水线生成全新 skill | 从零创建 skill | ⭐ 445 |
| skill-creator | Claude Code 内置 | 交互式 Q&A 生成 skill | 快速原型 | 内置 |

  EvoSkill 和 SkillAnything 解决的是不同阶段的问题：SkillAnything 帮你**从零创建**一个 skill，EvoSkill 帮你**持续优化**已有的 skill。两者互补——先用 SkillAnything 生成初版，再用 EvoSkill 跑进化循环打磨。

## 今日小结

  EvoSkill 适合那些已经在用 agent skill、但发现 skill 质量会随时间衰减的团队。它把一个原本靠直觉的手工调参过程，变成了有量化指标、可复现、可回滚的工程流程。配置花 10 分钟，跑一轮进化可能需要 1-2 小时（取决于样本量和 API 速度），但换来的是一套能随代码库一起进化的 skill，而不是一潭死水的手写文档。
