# SkillAlchemy — 把你脑子里那个「要是 AI 能这样做就好了」变成一行命令

  你有一个清晰的工作流——每轮代码审查都要先跑一遍安全检查、再对一遍设计文档、最后确认测试覆盖。你反复跟 Claude Code 说同样的前置指令，每次都要打一大段话。你知道应该把它写成一个 Skill，但打开 SKILL.md 空白文件的时候，你发现自己并不缺想法——你缺的是把想法翻译成 Agent 能稳定执行的指令格式。

  SkillAlchemy 干的就这一件事：你说「我要一个检查代码安全性的 skill」，它派出 5 个研究 Agent 并行搜罗最佳实践，自动检索现有类似 skill 作为参照，然后编译出一个结构完整、可直接安装的 SKILL.md。你不写 prompt，你只描述需求。69 stars，MIT 协议，一行命令安装。

---

### 它解决什么问题

  给 Agent 写 Skill 这件事，有一个经典的鸡生蛋问题：你不会写 Skill 的时候，Agent 也帮不了你；Agent 帮不了你的时候，你就永远学不会写 Skill。

  现有的解决方案分两派。第一派是「模板填空」——Vercel 的 `npx skills init` 和 Anthropic 的 `skill-creator` 给你一个骨架，让你自己填内容。这对知道自己要写什么的人有用，但对「我知道要做什么但不知道怎么表达成步骤」的人没用。第二派是「抄别人的」——去 awesome-claude-skills 找一个类似的，改改就用。问题是别人的 Skill 解决的是别人的场景，你改得越多越容易改坏。

  SkillAlchemy 选了第三条路：**不给你模板，也不让你抄。它直接把你的需求当成研究课题，用多 Agent 并行深挖，自动生成完整的 Skill。** 这个思路的关键差异在于——它把 Skill 创作从"写作任务"变成了"研究任务"。你不需要知道 SKILL.md 的最佳实践、不需要理解 frontmatter 的每个字段、不需要琢磨触发条件的措辞——你只需要描述你想要什么。

  SkillsBench（84 个任务，11 个领域）的基准测试数据很能说明问题：Claude Code + DeepSeek-V4-Pro 裸跑只有 20.2% 的通过率，加载人工精选的 Skill 后拉到 50.0%，而加载 SkillAlchemy 自动生成的 Skill 后达到了 57.1%。自动生成的 Skill 在基准测试上跑赢了人工精选的——这不是因为 AI 比人更懂怎么写 Skill，而是因为它能并行查阅的信息量远超一个人手动搜文档所能覆盖的。

### 核心原理

  SkillAlchemy 内部编排了两个子 Skill：**Lens** 和 **LEAP**。它们各自独立安装、独立运行，SkillAlchemy 只负责调度和用户交互。

  **Lens（看清问题）：** 用户说的通常是模糊的一句话，比如「我想要一个帮我写 commit message 的 skill」。Lens 的工作是把这句话展开成多维度分析——这个 skill 的目标用户是谁？触发条件是什么？输入输出是什么？边界条件有哪些？已有类似 skill 的不足在哪里？输出不是一段 prompt，而是一个结构化的维度矩阵，每个维度标注了意图类型（distill_persona / distill_method / fuse_skills）。

  **LEAP（执行落地）：** 拿到 Lens 的维度矩阵后，LEAP 进入 7 阶段管线——Research Swarm（4-5 个 Agent 并行搜索）→ Exemplar Discovery（用 find-skills 在线检索已有 skill 并自动评分）→ Synthesis（合成研究结果）→ Gate 1（质量门禁，不通过打回重做）→ Compile（编译为符合 agentskills.io 标准的 SKILL.md）→ Gate 2（最终验证）→ Cleanup（清理临时文件）。整条管线跑完，输出目录里只有三样东西：研究证据（R*.md）、审计追踪（intermediate/）、和最终的 Skill 包。

  三个深度级别控制整个流程的资源投入：

| 深度 | Agent 数 | 耗时 | 验证 | 适用场景 |
|------|---------|------|------|---------|
| quick | ≤3 | ~5-8 min | 跳过 | 快速原型，先跑一个看看方向对不对 |
| standard | 4-5 | ~15-20 min | 建议 | 日常使用，大多数场景的默认选择 |
| deep | 6-8 | ~25-35 min | 强制 + 双审核 | 发布级，要给别人用的 skill |

  整个流程中有 2 个确认点（任务简报、research plan），用户可以在每个节点介入修改。也支持"一路默认"模式——说一句「一路默认跑完」，全程不打扰。

### 安装配置

```bash
# 一行安装（推荐）
npx skills add agentsope/SkillAlchemy
```

也可以直接在 Claude Code 里说：

```
请帮我从 https://github.com/agentsope/SkillAlchemy 拉取这个 skill 到本地安装
```

SkillAlchemy 安装后会自动加载，但它依赖两个子 Skill——Lens 和 LEAP。首次使用时它会自动检测并提示安装：

```bash
npx skills add agentsope/SkillAlchemy/skills/Lens
npx skills add agentsope/SkillAlchemy/skills/LEAP
```

三个 Skill 全部到位后，直接跟 Claude Code 说你的需求即可触发。

### 使用指南

#### 场景一：蒸馏一个人的决策风格

  你崇拜某个人的思维模式——可能是 Paul Graham 的写作方法论、Naval 的决策框架、或者你公司里那个总能一眼看出架构问题的架构师。你想在做相关决策时有一个"虚拟顾问"可以参考。

  跟 Claude Code 说：

  ```
  帮我蒸馏「Paul Graham」→ persona skill
  ```

  SkillAlchemy 会派出 4-5 个研究 Agent，分别从他公开发表的文章、访谈、决策案例中提取思维模式、表达风格和价值观冲突。最终输出一个 100 行左右的 Persona Skill，你可以随时 `/paul-graham` 问「这个创业想法值不值得做」。

#### 场景二：把一套方法论变成可执行 SOP

  你读了一本好书，比如《思考，快与慢》，里面有很多实用的决策框架——但每次做决策时你不可能去翻书。你想把这些框架变成一个 Agent 可以按步骤执行的 SOP。

  ```
  蒸馏「思考快与慢中的决策框架」→ 方法论 skill
  ```

  LEAP 的输出格式是：前置条件 → 执行步骤 → 分支判断 → 失败处理。每一步都有可验证的证据。最终产出的 Skill 不是"建议你考虑系统1和系统2"，而是"当问题涉及概率判断时，先列出基准概率，再逐一调整，每一步标注调整依据"。

#### 场景三：融合两个已有 Skill 创造新能力

  你已经装了「第一性原理分析」skill 和「费曼学习法」skill，各自用起来都不错。但你想让 Agent 在解释复杂概念时先用第一性原理拆到底、再用费曼的方式讲出来——两个 skill 的衔接点在哪你不太确定。

  ```
  融合「第一性原理」和「费曼学习法」→ 深度理解引擎
  ```

  SkillAlchemy 的融合管线会自动检索两个 Skill 的完整内容，找到它们的输入输出接口，生成一个新的 Skill。新 Skill 不是两个旧 Skill 的简单拼接——它会识别出"拆解完成后需要做一次难度评估，根据评估结果选择不同深度的解释策略"这种只有组合后才会出现的中间步骤。

### 进阶技巧

  **技巧一：用 SkillsBench 跑分验证。** SkillAlchemy 生成的 Skill 质量可以量化验证。去 [SkillsBench](https://github.com/benchflow-ai/skillsbench) 把生成的 Skill 丢进 84 个任务里跑一圈，看看通过率。如果不够高，换 deep 模式重新蒸馏一遍——deep 模式会强制双审核，质量通常比 standard 高 5-10 个百分点。

  **技巧二：混合路线——先蒸馏缺失的 Skill，再融合。** 假如你想把「TDD」和「你团队的代码规范」融合成一个 Skill，但"你团队的代码规范"还没有对应的 Skill。SkillAlchemy 支持先蒸馏缺失的那部分：`蒸馏「我们团队的代码规范文档」→ 方法论 skill`，然后再融合。不需要手动补齐任何环节。

### 常见踩坑

  **子 Skill 找不到。** 最常遇到的问题。SkillAlchemy 首次运行时会检查 Lens 和 LEAP 是否已安装——如果没装，它会提示你安装。确保三个 Skill（SkillAlchemy + Lens + LEAP）都在 `~/.claude/skills/` 下。

  **蒸馏人物时资料太少。** 项目的 README 明确写了边界：不适合资料稀缺的人物复刻——三条新闻不足以召唤一个完整的人格。蒸馏人物前先确认是否有足够的公开资料（至少 10 篇以上长文或等量的访谈/演讲内容）。资料不足时考虑改为蒸馏方法论——方法是普适的，资料门槛低得多。

  **deep 模式耗时超出预期。** deep 模式跑 6-8 个 Agent，实际耗时可能在 30-40 分钟（取决于网络和模型响应速度）。如果中间某个 Agent 超时，不要从头重跑——SkillAlchemy 的中间产物都在 `output/` 目录下，可以让它从中断点继续。

| 技能 | 安装方式 | 核心特点 | 适用场景 | 热度 |
|------|---------|---------|---------|------|
| **SkillAlchemy**（今日推荐）| `npx skills add` | 多 Agent 并行研究 → 自动编译 SKILL.md，支持人物/方法论/技能融合三种模式 | 有想法但不知道怎么写成 Skill | ⭐ 69 |
| Anthropic skill-creator | `/plugin install skill-creator@claude-plugins-official` | 交互式问答 → 生成模板骨架，需要用户自己填充内容 | 知道要写什么，需要一个好模板 | ⭐ 官方 |
| Vercel `npx skills init` | `npx skills init` | 生成空白 SKILL.md 模板 | 完全从零手写 | ⭐ 27k（skills CLI） |

  SkillAlchemy 和 skill-creator 不是竞争关系，是上下游关系。如果你已经清楚知道 Skill 的每一步应该做什么，skill-creator 的交互式模板更快。如果你只有一个模糊的方向、不知道具体怎么落地，SkillAlchemy 的研究型流程更合适。两者的组合用法：先用 SkillAlchemy 生成初版 → 再用 skill-creator 打磨和评估。

## 今日小结

  SkillAlchemy 是为那些「有想法但缺翻译层」的人准备的。你不需要学习 SKILL.md 的最佳实践、不需要研究 frontmatter 的字段语义、不需要参考别人的 Skill 来模仿格式——你只需要描述你想要什么，剩下的由 5 个 Agent 并行研究、自动检索参照、编译验证。SkillsBench 的数据表明它自动生成的 Skill 质量已经跑赢了人工精选的，这背后的原因很简单：多 Agent 并行查阅的信息广度，远超一个人手动搜文档。

  安装到能用只需要 3 条命令（SkillAlchemy + Lens + LEAP），每条都是 `npx skills add` 一行搞定。花 2 分钟装好，以后每次有新想法直接说「帮我蒸馏 XXX」就行了。
