# caveman — Token 消耗直降 65%，让你的 AI 编程助手学会少说废话

  Claude Code 每次回复都在烧钱。一个 React 重渲染的解释用 69 个 token，其中 70% 是"您可能遇到了一个常见问题"、"我建议您可以尝试以下方案"这类礼貌性废话。caveman 一刀砍掉这些——同一个技术答案，19 个 token 说完，代码和命令原封不动。82,448 stars，MIT 协议，40 位贡献者，16 个版本，最近更新在 7 月 3 日。

### 它解决什么问题

  用 Claude Code 写代码的人都会遇到三个瓶颈：

  1. **Token 消耗太快**。Claude 的输出风格默认是"友好助教"模式——每个回答都有开场白、解释、建议和收尾。这些礼貌性包装在技术对话中只占 token、不提供信息。每月 $200 的订阅，有相当一部分烧在了"这是个好问题！让我来帮你分析一下"上。
  2. **响应速度被废话拖慢**。模型输出是逐 token 生成的，多出来的每一个"另外值得一提的是"都在增加你的等待时间。对于"改这个 bug"这种高频操作，你不需要一篇论文，你需要一行 fix。
  3. **上下文窗口被无效内容占据**。长对话中，每轮的多余文字累积起来，会提前撑爆上下文窗口，迫使你更频繁地 `/clear` 重置会话。

  caveman 解决这三个问题的思路很直接：让模型闭嘴，但不是让它变蠢。它只压缩模型的"嘴"——输出风格——不碰"脑"——推理能力。代码、命令、错误信息保持原样，被砍掉的只有连接词、客套话和过度解释。

  一篇 2026 年 3 月的论文 [*Brevity Constraints Reverse Performance Hierarchies in Language Models*](https://arxiv.org/abs/2604.00025) 测试了 31 个模型，发现限制大模型输出简洁后，某些基准测试的准确率反而提升了约 26 个百分点。少说废话有时不仅更便宜，还更正确。

### 核心原理

  caveman 本质上是一个 SKILL.md 文件加上几个 Hook 脚本。架构分四层：

  **第一层：SKILL.md 指令**。安装后写入 agent 的技能目录，核心指令一句话：去掉填充词、保留实质内容、用短句——但绝不修改代码块、CLI 命令或错误信息。技能描述本身约 100 tokens，匹配触发后加载完整指令。

  **第二层：SessionStart Hook**。在 Claude Code 上，安装脚本会自动写入一个 SessionStart hook——每次新会话启动时写入一个标记文件，让 Claude 从第一条消息起就进入 caveman 模式，无需手动输入 `/caveman`。Codex 和 Gemini 上同理。

  **第三层：六个压缩等级**。`/caveman lite`（去填充词）→ `full`（默认，电报式）→ `ultra`（只留关键词）→ `wenyan`（文言文，中文场景下 token 效率最高）。随时切换。

  **第四层：配套工具链**。`/caveman-stats` 读取会话日志统计实际节省量、写入 statusline 实时显示。`/caveman-commit` 生成 ≤50 字符的 Conventional Commit 信息。`/caveman-review` 用 `L42: 🔴 bug: user null. Add guard.` 格式做极简 PR 审查。

### 安装配置

  一条命令，自动检测机器上安装的所有 AI agent 并分别安装：

```bash
# macOS / Linux / WSL / Git Bash
curl -fsSL https://raw.githubusercontent.com/JuliusBrussee/caveman/main/install.sh | bash
```

```powershell
# Windows / PowerShell 5.1+
irm https://raw.githubusercontent.com/JuliusBrussee/caveman/main/install.ps1 | iex
```

  约 30 秒完成。需要 Node ≥18。自动跳过未安装的 agent，可安全重复运行。

  也可以只给特定 agent 安装：

```bash
# 仅 Claude Code（插件方式）
claude plugin marketplace add JuliusBrussee/caveman
claude plugin install caveman@caveman

# 仅 Gemini CLI
gemini extensions install https://github.com/JuliusBrussee/caveman

# 通过 skills 注册表安装给任意 agent
npx skills add JuliusBrussee/caveman -a cursor
```

  安装后自动生效——Claude Code、Codex 和 Gemini 上从第一条消息起就是 caveman 模式。如需关闭，说"normal mode"即可。

### 使用指南

#### 场景一：日常编码——每条回复都省 token

  安装 caveman 后，所有对话自动进入 `full` 模式。以修复一个 React 重渲染 bug 为例：

  **安装前**（69 tokens）：

  > The reason your React component is re-rendering is likely because you're creating a new object reference on each render cycle. When you pass an inline object as a prop, React's shallow comparison sees it as a different object every time, which triggers a re-render. I'd recommend using useMemo to memoize the object.

  **安装后**（19 tokens）：

  > New object ref each render. Inline object prop = new ref = re-render. Wrap in `useMemo`.

  代码建议完全一致，token 减少 72%。

#### 场景二：PR 审查——极简评论，精准定位

  用 `/caveman-review` 审查 PR 时，每条评论压缩为一行：

  ```
  L42: 🔴 bug: user null on first render. Add optional chaining.
  L87: 🟡 risk: no debounce on search input. Add 300ms.
  L156: 🔵 nit: var → const. Not reassigned.
  ```

  每个问题用严重度前缀标记，带行号和一句话修复建议。比传统 PR 审查节省约 60% 的输出 token。

#### 场景三：压缩 CLAUDE.md——省一次，省永远

  `/caveman-compress` 会重写你的 CLAUDE.md 等记忆文件，去掉冗余措辞，保留所有路径、命令和技术细节。官方测试数据显示平均压缩 46%：

  | 文件 | 原始 | 压缩后 | 节省 |
  |------|------|--------|------|
  | claude-md-preferences.md | 706 | 285 | 59.6% |
  | project-notes.md | 1145 | 535 | 53.3% |
  | claude-md-project.md | 1122 | 636 | 43.3% |

  输入 token 的节省是持续的——每个后续会话加载这个文件时都会省 token，不是一次性的。

#### 场景四：查看实际收益——用数据说话

  `/caveman-stats` 命令读取你的会话日志，计算实际节省的 token 数和等值美元金额。在 Claude Code 上还会自动写入 statusline，显示 `[CAVEMAN] ⛏ 12.4k` 这样的终身节省数。加上 `--share` 参数可以生成一条可发推的简洁总结。

### 进阶技巧

  **文言文模式——中文用户的最佳压缩比**。`/caveman wenyan` 切换到文言文输出。对中文用户来说，文言文的 token 效率是所有模式中最高的——同样的信息用更少的字符承载。文言文模式是唯一会"翻译"语言的模式，其余模式只压缩风格不改变语言。

  **组合 JuliusBrussee/skills 全家桶**。一条命令安装五个兄弟技能：

  ```bash
  npx skills@latest add JuliusBrussee/skills
  ```

  - `grill-me`：在动手写代码前，agent 先拷问你的方案设计
  - `interface-kit`：构建好看、快速、无障碍的 UI
  - `junior-to-senior`：对抗性审查——初级代码进，高级代码出
  - `loop-factory`：规格驱动的任务循环（inbox → active → archive）

  caveman 搭配这些技能后，不仅能省 token，还能在压缩的基础上保证输出质量。

### 常见踩坑

  **Net-negative 风险**。caveman 只压缩输出 token，不碰输入和推理 token。而且技能本身每轮会增加约 1-1.5k 的输入 token 开销。如果你的 agent 本身回复已经很简洁（比如你已经在 CLAUDE.md 中要求了简短回答），caveman 可能反而让总 token 消耗增加。官方在 `docs/HONEST-NUMBERS.md` 中诚实记录了这一点。建议用 `/caveman-stats` 观察一两周再决定是否长期使用。

  **关键信息可能被过度压缩**。在 `ultra` 模式下，模型有时会把重要的上下文信息一并砍掉。遇到复杂架构决策或安全相关讨论时，临时切回 normal mode 更安全。

  **非英语场景下压缩率会变化**。caveman 保持你的对话语言不变——用中文提问，它用压缩的中文回答。但压缩效果在中文这类信息密度本就较高的语言中不如英文显著。

## 今日小结

  caveman 适合所有重度使用 AI 编程助手的开发者——如果你每天和 Claude Code 对话超过 2 小时，安装它只需要 30 秒，回报是每条回复都更短、更快、更省钱。65% 的输出 token 减少意味着同等订阅费下能做更多轮对话，长会话撑得更久才需要 `/clear`。它不是银弹——在已经很简洁的场景下可能 net-negative，对安全关键讨论建议临时切回正常模式。但整体来看，这是当前 AI Agent Skill 生态中投资回报比最高的 30 秒安装之一。
