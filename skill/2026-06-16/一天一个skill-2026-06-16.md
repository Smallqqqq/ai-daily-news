# agents-cli — 让你的 AI 编程助手学会构建企业级 AI Agent

  你有没有试过让 Claude Code 帮你"写一个能自动回复客服邮件的 AI Agent"？它可能会写出一段不错的 Python 代码，但你很快会发现：评估体系怎么搭？部署到哪里？怎么接入可观测性？怎么管理 Agent 的生命周期？这些问题，一般的编程 agent 根本不知道怎么处理。Google 开源的 agents-cli 就是来解决这件事的——它给你的 Claude Code（或 Gemini CLI、Codex）装上一套"Agent 工厂"技能，从脚手架到评估到部署，一条龙搞定。

### 它解决什么问题

  AI 编程 agent（Claude Code、Codex、Gemini CLI）是写代码的高手，但在"构建 AI Agent"这条赛道上，它们普遍缺课：

  - **不会搭 Agent 脚手架**：ADK（Agent Development Kit）的 agent/tool/orchestration 该怎么组织，模型只会瞎猜
  - **不会做评估**：Agent 好不好用，不是"看起来能跑"就行。需要 evalset、LLM-as-judge、trajectory scoring 等专业手段
  - **不会部署上线**：代码写完了，怎么推到 Cloud Run 还是 GKE？CI/CD 怎么配？密钥怎么管理？
  - **不会注册和监控**：企业级 Agent 平台需要注册、配置可观测性，这些完全是编程 agent 的知识盲区

  agents-cli 用 7 个 Skill 把上述每个环节都变成了你的编程 agent 的内置知识。它不是替代 Claude Code，而是给 Claude Code 装上"Agent 工程专家"这个角色。

### 核心原理

  agents-cli 的设计分两层：

  1. **CLI 层**：`agents-cli` 是一个 Python CLI 工具（通过 `uvx` 运行），提供 scaffold、eval、deploy、publish 等命令。它可以直接在终端里用，也可以被编程 agent 调用。
  2. **Skill 层**：7 个 SKILL.md 文件，每个覆盖一个子领域。安装后，你的编程 agent 会在启动时加载这些 Skill 的描述（约 100 tokens），当你的任务匹配某个领域时自动激活完整的 Skill 内容。

  以 `google-agents-cli-scaffold` 为例：当你说"帮我创建一个 Agent 项目"，Claude Code 会激活这个 Skill，然后按照 ADK 的最佳实践生成项目结构——agent 定义、tool 注册、orchestration 逻辑，全部按照 Google 的推荐模式来。

  7 个 Skill 的分工：

  | Skill | 教会编程 agent 什么 |
  |-------|-------------------|
  | `google-agents-cli-workflow` | 完整开发流程、代码保护规则、模型选择策略 |
  | `google-agents-cli-adk-code` | ADK Python API——agent、tool、orchestration、callback、state |
  | `google-agents-cli-scaffold` | 项目脚手架——`create`、`enhance`、`upgrade` |
  | `google-agents-cli-eval` | 评估方法论——metrics、evalset、LLM-as-judge、trajectory scoring |
  | `google-agents-cli-deploy` | 部署——Agent Runtime、Cloud Run、GKE、CI/CD、secrets |
  | `google-agents-cli-publish` | Gemini Enterprise 平台注册 |
  | `google-agents-cli-observability` | 可观测性——Cloud Trace、日志、第三方集成 |

### 安装配置

  **前提条件**：Python 3.11+、[uv](https://docs.astral.sh/uv/getting-started/installation/)、Node.js。

  **方式一：完整安装（推荐，CLI + Skill 一起装）**

  ```bash
  uvx google-agents-cli setup
  ```

  这条命令会同时安装 `agents-cli` 命令行工具和所有 7 个 Skill 到你的编程 agent 的技能目录。

  **方式二：只装 Skill（如果你不需要 CLI）**

  ```bash
  npx skills add google/agents-cli
  ```

  安装完成后重启 Claude Code（或你用的编程 agent），Skill 就会自动生效。不需要额外配置。

  **本地开发不需要 Google Cloud**：只要有一个 [AI Studio API Key](https://aistudio.google.com/apikey)，就可以在本地完成 scaffold → run → eval 的完整流程。只有部署到云上时才需要 Google Cloud 账号。

### 使用指南

  #### 场景一：从零构建一个客服 Agent

  你是一家 SaaS 公司的后端工程师，老板说"搞个 AI Agent 自动处理用户退款请求"。你对 Agent 开发不熟，但你有 Claude Code。

  直接对 Claude Code 说：

  > 用 agents-cli 帮我创建一个退款处理 Agent，要求能理解用户意图、查询订单状态、执行退款或升级人工。

  Claude Code 会自动激活 `google-agents-cli-scaffold` 和 `google-agents-cli-adk-code` 两个 Skill，然后：
  1. 运行 `agents-cli scaffold refund-agent` 创建项目骨架
  2. 按照 ADK 规范定义 Agent、注册 tool（查订单、退款的 API 调用）
  3. 配置 orchestration 逻辑（意图识别 → 订单查询 → 退款执行 / 人工升级）
  4. 生成评估用例

  你不需要知道 ADK 的 agent 怎么定义、tool 怎么注册——Skill 里已经编码了这些知识。

  #### 场景二：给现有 Agent 加上评估体系

  你已经有一个跑了一段时间的 Agent 项目，但改代码时经常"修好一个 bug 引入三个新问题"——因为你没有自动化评估。

  对 Claude Code 说：

  > 用 agents-cli 给我的 Agent 项目加上评估体系，用 LLM-as-judge 方式。

  Claude Code 激活 `google-agents-cli-eval` Skill，然后：
  1. 分析你的 Agent 的功能边界，生成 evalset（覆盖正常路径 + 边界 case）
  2. 配置 LLM-as-judge 打分标准（准确性、完整性、安全性等维度）
  3. 集成 `agents-cli eval run` 到你的开发流程中
  4. 每次改代码后自动跑评估，生成对比报告

  #### 场景三：把本地 Agent 部署到生产环境

  你的 Agent 在本地跑得很好，现在要上线。但你对 Cloud Run 不熟，不知道 Dockerfile 怎么写、密钥怎么管理、CI/CD 怎么配。

  对 Claude Code 说：

  > 用 agents-cli 把这个 Agent 部署到 Google Cloud，配置自动扩缩容和 CI/CD。

  Claude Code 激活 `google-agents-cli-deploy` Skill，然后：
  1. 分析项目结构，自动生成 Dockerfile 和 Cloud Run 配置
  2. 配置 Secret Manager 管理 API 密钥
  3. 用 `agents-cli infra cicd` 搭建 CI/CD 流水线（GitHub Actions → Cloud Build → Cloud Run）
  4. 执行 `agents-cli deploy` 完成首次部署
  5. 配置 Cloud Trace 和日志导出，确保上线后可观测

### 进阶技巧

  **组合技：agents-cli + addyosmani/agent-skills**。agents-cli 负责 Agent 工程（脚手架、评估、部署），addyosmani/agent-skills 负责通用软件工程（spec、code review、testing）。两个一起装，你的编程 agent 就同时具备"写代码"和"建 Agent"两种专业能力。

  ```bash
  npx skills add google/agents-cli
  npx skills add addyosmani/agent-skills
  ```

  **独立使用 CLI**：即使不用编程 agent，`agents-cli` 的 CLI 也可以直接在终端使用。比如你想快速创建一个 Agent 项目骨架：

  ```bash
  agents-cli scaffold my-agent
  agents-cli eval run
  agents-cli deploy
  ```

  不需要打开任何 AI 工具，纯命令行操作。

### 常见踩坑

  **Windows 用户注意 uv 安装**：`uvx` 命令依赖 `uv` 包管理器。Windows 上推荐用 PowerShell 安装：

  ```powershell
  powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
  ```

  安装后重启终端，确保 `uv` 在 PATH 中。

  **AI Studio API Key 的权限范围**：本地开发时用 AI Studio API Key 就够了，但这个 Key 不能用于部署。部署到 Google Cloud 需要先执行 `agents-cli login` 完成 GCP 认证。

  **Skill 不生效？** 安装后必须重启 Claude Code（或你的编程 agent），因为 Skill 是在启动时扫描加载的。如果重启后仍然不生效，检查 Skill 是否正确安装到了对应目录：

  ```bash
  ls ~/.claude/skills/google-agents-cli-*/
  ```

## 今日小结

  今天的 agents-cli 适合两类人：一是正在探索 AI Agent 开发的工程师，想快速把 idea 变成可评估、可部署的 Agent；二是已经在用 Claude Code 日常写代码，但每次涉及 Agent 工程就卡壳的开发者。安装只要一条命令，10 分钟内就能跑通"脚手架→评估→部署"的完整流程。它不是替代你的编程 agent，而是给它装上"建 Agent"这项专业技能——你用自然语言描述需求，它按照 Google 的工程标准交付结果。
