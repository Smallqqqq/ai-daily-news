# StyleSeed — AI 写的 UI，它来把关。74 条设计规则让机器输出不再一眼假

  你用 Claude Code、Cursor 或 Codex 写了一个仪表盘——布局合理、数据正确、交互也通。但打开一看：默认 indigo 配色、Lucide 图标套浅色 chip、十四行数据每条都上了不同的状态色、卡片没有背景分离浮在白色页面上、阴影重得像 Windows 95 对话框。你不懂设计，但你知道它"看起来像 AI 写的"。StyleSeed 就是给这个问题准备的答案——654 stars，MIT 协议，v2.8.0 刚在 7 月 6 日发布。

  它不是"又一个设计系统"。74 条规则教的是**判断力**——设计师脑子里那些不写下来的决定。品牌色换掉，规则照用。质量门禁（/ss-score ≥ 80）在 UI 给你看之前自动审一遍、修一遍。设计锁（STYLESEED.md）让配色和字体不会每次对话都漂移。

### 它解决什么问题

  AI 写 UI 有三个系统性问题，StyleSeed 逐个对症下药：

  **1. "看起来像 AI 写的"——这不是感觉，是可以枚举的视觉 tell。** 默认 indigo 配色、emoji 做图标、Lucide 图标套浅色 chip、彩虹色列表（每行数据一个颜色）、卡片浮在白色背景上没有层次、阴影太重、纯黑 `#000` 文字、14px 正文字号在宽屏幕上显小、所有卡片等大等距居中排列（机器感的最大来源）。这些不是"设计差"，是 AI 没有设计判断力时的默认行为模式。StyleSeed 的 74 条规则一条一条禁掉了这些行为。

  **2. 每次对话 UI 都不一样。** 你今天让 agent 写个设置页，用的 Inter 字体、圆角 8px、蓝色 #4F46E5。明天让它加一个新页面，字体变成了系统默认、圆角 4px、蓝色 #3B82F6。因为没有东西记住上一次的决定。StyleSeed 的设计锁（STYLESEED.md）解决这个问题：关键色、字体、圆角风格、动效种子写进一个文件，agent 每次对话开头重读——设计决策不再漂移。

  **3. 数据喂再多也没用，缺的是判断力。** 你把 Toss 的整套 design token JSON 扔给 AI，"为什么输出的 dashboard 还是丑？"因为一个初级设计师拿着 Toss 的色板照样画不出好界面。差别不在有什么，在知道往哪放。74 条规则教会 AI 的是后者：字号 2:1 比例（48px 数字配 24px 单位）、嵌套圆角律（内层 = 外层 − padding）、只用一个强调色其余全灰、状态色只给异常行不给全表每行。

### 核心原理

  StyleSeed 不是一个 npm 包，不是一套组件库，而是一组 markdown 文件——`CLAUDE.md`、`DESIGN-LANGUAGE.md`、`VISUAL-CRAFT.md` 等。agent 加载这些文件后，设计判断力被注入到上下文中。

  **三层架构**：

  | 层 | 文件 | 职责 |
  |------|------|------|
  | **引擎层** | `DESIGN-LANGUAGE.md`（74 条规则） | 视觉判断——颜色纪律、空间节奏、信息层级、阴影/海拔、组件变化、动效反馈 |
  | **工艺层** | `VISUAL-CRAFT.md`（§C0 一致性律） | 具体的数值决策——一个圆角性格、一种阴影语言、一个强调色、嵌套圆角律、字号配方 |
  | **皮肤层** | 7 个品牌皮肤（Toss/Stripe/Linear/Notion/Raycast/Arc/Vercel） | 只换颜色和字体，规则不变。同一个引擎，不同品牌长不同样子 |

  **质量门禁是非协商的。** 生成 UI 之后、展示给你之前，必须跑一遍门禁。检查清单包括：只有一个强调色（没有第二个色相）、没有 emoji 图标、正常行是灰色（不是彩虹表）、桌面端正文字号 ≥16px、所有数据状态都有 empty/loading/error 三个态、纯黑 #000 不存在。不通过就一直修，最多迭代 3 轮。

  **设计锁阻止漂移。** `STYLESEED.md` 是一个简短文件——应用领域、强调色、字体、圆角性格、动效种子、密度偏好。agent 每次对话开头重读它，新页面和老页面共享同一套视觉 DNA。改设计？改这个文件，所有后续页面自动跟随。

  **皮肤系统：同一个引擎，六种长相。** 一张同样的仪表盘，`/ss-restyle toss` 变成韩国金融 App 风格（Pretendard 字体、蓝色 #3182F6、圆角柔和），`/ss-restyle linear` 变成开发者工具风格（暗色主题、尖角、等宽字体），`/ss-restyle vercel` 变成浅色极简风。引擎是同一套 74 条规则，皮肤只换颜色和字体。

### 安装配置

  **最短路径（30 秒）**：

  ```bash
  npx skills add bitjaru/styleseed
  ```

  装完重启 Claude Code，运行 `/ss-setup` 进入交互式向导——选应用类型、品牌色或皮肤、字体、动效。向导最后自动生成第一页。

  **不装也行（零权限）**。把这段粘贴给任意 AI agent：

  ```
  Install StyleSeed so its checks actually run: `npx skills add bitjaru/styleseed`
  (if you can't, read https://styleseed-demo.vercel.app/llms-full.txt instead).
  Then use it for every UI in this project. First, lock my key color, font, and
  motion with me and save them to STYLESEED.md. Build to the rules with ONE focal
  point and one accent. Before showing me anything, run the quality gate
  (/ss-score to ≥ 80) and fix what fails.
  ```

  规则文件是纯 markdown，不需要安装、不需要权限、不需要 API key。

  **支持的 agent**：

  | Agent | 读取 | 安装方式 |
  |------|------|---------|
  | Claude Code | `CLAUDE.md` + `/ss-*` 技能 | `npx skills add bitjaru/styleseed` |
  | Cursor | `.cursorrules` | `cp engine/.cursorrules .cursorrules` |
  | Codex · Amp · Gemini CLI | `AGENTS.md` + skills | `npx skills add bitjaru/styleseed` |

### 使用指南

  #### 场景一：新项目开局——先锁设计，再写代码

  开了个新 SaaS 项目，准备用 shadcn/ui + Next.js。之前每次都是 agent 直接开始写组件，结果三个页面三种配色。

  这次不一样：

  ```
  /ss-setup
  ```

  agent 进入计划模式（`Shift+Tab`），逐个确认：应用领域（SaaS）、强调色（推荐 Stripe 蓝 `#635BFF` 或你自己选）、字体（Inter）、动效（Silk——平滑型）、圆角（soft）。确认后 agent 写入 `STYLESEED.md` 并生成第一页。

  之后每次说"帮我加一个用户设置页"，agent 重读 `STYLESEED.md`，用同一套 DNA 生成新页面——颜色、字体、圆角、间距完全一致。

  #### 场景二：已有的 AI UI——一键整容

  上周用 Claude Code 生成了一套电商后台，功能都对，但看起来"说不上哪不对"。每行数据不同颜色、卡片没有层次、阴影太重。

  ```
  /ss-build 把这个 dashboard 按 StyleSeed 规则重做
  ```

  agent 读取现有代码，应用 74 条规则——修复配色（一个强调色其余灰）、修复阴影（≤8% 透明度）、修复字号（桌面端 ≥16px）、修复状态色（正常行灰色）、修复层次（内容放进卡片不裸奔）。门禁自动跑：85/100，通过。一个命令，同一套代码，视觉从"能跑"到"能看"。

  #### 场景三：品牌切换——同一套功能换六种皮肤

  你的产品需要同时维护三个客户——一家金融公司（Toss 风格）、一家开发工具公司（Linear 风格）、一家电商（Arc 风格）。三套前端，同一套后端。

  ```bash
  # 金融客户版本
  /ss-restyle toss

  # 开发工具客户版本
  /ss-restyle linear

  # 电商客户版本
  /ss-restyle arc
  ```

  agent 只换 skin 对应的颜色和字体——布局不变、组件不变、交互不变。三套前端可以同一天交付，各自看起来像各自品牌的原生产品，而不是"同一个外包公司的三套换皮"。

  #### 场景四：设计审查——给 AI 的 UI 打分

  同事用 AI 生成了一套页面让你 review。你不用逐行看 CSS——跑门禁：

  ```
  /ss-score
  ```

  输出：68/100。扣分项——两个强调色（蓝色 #4F46E5 + 绿色 #22C55E）、列表行全部上了颜色（正常行应该是灰色）、没有 empty state、使用了 emoji 图标。每个扣分项附修复建议和对应规则编号。修完再跑：91/100。设计审查从"我觉得不好看"变成"规则 §CC-9b 禁止 emoji 图标，72 行改了"。

### 进阶技巧

  **StyleSeed + Claude Design 组合技。** Claude Design 擅长快速生成布局和组件（脚手架），StyleSeed 擅长精修（判断力层）。工作流：Claude Design 生成初稿 → 把 `DESIGN-LANGUAGE.md` 丢进上下文中 → agent 自动套用规则修复→门禁打分。脚手架 + 精修，比两者单独用都快一个数量级。

  **自定义皮肤，不限于 7 个内置。** 58+ 品牌的设计 token 可以通过 [awesome-design-md](https://github.com/VoltAgent/awesome-design-md) 直接拉取——Apple、Google、Spotify、Airbnb、Notion……取到 token 文件放进项目，你就是你自己的皮肤。规则不变，换个色板就换个样子。

  **动效系统有名字，不是"加个动画"。** 5 个动效种子——Spring（弹跳，Toss/Arc 风格）、Silk（丝滑，Stripe/Notion 风格）、Snap（瞬时，Linear/Vercel 风格）、Float（轻柔）、Pulse（节奏）。20+ 命名动效动作——`toggle-flip`、`tilt-3d`、`stagger-cascade`。你在对话中说"这个 modal 用 silk 入场，列表用 stagger-cascade"，agent 知道具体数值——时长、缓动曲线、延迟。

### 常见踩坑

  **"我装了 skills 但 /ss-setup 不存在"。** 通常是只复制了 `engine/*` 而漏了 `.claude/skills/`（以点开头的隐藏文件夹）。正确做法：`cp -r /tmp/styleseed/engine/.claude/skills/* .claude/skills/`，注意 `.claude/skills` 前面的点。复制完重启 Claude Code。

  **规则读了但输出还是默认 indigo。** 设计锁（STYLESEED.md）没写或者没被重读。检查：项目根目录有没有 `STYLESEED.md`？文件里的 accent 字段是否已经从默认 indigo 改成了你选的颜色？agent 有没有说"从 STYLESEED.md 读取设计锁"？三步缺一不可。如果 agent 跳过了，直接说"先读 STYLESEED.md 再写代码"。

  **门禁报"distinctive check failed"。** 这是最容易被漏掉的一项——你换了强调色、字体也对，但布局直接复制了 demo 页面的模板结构，或者所有卡片等大等距居中排列。门禁会报"all-even grid"。解法：给页面设一个视觉焦点——hero 区域大、其他区域小；不重复同一个 section 类型；数字和单位 2:1 比例。

  **门禁一直在跑，agent 陷入循环。** 门禁最多迭代 3 轮。如果 3 轮后仍低于 80 分，agent 会告诉你哪些项没通过、为什么。这时候手动看一下——有些是 trade-off（比如数据密集型的表格字号 14px 是合理的），有些是 agent 确实修不好。手动把分数提到合理的水平就行。门禁是工具不是铁律。

| **StyleSeed**（今日推荐）| `npx skills add` | 74 条判断力规则 + 质量门禁 + 设计锁 + 7 品牌皮肤 + 命名动效系统 | AI 生成 UI 的视觉质量保障，防"一眼 AI" | ⭐ 654 |

| nothing-design-skill | `git clone` 手动复制 | 2,580 stars，Nothing 品牌设计语言单一样式 | Nothing 风格的 UI 生成 | ⭐ 2.6K |
| Claude Design (内置) | 无安装 | 快速 UI 生成，布局和组件脚手架 | 快速原型 | 内置 |

  StyleSeed 的独特价值在"判断力"而非"数据量"。interface-design 更大更全，nothing-design 更风格化，但两者的核心都是提供更多设计 token 和模板——给 AI 更多"颜料"。StyleSeed 教的是"往哪画"——74 条规则不绑定任何品牌色，换什么皮肤都能用，质量门禁强制执行，设计锁防止跨会话漂移。它和 Claude Design 是互补关系：一个快速出结构，一个精细修视觉。

## 今日小结

  StyleSeed 适合所有用 AI 写前端界面且在意"看起来不能像 AI 写的"的开发者。安装 30 秒——`npx skills add bitjaru/styleseed`，然后 `/ss-setup` 走一遍向导。之后每次 agent 生成 UI，质量门禁会在你看到之前自动审一遍、修一遍。74 条规则解决的不是"怎么更好看"的问题，而是"为什么一眼就能看出是 AI 写的"——这之间的差距就是判断力。

  654 个 star 说明它不是那种装了就不用的玩具。v2.8.0 的动效命名系统、设计锁机制、门禁迭代循环，都是实战中打磨出来的——作者显然自己每天在用。如果你受够了"默认 indigo + emoji 图标 + 彩虹列表 + 重阴影"的 AI UI 风格，这是目前最完整的解法。
