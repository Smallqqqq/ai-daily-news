# 一天一个 Skill · 2026-06-02

  在 Claude Code 里 `git add` → `git commit` → `git push` 这个三步舞每天要跳几十遍，而且 Claude 默认的提交行为经常带上 AI 署名水印、偶尔用 `--amend` 改写历史。今天这个 Skill 用一个 `/qc` 命令把这一切收拾干净——自动匹配你项目的提交规范、拒绝跳过 hooks、拒绝改写历史、拒绝 AI 水印。

---

## claude-code-qc — 一个 `/qc` 搞定 stage + commit + push，干净不越界

### 它解决什么问题

  在 Claude Code 里写完代码后，提交是一个高频但容易出问题的环节。Claude 默认行为有几个痛点：它会在 commit message 里加 `Co-Authored-By: Claude ...` 这样的 AI 署名水印；它有时会用 `--amend` 折叠修改；它习惯 `git add -A` 一把梭——万一把 `.env` 也加进去了就很尴尬。

  `claude-code-qc` 是一个 Claude Code Skill，安装后你只需要在对话里打 `/qc`，它就会按照你项目已有的提交规范，完成从 stage 到 push 的全流程。它不会跳过 pre-commit hooks、不会去掉 GPG 签名、不会改写历史、不会加 AI 水印、不会 force push——每一件事都踩在安全边界内。

### 核心原理

  这个 Skill 本质上是一个 Markdown 指令文件（`SKILL.md`），Claude Code 在收到 `/qc` 命令时加载它作为行为指令。它的工作流分六步：

  1. **检测 gh CLI**：检查 GitHub CLI 是否安装并已认证，没装就自动 bootstrap（支持 apt/brew/dnf/pacman/winget）
  2. **检测提交规范**：按优先级读取 `CLAUDE.md` → `CONTRIBUTING.md` → `.gitmessage` → 最近的 git log，自动匹配项目的提交风格（Conventional Commits / JIRA 前缀 / 自由格式）
  3. **按文件名 stage**：不用 `git add -A`，而是精确添加变更的文件
  4. **生成 Conventional Commit 信息**：祈使句、聚焦"为什么"而非"改了什么"
  5. **带 hooks + GPG 签名提交**：绝不跳过 `--no-verify` 或 `--no-gpg-sign`
  6. **push 到 origin**：`git push -u origin HEAD`

  最巧妙的点是它对提交规范的**自动适配**——不需要任何配置文件。如果你的 `CLAUDE.md` 里写了"提交信息要用 `[W1.A.0]` 格式"，它会从最近的 commit 中提取当前任务的 ID 填入。如果项目是标准的 Conventional Commits，它就生成 `feat(scope): summary` 格式。

### 安装配置

  **一行安装（全局生效）**：

```bash
mkdir -p ~/.claude/skills/qc && \
  curl -fsSL https://raw.githubusercontent.com/TimothyVang/claude-code-qc/main/SKILL.md \
  -o ~/.claude/skills/qc/SKILL.md
```

  重启 Claude Code 后 `/qc` 即可使用。

  **仅当前项目安装**（团队成员共享）：

```bash
mkdir -p .claude/skills/qc && \
  curl -fsSL https://raw.githubusercontent.com/TimothyVang/claude-code-qc/main/SKILL.md \
  -o .claude/skills/qc/SKILL.md && \
  git add .claude/skills/qc/SKILL.md
```

  **Windows 用户**：

```powershell
New-Item -ItemType Directory -Force "$env:USERPROFILE\.claude\skills\qc"
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/TimothyVang/claude-code-qc/main/SKILL.md" -OutFile "$env:USERPROFILE\.claude\skills\qc\SKILL.md"
```

### 使用指南

#### 场景一：日常功能分支提交（最常用）

  在 Claude Code 里写了几行代码后，直接输入：

```
/qc
```

  Claude 自动分析 diff、匹配提交规范、生成信息、提交并推送。典型交互：

```
You: /qc
Claude: 检测到你修改了 src/auth.ts（3行）—— 添加了 session 超时机制。
        生成信息: feat(auth): add 30-minute idle session timeout
        已 stage 1 个文件。确认提交+推送？
You: yes
Claude: ✓ 已提交, ✓ 已推送至 origin/feat/idle-timeout
```

#### 场景二：直接提交到主分支（个人项目快速流）

```
/qc main
```

  Skill 会自动切到 `main`/`master` 分支，stash 当前修改、提交、推送。如果 stash pop 遇到冲突会停下来让你处理，不会丢代码。适合自己说了算的小项目。

#### 场景三：带任务 ID 的企业项目

```
/qc
```

  如果你的 `CLAUDE.md` 要求提交信息带 `[W2.C.4]` 这种任务编号，Skill 会自动从最近的 commit 中提取：

```
Claude: 从最近提交中检测到活跃任务: [W2.C.4]
        生成信息: fix(graph): correct executor fanout reducer race [W2.C.4]
        已 stage 2 个文件。确认提交+推送？
```

### 进阶技巧

  **按项目自定义行为**：Skill 是纯 Markdown 指令，你可以直接编辑 `SKILL.md`。放在 `~/.claude/skills/qc/SKILL.md` 的是全局默认值，放在 `.claude/skills/qc/SKILL.md` 的会覆盖全局。比如你不想让它自动 push，把 Step 6 删掉就行。

  **搭配 `/ship` 使用**：这个 Skill 只负责 commit + push，不创建 PR。如果你需要一键 PR 创建，可以再装一个专门的 `/ship` Skill，形成 `/qc`（提交）+`/ship`（提 PR）的两步流水线。

### 常见踩坑

  **Q: 提示 `gh` 未安装，安装卡住了？**
  1. 手动安装：`winget install GitHub.cli`（Windows）或 `brew install gh`（macOS）
  2. 认证：`gh auth login --hostname github.com --git-protocol https --web`
  3. 之后再跑 `/qc`，Skill 检测到 `gh` 已就绪会自动跳过 bootstrap

  **Q: 用了 SSH Remote 还需要 `gh auth login` 吗？**
  SSH key 已经能搞定 `git push`，所以 Skill 不会阻塞。但它仍然会建议 `gh auth login --git-protocol ssh`，因为 `gh` 的其他功能（如 PR 命令）需要认证。

  **Q: 我的项目用 GitLab，能用吗？**
  提交+推送部分完全不受影响（host-agnostic）。`gh` bootstrap 部分只对 GitHub 有效，GitLab 用户用 `glab auth login` 配置好 credential helper 后 Skill 自动跳过 bootstrap。

  **Q: Skill 会动我的 `.gitconfig` 吗？**
  不会。它不会修改 `.gitconfig`、不会设置 committer 身份、不会改任何 git 全局配置。

### 同类技能对比

| 技能 | 安装方式 | 核心特点 | 安全边界 | 热度 |
|------|---------|---------|---------|------|
| **claude-code-qc**（今日推荐）| 一行 curl 安装 | 自动适配提交规范 + bootstrap gh + 拒绝不安全操作 | 不跳过 hooks/签名/amend/force | ⭐ 新发布 |
| Claude Code 默认提交 | 内置 | `git add -A` + 自动生成信息 | 带 AI 署名水印、偶尔 amend | — |
| commit-push-pr Skill | 插件安装 | commit + push + 自动创建 GitLab MR | 偏 GitLab 生态 | — |
| 手动 git 命令 | 无需安装 | 完全控制 | 取决于个人习惯 | — |

  claude-code-qc 的差异化在于"自动化但不越界"——它帮你省掉重复劳动，但每次提交前都会让你确认，绝不替你决定 force push 或跳过 hooks。对比内置的 Claude 提交行为，它的提交信息更干净（无水印）、行为更可控（不 amend）。

## 今日小结

  这个 Skill 适合所有在 Claude Code 里频繁提交的开发者。安装 30 秒，之后每次提交从三步变成一句话。它的价值不在功能多，而在边界清晰——帮你做该做的事，绝不碰不该碰的东西。如果你一天要提交 20 次，装完这个每天能省至少 5 分钟和一堆心智负担。
