# SkillOpt — 像训练神经网络一样训练你的 Agent Skill

  SkillOpt 是微软研究院推出的 Agent Skill 自动优化框架。它不碰模型权重，而是把 SKILL.md 本身当作可训练的外部状态，通过" rollout → 反思 → 编辑 → 验证门控 → 部署"这个完整训练循环，让你的 Skill 在真实基准上稳定提升。上线不到一个月，GitHub 已破 5000 stars，52 场对比测试全胜。

---

### 它解决什么问题

  手写 Agent Skill 有个尴尬的现实：你很难知道一个 SKILL.md 到底是"够用了"还是"还能更好"。改一句话可能让 Agent 在某个场景变聪明，也可能破坏另一个场景的表现。就算你愿意花时间反复调试，改几轮后也不知道该往哪个方向改了。

  SkillOpt 把这个问题变成了一个有数学保证的优化过程。它引入了一个独立的"优化器模型"来观察 Agent 的执行轨迹，提出对 SKILL.md 的增/删/改操作，然后只在验证集上严格提升时才保留这个修改。整个过程有学习率预算、拒绝编辑缓冲区、轮次级慢更新，跟训练神经网络的优化器一样严谨。

### 核心原理

  SkillOpt 的核心洞察：**把 Skill 文档当作 Agent 的"外部可训练权重"**。既然我们不能改模型参数（frozen model），那为什么不像训练参数一样训练文档？

  整个优化循环分五步：

1. **Rollout（执行采样）**：目标模型带着当前 Skill 执行一批任务，记录每次轨迹和评分。
2. **Reflect（反思分析）**：优化器模型分析成功和失败的轨迹，找出可复用的模式——哪些指令奏效了，哪些造成了混乱。
3. **Edit（约束编辑）**：优化器提出对 SKILL.md 的 add / delete / replace 操作。关键设计是 **文本学习率（textual learning rate）**——限制每次编辑的字符数预算，防止一次改太多导致不稳定。
4. **Gate（验证门控）**：候选 Skill 必须在留出的验证集上严格优于当前版本才被接受。不提升就丢弃，绝不退步。
5. **Slow/Meta Update（慢更新）**：每个 epoch 结束时，用累积的高质量编辑做一次更谨慎的更新，类似于深度学习的动量或学习率衰减。

  最终导出的是一个紧凑的 `best_skill.md`（通常 300-2000 tokens），部署时零额外推理开销——目标模型只需读取这份文件。

### 安装配置

```bash
# 一键安装（需要 Python 3.10+）
pip install skillopt
```

  配置 API 密钥（支持 OpenAI / Azure / Claude / Qwen / MiniMax）：

```bash
# 设置目标模型和优化器模型的 API 密钥
export OPENAI_API_KEY="sk-..."
export ANTHROPIC_API_KEY="sk-ant-..."

# 或者写入 .env 文件
echo 'OPENAI_API_KEY=sk-...' > .env
echo 'ANTHROPIC_API_KEY=sk-ant-...' >> .env
```

  克隆官方仓库获取示例和基准测试：

```bash
git clone https://github.com/microsoft/SkillOpt.git
cd SkillOpt
pip install -e ".[dev]"
```

### 使用指南

#### 场景一：优化一个已有的 Claude Code Skill

  你写了一个 `/code-review` Skill，但不确定它的指令是否最优。用 SkillOpt 自动优化：

```bash
# 准备基线 Skill 和评测数据
skillopt train \
  --skill ./my-skills/code-review/SKILL.md \
  --benchmark code_review \
  --target-model gpt-5.5 \
  --optimizer-model claude-sonnet-4-6 \
  --epochs 5 \
  --batch-size 8 \
  --output ./optimized-skills/
```

  跑完后 `./optimized-skills/best_skill.md` 就是你优化好的 Skill，直接替换原来的 SKILL.md 即可。每次 epoch 结束后会打印验证分数，你能看到 Skill 性能的逐步提升曲线。

#### 场景二：零-shot 创建新 Skill 并优化到可用

  你有一个新任务场景（比如"自动生成 API 文档"），手头没有任何 Skill。让优化器从零生成：

```bash
# 从空模板开始，让优化器自己摸索
skillopt train \
  --skill ./templates/empty_skill.md \
  --benchmark api_doc_generation \
  --target-model gpt-5.5 \
  --optimizer-model claude-opus-4-7 \
  --epochs 10 \
  --learning-rate medium \
  --output ./generated-skills/
```

  10 个 epoch 后你会得到一个从随机探索中收敛出来的专业 Skill。这个过程对人来说可能很费时（你需要反复试各种措辞），但优化器 30 分钟就能跑完。

#### 场景三：跨模型迁移 Skill

  SkillOpt 优化出的 Skill 有很好的迁移性。你可能是用 Claude 做主力开发，但团队里有人用 Codex：

```bash
# 先针对 Codex 训练
skillopt train \
  --skill ./baseline-skill.md \
  --benchmark spreadsheet_bench \
  --target-model gpt-5.5 \
  --harness codex \
  --output ./codex-optimized/

# 直接把 codex 训练的 best_skill.md 部署到 Claude Code
# 论文数据显示跨 harness 迁移仍然能提升 +31.8 分
```

  不需要重新训练，直接复制 best_skill.md 到 Claude Code 的 skills 目录即可。

### 进阶技巧

  **技巧一：用弱模型优化强模型的 Skill。** 这是一个反直觉但很实用的策略。你可以用便宜的 GPT-5.4-nano 作为优化器，训练给 Claude Opus 用的 Skill。论文数据显示这种"弱优化器"配置在 SpreadsheetBench 上仍然比基线高 10.4 分。适合预算有限但想批量优化 Skill 的团队。

  **技巧二：设置"拒绝编辑缓冲区"。** 如果某个 epoch 反复提出同样的无效编辑，SkillOpt 会把它丢进缓冲区，后续 epoch 自动跳过。这相当于深度学习的 gradient clipping——防止优化器在某些方向上浪费算力。命令行加 `--rejected-edit-buffer-size 5` 即可启用。

### 常见踩坑

  **评测数据质量决定优化上限。** SkillOpt 的验证门控完全依赖你的评测集。如果评测集本身不能准确反映"好"和"坏"，优化器会向错误方向收敛。建议至少准备 50 个有明确评分标准的测试用例，并且留出 20% 做验证集。官方仓库的 `benchmarks/` 目录下有 6 个内置基准可直接使用。

  **文本学习率设太高会震荡。** 如果你发现 Best Skill 的验证分数在 epoch 间剧烈波动，大概率是 textual learning rate 太高了。降低到 `--learning-rate low`，或者手动限制 `--max-edit-chars 200`。类似于训练神经网络时学习率设太高导致 loss 不收敛。

## 今日小结

  SkillOpt 适合已经在维护 Agent Skill 的团队——你有一组 SKILL.md，知道它们"差不多能用"，但没法量化它们到底有多好，更不知道怎么系统性地改进。花 30 分钟配好评测集、跑一轮训练，你就能拿到一个验证分数明确优于手写版本的 Skill。对于还在手写和瞎调 Skill 的阶段，这是一个从"凭感觉"到"有数据"的质变。

  配置门槛不高：Python 3.10+ 加一个 API 密钥即可。优化是离线进行的，部署时只是替换 SKILL.md 文件，不增加任何运行时开销。
