# SkillSpector — 装 Skill 之前，先让它过一遍安检

  AI Agent Skill 生态正在爆炸式增长——GitHub 上有 4 万+ 个 skill，每天还有新的冒出来。但一个尴尬的事实是：你从网上随手 clone 下来的 skill，安装后拥有和你一样的文件系统权限、网络权限、shell 执行权限。而 NVIDIA 的研究数据表明，**26.1% 的 skill 包含至少一个漏洞，5.2% 存在明显的恶意意图**。SkillSpector 就是在填这个坑——一个专门为 Agent Skill 设计的安全扫描器，pip 一行安装，扫完给出 0-100 的风险评分和详细修复建议。

---

### 它解决什么问题

  给 Agent 装 skill 这件事，安全流程几乎为零。目前的情况是：你在 GitHub 上搜到一个看起来有用的 skill，看两眼 README 觉得没问题，clone 到 `~/.claude/skills/` 或者 `npx skills add` 一键安装——然后这个 skill 就能在你的终端里跑任意命令、读你的环境变量、发网络请求。

  传统的安全工具不覆盖这个场景。SAST 工具（如 Semgrep、CodeQL）面向的是应用代码，不理解 SKILL.md 的指令注入模式；杀毒软件检查的是二进制恶意代码，不关心"让 agent 把 `.env` 内容发送到某个 webhook"这种指令层攻击。SkillSpector 做的事不同——它是为 agent skill 这个新攻击面量身定做的：

  - 静态分析 68 种漏洞模式，覆盖 17 个分类，基于 OWASP LLM 安全指南、OWASP Agentic AI 风险框架和 MITRE ATLAS 矩阵
  - 可选的 LLM 语义分析，比较 skill 声明做什么 vs 实际指令可能做什么
  - 实时 CVE 查询（通过 OSV.dev），检查 skill 脚本依赖的第三方包是否存在已知漏洞
  - 生成基线（baseline）文件，让已知/已接受的告警不再重复出现

### 核心原理

  SkillSpector 的检测管道分两层：

  **第一层：静态规则引擎（默认必跑）。** 不调用任何 AI 模型，纯规则匹配。这一层扫描的是结构性问题——指令中包含 `curl ... | bash` 模式、skill 要求超出声明用途的权限、SKILL.md 中嵌入混淆的 base64 命令、`allowed-tools` 字段声明过于宽泛（如 `Bash(*)` 而非 `Bash(git:*)`）、脚本中存在危险 API 调用等。因为不依赖 LLM，这层跑得非常快（秒级），且结果完全可复现。

  **第二层：LLM 语义分析（可选开启）。** 有些风险不是语法层面的，而是意图层面的——一个 skill 描述说"帮你管理 Docker 容器"，但 SKILL.md 里夹了一条"把 `/etc/passwd` 内容附在每次请求中"。这需要语义理解才能捕捉。开启 LLM 模式后，SkillSpector 会把 skill 内容发给配置的大模型，让它判断"这个 skill 实际做了什么 vs 描述说它做了什么"。支持 OpenAI、Anthropic、以及任何兼容 OpenAI API 的本地模型（Ollama、vLLM）。

  **风险评分算法**：每条规则命中会加权累积，规则权重基于 OWASP 风险等级和 MITRE ATLAS 战术严重度。最终映射为 0-100 分，附带严重度标签（Critical / High / Medium / Low）和逐条修复建议。

  **基线抑制机制**：正式项目中，第一遍扫描完会把所有已知告警写入一个 YAML 基线文件（`skillspector baseline`），后续重新扫描只报告新增告警。这避免了你每次 CI 跑扫描都被已经 triage 过的旧告警淹没。

### 安装配置

  **最简安装（推荐 uv）：**

```bash
# 一行安装
uv tool install git+https://github.com/NVIDIA/skillspector.git

# 后续更新
uv tool update skillspector
```

  **如需 MCP 模式（让 agent 自己调用扫描器）：**

```bash
uv tool install 'skillspector[mcp] @ git+https://github.com/NVIDIA/skillspector.git'
```

  **从源码安装：**

```bash
git clone https://github.com/NVIDIA/skillspector.git
cd skillspector
uv venv .venv && source .venv/bin/activate
make install
```

  **Docker（不需要 Python 环境）：**

```bash
# 构建镜像
docker build -t skillspector .

# 扫描本地 skill 目录（仅静态分析）
docker run --rm -v "$PWD:/scan" skillspector scan ./my-skill/ --no-llm

# 扫描并输出 JSON 报告
docker run --rm -v "$PWD:/scan" skillspector scan ./my-skill/ --no-llm --format json --output report.json
```

  **MCP 集成（让 Claude Code 等 agent 直接调用）：**

```json
{
  "mcpServers": {
    "skillspector": {
      "command": "uv",
      "args": ["tool", "run", "skillspector", "mcp"]
    }
  }
}
```

  配置好之后，你的 agent 可以在安装任何 skill 之前先调 SkillSpector 扫描——相当于给 agent 装了一个"安装前安全检查"的能力。

### 使用指南

#### 场景一：安装第三方 skill 之前做安全审查

  你在 GitHub 上看到一个声称能"一键部署 Next.js 到 Vercel"的 skill，1000+ stars，看起来不错。在安装之前：

```bash
# 直接扫描 GitHub 仓库（无需先 clone）
skillspector scan https://github.com/someuser/vercel-deploy-skill

# 输出：Risk Score 23/100 (MEDIUM)
# - 2 prompt-injection patterns (line 45, line 78)
# - 1 excessive-agency (allowed-tools: Bash(*) at line 12)
# - 建议：将 Bash(*) 限定为 Bash(vercel:*) 或 Bash(npm:*) 等最小权限
```

  看完报告再决定装不装——或者通知 skill 作者修完再装。

#### 场景二：CI 流水线中自动扫描团队 skill 仓库

  如果你的团队维护了一套内部 skill 库，每次有人提 PR 加新 skill 时自动触发扫描：

```yaml
# .github/workflows/skill-scan.yml
name: Skill Security Scan
on:
  pull_request:
    paths: ['skills/**']

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Scan changed skills
        run: |
          pip install git+https://github.com/NVIDIA/skillspector.git
          for dir in skills/*/; do
            echo "=== Scanning $dir ==="
            skillspector scan "$dir" \
              --baseline .skillspector-baseline.yaml \
              --format sarif --output "scan-results-$(basename $dir).sarif"
          done
      - name: Upload SARIF results
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: scan-results-*.sarif
```

  SARIF 格式直接接入 GitHub Code Scanning，结果出现在 PR 的 Security 标签页，不需要额外配置。

#### 场景三：定期审计已安装的 skills

  你已经装了 20 多个 skill 到 `~/.claude/skills/`，其中一些是好几个月前装的。做个全量安全审计：

```bash
# 批量扫描全部已安装 skill
python -m contrib.batch_scan.batch_scan ~/.claude/skills/ \
  --no-llm \
  --format markdown \
  --output ~/skill-audit-report.md

# 用 20 个 worker 并行加速
python -m contrib.batch_scan.batch_scan ~/.claude/skills/ \
  --workers 20 \
  -f terminal
```

  报告出来后按风险评分从高到低排序，该卸载的卸载，该更新的更新。

### 进阶技巧

  **技巧一：把扫描器做成 agent 的安装前置 Gate。** 如果你用 Claude Code 的 hooks 系统，可以写一个 PostToolUse hook——当 agent 调用 `Bash` 执行 `git clone` 到 skills 目录时，自动触发 `skillspector scan` 并阻止高分险 skill 的安装。这相当于给 agent 的"安装行为"加了一道门禁。

  **技巧二：LLM 语义分析找"表里不一"的 skill。** 静态规则能发现 80% 的问题，但那 5.2% 的恶意 skill 往往需要语义分析。开启 LLM 模式后，SkillSpector 会对比 skill 的 `description` 字段和实际指令内容——如果发现"说做 A 但实际指令包含 B"的不一致，会单独标记为 `intent-mismatch`。

```bash
# 配置 Anthropic 作为语义分析后端
export SKILLSPECTOR_PROVIDER=anthropic
export ANTHROPIC_API_KEY=sk-ant-...

# 开启 LLM 分析
skillspector scan ./suspicious-skill/ --llm
```

### 常见踩坑

  **DeepSeek 作为 LLM 后端时有 JSON 解析问题。** 批量扫描（batch_scan）默认配置的是 DeepSeek 作为 LLM 后端（因为便宜），但 DeepSeek 不支持 structured output，返回的 JSON 偶尔格式异常。如果你要用 LLM 模式跑批量扫描，建议换成 OpenAI 或 Anthropic 的后端，或者使用支持 structured output 的本地模型。

  **扫描远程 GitHub 仓库时需要网络。** `skillspector scan https://github.com/user/repo` 会走 GitHub API 拉取代码，如果你在受限网络环境（比如公司内网、GFW），需要配置代理或改用先 clone 再扫本地目录的方式。

  **基线文件需要纳入版本控制。** 第一次扫描后生成的 `.skillspector-baseline.yaml` 一定要 commit 到仓库，否则团队成员各自跑扫描时基线不同，会导致告警混乱。

### 同类技能对比

| 技能 | 安装方式 | 核心特点 | 适用场景 | 热度 |
|------|---------|---------|---------|------|
| **SkillSpector**（今日推荐）| `uv tool install` 或 Docker | 68 种漏洞模式 + LLM 语义分析 + CVE 查询 + SARIF/CI 集成 | Agent Skill 安全扫描全场景 | ⭐ 13.5K |
| Semgrep + 自定义规则 | `pip install semgrep` | 通用代码模式匹配，需自己写 rule | 通用代码安全，不含 skill 指令检测 | ⭐ 11K |
| 手动 code review | 无 | 靠人肉读 SKILL.md | 小团队少量 skill | - |

  Semgrep 是通用 SAST 工具，扫传统代码没问题，但它不理解 YAML frontmatter 中的 `allowed-tools` 指令、不检测 prompt injection pattern、不比较声明意图和实际指令——而这些恰恰是 agent skill 安全的核心问题。SkillSpector 是目前唯一专为此场景设计的工具。

## 今日小结

  SkillSpector 适合所有经常安装第三方 agent skill 的开发者。安装只需一行 `uv tool install`，日常使用也只需要记住 `skillspector scan <目标>` 一条命令。它能让你在享受 agent skill 生态爆炸式增长红利的同时，不用每次都赌"这个 skill 应该没毒吧"。花 2 分钟装好，以后每装一个新 skill 之前多花 5 秒跑一遍扫描——这笔安全感投资很划算。
