# AG Kit — 给 AI 编程助手装上团队协作与持久记忆

  AG Kit 是一个开箱即用的 AI Agent 开发模板包，把 20 个专家 Agent、45 个领域技能和 13 个开发工作流打包在一起，一条命令注入项目。它的核心卖点不是"技能多"，而是 Coordinator Mode（多 Agent 协调）、Persistent Memory（跨会话记忆）和 Context Compression（上下文压缩）三个生产级设计模式，官方宣称能减少 13%~33% 的 token 消耗。

---

### 它解决什么问题

  用 AI 编程助手写代码的都知道几个痛点：每次新会话都要重新解释项目规范、复杂任务让 AI 在一个上下文里硬啃导致 token 爆炸、多个子任务串行执行慢得让人想关掉终端去喝杯咖啡。AG Kit 就是针对这三件事设计的——用一套模板化的 Agent + Skill + Workflow 体系，让 AI 助手按"正规军"的章法干活，而不是每次都从零开始。

### 核心原理

  AG Kit 是一套注入到你现有编程助手（Claude Code、Cursor、Codex 等）里的配置模板。安装后会在项目根目录生成 `.agents/` 文件夹，里面有三层结构：

  **Agent 层（20 个）**：每个 Agent 是一个专精某个领域的人格——前端专家、后端专家、安全审计、PM、QA 等。当你输入"给登录接口加 JWT 认证"，系统自动识别并路由到 `@security-auditor` + `@backend-specialist`。

  **Skill 层（45 个）**：按需加载的领域知识模块。每个 Skill 都有条件加载规则（conditional loading），只在相关任务触发时才注入上下文，避免闲置指令撑爆上下文窗口。

  **Workflow 层（13 个）**：可交互的斜杠命令（`/plan`、`/debug`、`/deploy` 等），把常见的开发流程固化为结构化步骤。

  三条架构主线：
  - **Coordinator Mode**：多 Agent 并行执行 + 结果合成，替代昂贵的串行重试
  - **Persistent Memory**：4 类型记忆引擎（`MEMORY.md`），跨会话保留项目约定
  - **Context Compression**：自动摘要和微压缩，防止长会话上下文退化

### 安装配置

```bash
# 推荐方式：按需执行（无需全局安装）
npx @vudovn/ag-kit init

# 或全局安装
npm install -g @vudovn/ag-kit
ag-kit init
```

  多仓库共享同一套配置（符号链接）：

```bash
# macOS / Linux
mkdir -p ~/.ag-kit && cd ~/.ag-kit
npx @vudovn/ag-kit init
ln -s ~/.ag-kit/.agents .agents

# Windows PowerShell（管理员）
New-Item -ItemType SymbolicLink -Path ".agents" -Target "$env:USERPROFILE\.ag-kit\.agents"
```

  安装后重启 AI 编程助手即可。推荐的 `.gitignore` 策略：不把 `.agents/` 加入 `.gitignore`（否则编辑器无法索引斜杠命令），而是加到 `.git/info/exclude`。

### 使用指南

#### 场景一：零配置自动路由——不用手动指挥 Agent

  输入"给登录接口加上 JWT 认证"，系统自动识别需要安全审计和后端专家两个 Agent，分别加载对应的 Skill 并返回结果。不需要你手动指定用哪个 Agent，不需要写 `/coordinate`。

#### 场景二：结构化开发流程——用斜杠命令管住 AI 的节奏

  输入 `/plan` 生成实现计划 → `/build` 逐步实现 → `/test` 生成和执行测试 → `/verify` 验证代码能跑通 → `/deploy` 发布。每个命令背后是预设的检查清单和验证步骤，AI 不会跳过关键环节。

#### 场景三：跨会话记忆——今天交代的事明天还记得

  用 `/remember` 把项目约定写入持久记忆。下次新开会话，AI 自动读取 `MEMORY.md`，不需要你重新解释"我们的 API 统一用 RESTful 风格，错误码格式是 XXX"。记忆引擎按 4 种类型（约定、决策、踩坑、偏好）索引，AI 知道什么时候该加载哪条。

### 进阶技巧

  **多仓库共享**：用符号链接把 `.agents/` 指向一个中心化的 AG Kit 目录，所有项目共享同一套 Agent 和 Skill 配置。更新一处，全局生效。

  **自定义 Agent**：AG Kit 的 Agent 模板是纯 Markdown 文件，直接编辑 `.agents/` 下的 Agent 定义就能定制。比如给 `backend-specialist` 加上你们团队的特定框架约定。

### 常见踩坑

  **`.gitignore` 配置陷阱**：如果把 `.agents/` 加入 `.gitignore`，Cursor/Windsurf 等 AI 原生编辑器的语言服务器将无法索引工作流文件，斜杠命令的自动补全会失效。正确做法是加到 `.git/info/exclude`。

  **符号链接权限**：Windows 上创建符号链接需要管理员权限或以开发者模式运行。如果 `New-Item -ItemType SymbolicLink` 报错，先开开发者模式（设置 → 更新和安全 → 开发者选项）。
