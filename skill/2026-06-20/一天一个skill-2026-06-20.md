# SkillAnything — 一个能自动生成 Agent Skill 的元技能

  手写一个 SKILL.md 往往需要几小时——研究 CLI 参数、设计触发词、写示例、调优 description。SkillAnything 把这个过程压缩到一句话：告诉它你要给什么工具生成技能（jq、Stripe API、pandas……），它自动跑完分析→设计→生成→评估→优化→打包的完整流水线，产出一套可直接安装到 Claude Code / Codex / OpenClaw 的技能包。

  如果说 SkillAlchemy 帮你把"想法"蒸馏成技能，那 SkillAnything 帮你把"已有工具"变成技能。二者互补，但 SkillAnything 的独特价值在于：你不需要手动整理工具文档、写使用示例——它自动检测目标类型，自动提取能力边界，自动生成测试用例，甚至自动优化 description 的触发准确率。

---

### 它解决什么问题

  给 Agent 写技能是件重体力活。比如你要让 Claude Code 精通 `jq`，你需要：

  1. 梳理 jq 的常用参数和语法
  2. 设计触发词和 description，让技能在该出现时出现、不该出现时不出现
  3. 写使用示例
  4. 反复测试 description 的触发准确率
  5. 如果还要兼容 Codex、OpenClaw，就得手动适配格式

  这套流程走下来，几个小时就没了。SkillAnything 把整个过程自动化：分析目标 → 设计技能架构 → 生成 SKILL.md → 生成测试用例 → 跑基准测试 → 优化 description → 打包多平台分发。你只需要一句 "Create a skill for jq"。

### 核心原理

  SkillAnything 本身是一个 Claude Code Skill（元技能），它的工作机制借鉴了 CLI-Anything 的 7 阶段流水线方法论，具体流程如下：

  **Phase 1 — Analyze（分析）**：自动检测目标类型。CLI 工具（通过 `which` + `--help` 解析）、REST API（抓取 OpenAPI/Swagger spec）、Python 库（pip 包信息）、工作流描述（自然语言步骤解析）、Web 服务（文档抓取），每种类型有不同的分析策略。

  **Phase 2 — Design（设计）**：根据分析结果选择技能架构模式——CLI 工具用 tool-augmentation 模式，API 用 endpoint-reference 模式，工作流用 workflow-orchestrator 模式。

  **Phase 3 — Implement（生成）**：生成完整的 SKILL.md + 辅助脚本 + references 文档，遵循 Agent Skills 开放标准。

  **Phase 4 — Test Plan（测试）**：自动生成 eval 用例——包括触发查询（验证技能是否在正确的场景下被激活）和功能断言。

  **Phase 5 — Evaluate（评估）**：在有无技能两种条件下跑基准测试，对比 pass rate。例如 jq 技能的基准测试：无技能时 42% 通过率，有技能后 87%。

  **Phase 6 — Optimize（优化）**：通过 train/test 循环优化 description 的触发准确率。目标是让技能在该出现时出现（高召回），不该出现时不出现（高精确）。例如 jq 技能优化后达到 18/20 的触发准确率。

  **Phase 7 — Package（打包）**：生成多平台分发包——Claude Code（`~/.claude/skills/`）、OpenClaw（`~/.openclaw/skills/`）、Codex（`~/.codex/skills/`）、以及通用 `.skill` zip 包。

### 安装配置

```bash
# Claude Code
git clone https://github.com/AgentSkillOS/SkillAnything.git ~/.claude/skills/skill-anything

# OpenClaw
git clone https://github.com/AgentSkillOS/SkillAnything.git ~/.openclaw/skills/skill-anything

# Codex
git clone https://github.com/AgentSkillOS/SkillAnything.git ~/.codex/skills/skill-anything
```

安装后在 Claude Code 中直接说人话即可触发：

```
> Create a skill for the httpie CLI tool
> Generate a multi-platform skill for the Stripe API
> Turn this data pipeline workflow into a skill
```

如需手动控制单个阶段（调试或自定义场景）：

```bash
# Phase 1: 分析目标
python -m scripts.analyze_target --target "jq" --output analysis.json

# Phase 3: 生成技能骨架
python -m scripts.init_skill my-skill --template cli --output ./out

# Phase 5: 跑评估
python -m scripts.run_eval --eval-set evals.json --skill-path ./out/my-skill

# Phase 6: 优化 description
python -m scripts.run_loop --eval-set trigger-evals.json --skill-path ./out/my-skill --model claude-sonnet-4-20250514

# Phase 7: 打包多平台
python -m scripts.package_multiplatform ./out/my-skill --platforms claude-code,openclaw,codex
```

配置文件 `config.yaml`：

```yaml
pipeline:
  auto_mode: true              # 全自动模式或交互模式
  skip_eval: false             # 快速原型可跳过评估阶段

platforms:
  enabled: [claude-code, openclaw, codex, generic]
  primary: claude-code

eval:
  max_optimization_iterations: 5  # description 优化最大迭代次数
  runs_per_query: 3               # 每个查询重复测试次数
```

### 使用指南

#### 场景一：为 CLI 工具生成技能

  你经常在 Agent 中用到 `ffmpeg`、`jq`、`httpie` 等命令行工具，但 Agent 对这些工具的用法一知半解，经常写出错误的参数组合。

  在 Claude Code 中说：

  ```
  > Create a skill for the ffmpeg CLI tool, focus on video transcoding
  ```

  SkillAnything 自动检测 ffmpeg 已安装，解析 `--help` 输出，提取常用参数和子命令，生成一个包含完整参数说明 + 使用示例 + 触发词的技能。评估结果显示有技能后 Agent 的 ffmpeg 命令正确率从 35% 提升到 82%。

#### 场景二：为 REST API 生成技能

  你频繁用到某个 REST API（如 Stripe Payment Intents、GitHub Issues API），每次都要翻文档确认端点路径和参数格式。

  ```
  > Generate a skill for the Stripe API, focus on payment_intents and customers
  ```

  SkillAnything 拉取 Stripe 的 OpenAPI spec，提取 payment_intents 和 customers 相关端点，生成包含认证配置 + 端点参考 + 常用操作的技能文件。之后 Agent 就能直接写出正确的 API 调用。

#### 场景三：将工作流固化为技能

  你有一套固定的数据处理流水线："从 Postgres 取数据 → 用 pandas 清洗 → 上传到 S3"。每次做都要重新描述步骤。

  ```
  > Turn this into a skill: fetch from Postgres, clean with pandas, upload to S3
  ```

  SkillAnything 检测到这是一个 3 步工作流，使用 workflow-orchestrator 模式生成技能，包含每个步骤的输入输出约定、错误处理策略和依赖关系图。

#### 场景四：批量生成团队技能库

  团队有一批内部工具和 API 需要让 Agent 掌握。手动写 20 个 SKILL.md 不现实。

  写一个脚本循环调用 SkillAnything 的单阶段模式：

  ```bash
  for tool in jq httpie ffmpeg gh aws; do
    python -m scripts.analyze_target --target "$tool" --output "analysis_$tool.json"
    python -m scripts.init_skill "$tool-skill" --template cli --output "./out/$tool"
    python -m scripts.package_multiplatform "./out/$tool" --platforms claude-code
  done
  ```

  半小时内产出 20 个技能包，安装到团队共享的 skills 目录即可。

### 进阶技巧

  **跳过评估加速迭代**：原型阶段在 `config.yaml` 中设 `skip_eval: true`，跳过 Phase 4-6，直接从分析到打包，1 分钟产出一个可用的技能初版。等基本可用后再开评估做精细优化。

  **与 SkillAlchemy 组合使用**：SkillAnything 擅长把"已有工具"变成技能，SkillAlchemy 擅长把"模糊想法"蒸馏成技能。实际工作流可以先用 SkillAlchemy 理清"我要做什么"，再用 SkillAnything 把涉及的每个工具固化为可复用的技能模块。

### 常见踩坑

  **CLI 工具未安装时分析阶段会失败**：SkillAnything 通过 `which` + `--help` 解析 CLI 工具的用法。目标工具必须先安装并加入 PATH。如果只是分析第三方工具而不需要本地执行，可以考虑以 "workflow" 类型提交描述文本，跳过 CLI 自动检测。

  **生成的技能 description 不一定完美**：Phase 6 的优化循环依赖 LLM 打分，偶尔会出现过拟合——description 太窄导致只匹配测试用例但不匹配实际使用场景。建议用自然语言做几次实际触发测试，必要时手动微调 description。

  **Python 3.9+ 是硬性要求**：脚本使用了较新的 Python 特性，如果系统默认 Python 版本过低，需要先升级。

## 今日小结

  SkillAnything 适合所有需要频繁为 Agent 扩展能力的开发者。如果你手里有一堆 CLI 工具、内部 API、标准工作流想让 Agent 掌握，手动写 20 个 SKILL.md 是噩梦，用 SkillAnything 几分钟就能产出初版。它的自动化流水线虽然不能完全替代人工审校（description 优化仍有局限性），但能把最耗时的"从零梳理"阶段压缩到几乎为零。花 5 分钟 clone 安装，省下的可能是每周数小时的技能编写时间。
