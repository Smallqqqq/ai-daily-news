# agent-skill-manager (asm) — 你的所有 AI Agent 技能，一个工具全管住

  你在 Claude Code 里装了几个 skill，Codex 那边又有几个，Cursor 里还散落着一些——每个工具把技能藏在各自的隐藏目录里，版本混乱、重复安装、安全状况一无所知。asm 就是来解决这个问题的：一个命令，管理你所有 AI agent 的 skills。

---

### 它解决什么问题

  用多个 AI 编程 agent 的人都有一个共同的痛点：skills 散落在各处。Claude Code 的 skills 在 `~/.claude/skills/`，Codex 的在 `~/.codex/skills/`，OpenClaw 的在 `~/.openclaw/skills/`，还有项目级的 `.claude/skills/`……同一个 skill 可能被装了三次，你不知道哪个版本最新，也不知道哪些已经过期。

  更麻烦的是安全——你从 GitHub clone 一个 skill 仓库，复制到 skills 目录，整个过程没有任何安全检查。Snyk 2026 年 2 月的 ToxicSkills 报告显示，社区中有 37% 的 Claude Code skill 包含恶意代码（base64 混淆、`curl | bash` 后门等）。

  asm 用一个统一的 TUI 和 CLI，把 18 种 AI agent 的 skills 全部管起来：统一搜索、一键安装、自动去重、安全扫描。

### 核心原理

  asm 启动时扫描所有已配置 agent 的 skills 目录（global + project），建立统一的技能索引。安装技能时，它从 GitHub 克隆仓库，解析 SKILL.md 的 YAML frontmatter，验证结构完整性，运行安全扫描（检测 shell 执行、网络访问、凭据暴露、混淆代码等危险模式），然后放入目标 agent 的 skills 目录。

  内置的 TUI 基于 Ink（React for terminal）+ `@opentui/core`（Zig 原生的终端 UI 库），提供键盘导航、搜索过滤、详情查看等交互能力。CLI 模式支持 `--json` 输出，方便脚本集成。

  技能注册表（ASM Registry）是一个独立的 GitHub 仓库，存储社区发布技能的 manifest 文件。安装时 asm 从注册表解析技能名称 → 定位仓库 → 检出精确 commit → 安装到目标 agent，全程锁定版本。

### 安装配置

```bash
# npm 全局安装（推荐，Node.js ≥ 18）
npm install -g agent-skill-manager

# 或一键安装脚本
curl -sSL https://raw.githubusercontent.com/luongnv89/agent-skill-manager/main/install.sh | bash
```

安装完成后直接运行 `asm` 进入 TUI 界面，或使用 CLI 命令：

```bash
asm                          # 进入交互式 TUI
asm list                     # 列出所有已安装 skill
asm search "code review"     # 搜索 skill
asm stats                    # 查看统计仪表盘
```

首次运行会自动创建配置文件 `~/.config/agent-skill-manager/config.json`，包含 18 个预配置 agent provider，默认启用 Claude Code、Codex、OpenClaw 和通用 Agents 路径，其他按需开启。

### 使用指南

#### 场景一：快速安装一个社区 skill

  在 GitHub 上发现一个好用的 skill，不需要手动 clone、复制、验证——一条命令搞定：

```bash
# 从 GitHub 安装（自动验证 + 安全扫描）
asm install github:anthropics/skills --path skills/skill-creator

# 从注册表安装（无需记住 GitHub URL）
asm install code-review

# 安装到指定 agent
asm install github:user/my-skill -p claude
```

  asm 会自动克隆仓库、解析 SKILL.md、运行安全扫描、展示元数据预览，确认后安装到目标 agent 的 skills 目录。

#### 场景二：检查所有 agent 的技能状况

  在 Claude Code 和 Codex 之间切换工作时，想知道哪些 skill 重复了、哪些需要更新：

```bash
asm list --group-by tool      # 按 agent 分组显示
asm audit                     # 检测重复 skill
asm audit security --all      # 对所有已安装 skill 做安全审计
```

  TUI 中按 `a` 键触发重复审计，按 `/` 键搜索过滤。重复的 skill 可以一键清理。

#### 场景三：开发并发布自己的 skill

  从零开始写一个 skill，本地测试迭代，然后发布到注册表让其他人安装：

```bash
# 1. 脚手架
asm init my-awesome-skill -p claude

# 2. 编辑 SKILL.md，然后符号链接到 agent 目录实时测试
asm link ./my-awesome-skill -p claude

# 3. 安全审计
asm audit security my-awesome-skill

# 4. 质量评分（静态 rubric，检查结构/清晰度/安全性等）
asm eval ./my-awesome-skill

# 5. 推到 GitHub 后，发布到注册表
asm publish ./my-awesome-skill
```

  `asm link` 创建的是符号链接，编辑源文件后 agent 立即可见，不需要反复复制。`asm publish` 会自动生成 manifest、fork 注册表仓库、开启 PR。

### 进阶技巧

  **技能套装（Bundle）一键安装**：asm 内置了预定义的技能组合，适合常见工作流：

```bash
asm bundle list --predefined   # 查看预定义套装
asm bundle install frontend-dev  # 一键安装前端开发全套技能
```

  你也可以把自己的常用技能打包成 bundle JSON 文件，分享给团队：

```bash
asm bundle create my-workflow
asm bundle export my-workflow ./my-workflow.json
```

  **自定义 agent 路径**：如果你用的 agent 不在内置 18 种里，可以通过 `customPaths` 配置添加任意 `SKILL.md` 目录，asm 会自动扫描纳入管理。

### 常见踩坑

  **多个 asm 二进制冲突**：如果你之前用其他包管理器装过 asm，`PATH` 上可能有多个 `asm` 可执行文件，导致升级后被旧版本"遮蔽"。运行 `asm doctor` 诊断，删除旧安装即可。

  **Windows 兼容性**：asm 基于 Node.js，理论跨平台。但 TUI（Ink + @opentui/core）在 Windows 终端可能有渲染问题。如果 TUI 显示异常，使用 CLI 模式（带 `--json` 标志）同样可以完成所有操作。

  **私有仓库安装**：需要用 SSH 传输：
  ```bash
  asm install github:user/private-skill -t ssh
  ```

## 今日小结

  agent-skill-manager 适合同时使用多个 AI 编程 agent 的开发者——特别是 Claude Code + Codex + Cursor 组合的用户。安装只需一条 npm 命令，5 分钟内就能把散落在各处的 skills 纳入统一管理。花半小时清理掉重复安装、做完安全审计之后，你对自己的 skills 资产会第一次有清晰的全局视野。它最核心的价值不是"又一个 skill"，而是"让你已有的 skills 不再是一团乱麻"。
