# Trail of Bits Skills — 把顶级安全审计公司的经验，装进你的 AI 编程助手

  你做代码审查的时候，关注的是逻辑对不对、命名好不好、有没有多余的 import。安全研究员看同一段代码，看到的是"这个反序列化入口没校验"、"那个常量比较不是 constant-time 的"、"这个依赖去年爆过 CVE"。两种视角的差距不是经验问题，是训练数据：你的 AI Agent 默认没有安全审计的思维模型。Trail of Bits 把自家团队做了十几年的安全审计方法论，打包成了 30+ 个 Claude Code 技能插件，一条命令装进你的终端。

### 它解决什么问题

  Trail of Bits 成立于 2012 年，是全球顶级的安全审计公司。他们的客户名单包括 DARPA、Google、Microsoft、Ethereum Foundation。安全审计这个行业有一个结构性矛盾：顶级专家的时间是不可扩展的。一个资深研究员一天只能审几百行关键代码，但每天有几十万行新代码被提交到各个仓库。

  这套技能集就是他们把"不可扩展的专家时间"变成"可复用的 AI 工作流"的产物。每个技能封装了一种安全分析模式：教会 Claude Code 用安全研究员的思维方式审视代码，而不是背 checklist。当你写 `@differential-review` 审查一个 PR，Claude 会分析 git diff 的上下文，理解变更的意图，然后判断"这个改动有没有引入新的攻击面"。它不是简单地 grep 危险函数。

### 核心原理

  Trail of Bits Skills 是一个 **Claude Code 插件市场**，不是单个技能文件。安装市场后，你可以按需安装其中任意一个插件（每个插件 = 一个技能），Claude 在你调用时加载对应的 SKILL.md。

  整个技能集的架构有三层：

  **第一层：安全思维模型（Code Auditing 类）**。这是最核心的一层，包含 16 个代码审查技能。每个技能不是"检查有没有 SQL 注入"，而是封装了一种分析方法论。比如 `differential-review` 教 Claude 用"变更-影响"模型审视 diff：改了一个函数的签名，它会自动追踪所有调用点，判断是否有调用点因为新签名而产生了不安全的默认值。`variant-analysis` 教会 Claude "找到一个漏洞后，在代码库中搜索所有类似模式"。这是人类安全研究员的核心技能，现在 AI 也会了。

  **第二层：专用分析工具（Verification / Malware / Mobile 类）**。这一层针对特定领域的安全问题。`constant-time-analysis` 检测加密代码中是否存在编译器引入的时序侧信道。这个问题极其隐蔽，人类审查几乎不可能发现。`yara-authoring` 教 Claude 编写 YARA 恶意软件检测规则，附带 linting 和 atom 分析。`zeroize-audit` 专门检测 C/C++/Rust 中密钥清零操作被编译器优化掉的问题。

  **第三层：安全工程基础设施（Development / Tooling 类）**。这一层是把安全实践嵌入日常开发流程的技能。`supply-chain-risk-auditor` 审计项目依赖的供应链风险。`static-analysis` 集成了 CodeQL、Semgrep 和 SARIF 解析。`insecure-defaults` 检测框架和库的默认不安全配置。

  技能之间可以组合使用——先用 `audit-context-building` 建立代码库的架构全景，再用 `static-analysis` 跑自动扫描，最后用 `differential-review` 逐条验证每个发现的真实性。

  **渐进式披露**是这套技能的关键设计。每个技能有 `SKILL.md`（核心指令）+ `references/`（按需加载的详细文档）。日常使用时 Claude 只加载核心指令，上下文占用极小。只有深入分析时才加载完整参考文档。

### 安装配置

  安装整个市场只需要一条命令：

```
/plugin marketplace add trailofbits/skills
```

  然后浏览并安装你需要的插件：

```
/plugin menu
```

  你会看到一个交互式菜单，列出所有 30+ 个插件。按 Enter 选中，按 Esc 退出。想装哪个装哪个，不需要全装。

  如果你用 Codex CLI：

```sh
codex plugin marketplace add trailofbits/skills
codex plugin list
codex plugin add <plugin-name>@trailofbits
```

  装完无需重启。下次在 Claude Code 中使用 `/static-analysis`、`/differential-review` 等命令即可调用对应技能。Claude 也会在相关上下文（比如你要求"审查这段代码的安全性"）时自动激活匹配的技能。

### 使用指南

#### 场景一：PR 审查不只是看代码风格——用 differential-review 做安全变更分析

  你提交了一个 PR，改了认证模块的 token 验证逻辑。往常的 code review 会关注变量命名、测试覆盖、是否遵循团队规范。

  输入 `/differential-review`，Claude 会加载这个技能的工作流：先分析 git diff 的变更范围，再追踪每个变更点对上下游的影响，然后用 Trail of Bits 的安全检查清单逐条对照。这个 token 的签名算法是什么？过期时间合理吗？有没有新增的时序攻击面？

  输出不是"第 42 行少了个分号"，而是"第 58 行的 token 比较操作不是 constant-time 的，攻击者可以通过时序分析逐字节猜测有效 token"。建议用 `crypto.timingSafeEqual` 替代 `===`。

#### 场景二：依赖了 200 个 npm 包，哪些是供应链炸弹——用 supply-chain-risk-auditor

  你的项目 `package.json` 里躺了 200 个依赖。你知道其中有些维护者可能已经失联，有些包的发布流程没有双因素认证，有些曾经被植入过挖矿代码。

  输入 `/supply-chain-risk-auditor`，Claude 会：拉取每个直接依赖的元数据（维护者活跃度、发布频率、CI 配置、安全策略文件）→ 分析依赖树的攻击面（哪些包有 `postinstall` 脚本？哪些包的 npm 发布 token 没有过期策略？）→ 按风险等级排列输出，高风险项附带具体的缓解建议（钉版本、替换包、fork 审计）。

#### 场景三：审计团队积累的 Semgrep 规则库在新语言上也能用——用 semgrep-rule-variant-creator

  你们团队用 Python 写了 50 条 Semgrep 自定义规则——检测硬编码密钥、不安全的反序列化、缺失的权限检查。现在新项目用 Go，这些规则全部失效。

  输入 `/semgrep-rule-variant-creator`，指定源语言（Python）和目标语言（Go），Claude 会逐条分析每条规则的语义——不是语法，是"这条规则检测的是'用户输入未经过滤即传入系统命令'"——然后在 Go 中找到等价的语法模式，生成 Go 版本的规则，并自动跑测试验证新规则能捕获相同的漏洞模式。

#### 场景四：智能合约上线前的最后一道关——用 building-secure-contracts

  你写了一个 Solidity 智能合约，准备部署到主网。这个合约管着几百万美元的资产，一行代码错了就是灾难。

  输入 `/building-secure-contracts`，Claude 会加载 6 条区块链的漏洞扫描器，检查重入攻击、整数溢出、访问控制缺陷、闪电贷攻击向量等。不是泛泛地"检查是否用了 SafeMath"，而是模拟攻击路径："如果攻击者在 `withdraw` 函数执行到一半时通过 fallback 函数重新进入 `withdraw`，能否重复提取资金？"

### 进阶技巧

  **组合技能做全栈安全审计**。这不是一个按钮就能跑完的。安全审计本身就是多阶段的工作。推荐流程：

  1. `/audit-context-building` — 先建立代码库的架构全景（入口点、数据流、信任边界）
  2. `/static-analysis` — 用 CodeQL + Semgrep 并行扫描，产出 SARIF 报告
  3. `/fp-check` — 对扫描结果做系统性误报验证，标注真阳性/假阳性
  4. `/differential-review` — 对标记为真阳性的代码段做深度上下文分析
  5. `/variant-analysis` — 每发现一个漏洞，在代码库中搜索所有类似模式

  这个流程串起来的时间取决于代码库大小，但核心价值在于：它把一次性的"AI 帮我看看有没有安全问题"变成了系统化的安全审计流水线。每一步的输出是下一步的输入，不会漏也不会重复。

  **把静态分析集成到 CI**。`static-analysis` 技能生成的 SARIF 报告可以被 GitHub Code Scanning 直接消费。在你的 GitHub Actions 里加一个 step，让 Claude Code 跑 `/static-analysis`，SARIF 结果上传到 GitHub，以后每个 PR 都会自动标注安全发现。

### 常见踩坑

  **不要一次装所有插件**。30+ 个技能全部启用会让 Claude 的语义匹配变慢（每个技能的 description 都要参与匹配计算）。建议只装你现在需要的 3-5 个。代码审查日常用 `differential-review`、`static-analysis`、`insecure-defaults`；做区块链的加装 `building-secure-contracts`；做 C/C++ 的加装 `c-review` 和 `zeroize-audit`。

  **某些技能需要外部工具**。`static-analysis` 依赖 CodeQL CLI 或 Semgrep 已安装在本地。`burpsuite-project-parser` 需要 Burp Suite 项目文件。安装前先看插件的 README，确认前置依赖就绪。大部分代码审查类技能（differential-review、insecure-defaults、variant-analysis）不需要外部工具，纯 Claude Code 即可运行。

  **CC-BY-SA-4.0 协议的注意事项**。如果你在企业环境中使用这些技能并做了修改，SA（ShareAlike）条款要求修改后的版本也必须以相同协议开源。但这不影响你使用技能产出的安全分析结果——你审计出来的漏洞报告是你自己的。
