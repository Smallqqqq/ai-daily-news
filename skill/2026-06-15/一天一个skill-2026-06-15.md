# context-mode — 你的上下文窗口，省着点用

  每次 MCP 工具调用、每次 `curl`、每次 Playwright 截图，都在往 Claude Code 的 200K 上下文窗口里灌原始数据。一个 Playwright snapshot 吃掉 56KB，20 个 GitHub Issue 吃掉 59KB，跑一轮测试套件又是几十 KB。30 分钟过后，40% 的上下文已经被原始数据塞满，Claude 开始"失忆"。context-mode 就是解决这个问题的——它在 Claude Code 和工具输出之间插入一个沙箱层，315KB 的原始数据压缩到 5.4KB，节省 98%。还带 SQLite 会话追踪，compaction 之后自动恢复工作状态，不用从头解释"我们刚才在干什么"。

---

### 它解决什么问题

  Claude Code 的上下文窗口是有限资源（200K tokens）。每次 MCP 工具调用、bash 命令输出、Read 文件内容，都会直接灌入上下文。表面上看这没问题——模型需要看到数据才能做决策。但实际上，大量原始数据是冗余的：一个 56KB 的 Playwright snapshot 里你可能只关心 3 个按钮的状态；59KB 的 GitHub Issue 列表里你可能只关心标题和标签。

  **实测数据（来自官方）：**
  - Playwright snapshot：56.2 KB → 299 B（节省 99%）
  - GitHub Issues（20 条）：58.9 KB → 1.1 KB（节省 98%）
  - 访问日志（500 条）：45.1 KB → 155 B（节省 100%）
  - Git log（153 次提交）：11.6 KB → 107 B（节省 99%）
  - 全会话累计：315 KB → 5.4 KB（节省 98%）
  - 会话可用时长：从 ~30 分钟延长到 ~3 小时

  另外它还解决了**会话中断后的恢复问题**：当 Claude Code 因为上下文耗尽而 compaction 或重启后，你需要重新解释一遍"我们在做什么"。context-mode 用 SQLite + FTS5 全文索引追踪每一次文件编辑、git 操作、任务状态和用户决策，compaction 后自动恢复——模型从你上次的 prompt 继续，不用你重复。

### 核心原理

  context-mode 不是用 LLM 做摘要（那样反而更费 token），而是纯算法方案：

  1. **沙箱执行**：每次 `ctx_execute` 调用启动独立子进程，脚本在隔离环境中运行。stdout 被捕获并摘要化——只把关键信息（执行状态、输出摘要、错误信息）放入上下文。原始数据留在沙箱里，不会污染上下文窗口。
  2. **SQLite FTS5 + BM25 索引**：原始数据全文索引到 SQLite 中。当模型需要更多细节时，通过 `ctx_search` 用 BM25 排序检索相关片段。不是"丢掉"数据，而是"放到一边，需要时再查"。
  3. **PreToolUse Hook 自动路由**：拦截 Bash、Read、WebFetch、Grep 等工具调用，将大批量输出自动导向沙箱执行，而不是直接入上下文。
  4. **会话连续性**：PostToolUse、PreCompact、SessionStart 三个 Hook 协同工作，捕获文件编辑、git 操作、用户决策等关键事件。Compaction 发生时，从 SQLite 中检索最近的会话状态并注入恢复 prompt。

  **支持的 10 种语言运行时**：JavaScript、TypeScript、Python、Shell、Ruby、Go、Rust、PHP、Perl、R。Bun 自动检测，JS/TS 执行速度提升 3-5 倍。

### 安装配置

  **Claude Code（推荐，全自动）：**

  ```
  /plugin marketplace add mksglu/context-mode
  /plugin install context-mode@context-mode
  ```

  重启 Claude Code 即可。插件自动注册所有 Hook（PreToolUse、PostToolUse、PreCompact、SessionStart）和 11 个 MCP 工具。

  **可选：状态栏显示节省统计**

  在 `~/.claude/settings.json` 中添加：

  ```json
  {
    "statusLine": {
      "type": "command",
      "command": "context-mode statusline"
    }
  }
  ```

  状态栏会实时显示 `$ saved this session · $ saved across sessions · % efficient`。

  **前提条件**：Claude Code v1.0.33+，Node.js ≥ 22.5（或 Bun）。

  **跨平台支持**：除 Claude Code 外，还支持 Gemini CLI、VS Code Copilot、JetBrains Copilot、Cursor、OpenCode、KiloCode、OpenClaw、Pi、Antigravity、Kiro、Zed、Windsurf、Cline、RooCode 共 15 个平台。

### 使用指南

  #### 场景一：长会话不"失忆"

  你在做一个复杂功能，需要反复查 API 文档、运行测试、看 git log、检查数据库。通常 30 分钟后 Claude 开始忘记前面的讨论。

  安装 context-mode 后，所有这些工具调用的输出被自动沙箱化。你不需要改变任何工作习惯——照常使用 Bash、Read、WebFetch、Grep——Hook 会自动拦截并将大输出路由到沙箱。同一个会话可以持续 3 小时以上。

  #### 场景二：会话中断后无缝恢复

  你正和 Claude 讨论一个重构方案，已经确定了 5 个步骤，改了 3 个文件。突然上下文耗尽，会话 compaction 了。

  正常情况下你需要重新解释一遍整个方案。有了 context-mode，直接 `claude --continue` 继续——它从 SQLite 中检索最近的文件编辑、git 操作和你的决策记录，模型自动恢复到中断前的状态。

  #### 场景三：搜索历史工具输出

  之前跑过的测试输出里有一个关键错误信息，但你忘了具体内容。正常情况下上下文已经滚出窗口了。

  使用 `ctx_search` 工具：

  ```
  ctx_search "NullPointerException in UserService"
  ```

  BM25 排序返回最相关的历史输出片段，不用重新跑测试。

  #### 场景四：批量数据处理不占上下文

  你需要分析一个 500 行的 CSV 文件，找出其中的异常数据。正常情况下整个 CSV 内容都会进上下文。

  context-mode 下，`ctx_execute` 在沙箱中运行 Python 脚本处理 CSV，只返回分析结论和关键统计数字（222 B），而不是原始 85.5 KB 数据。原始 CSV 被索引到 SQLite，需要时可以搜索。

### 进阶技巧

  **组合 caveman + context-mode**：caveman 压缩模型输出（节省 ~65% 输出 token），context-mode 压缩工具输出（节省 ~98% 工具输出 token）。两者互补，一个管模型说话，一个管工具回话。同时使用后，一个典型的开发会话可以轻松撑 4-5 小时。

  **只在特定项目启用**：如果不想全局安装，可以在项目的 `.claude/settings.json` 中只对当前项目启用：

  ```json
  {
    "plugins": {
      "context-mode@context-mode": {
        "enabled": true
      }
    }
  }
  ```

### 常见踩坑

  **偶尔会过滤掉你需要看的详细输出**。context-mode 的摘要策略是启发式的，不是完美无缺。在复杂多文件任务中，建议定期检查会话日志确认上下文完整性。如果发现关键信息被沙箱过滤了，可以用 `ctx_search` 直接搜索原始数据。

  **Codex CLI 不支持 Hook**。Codex CLI 目前没有 Hook 机制，只能通过复制路由指令文件到 `AGENTS.md` 获得约 60% 的节省效果（手动触发沙箱），无法实现全自动路由和会话连续性。推荐在 Claude Code 或 Gemini CLI 上使用以获得完整体验。

  **安装后不生效**。确认 Claude Code 版本 ≥ v1.0.33（`claude --version`）。如果 `/plugin` 命令不被识别，先更新：`brew upgrade claude-code` 或 `npm update -g @anthropic-ai/claude-code`。安装后需要重启 Claude Code。

## 今日小结

  context-mode 解决的是一个每个 Claude Code 用户都会遇到的问题：上下文窗口是硬通货，但工具输出在疯狂烧钱。安装只要一行命令，零配置，装完即用。如果你每天用 Claude Code 超过 1 小时，这个技能能让你省下大量"重新解释上下文"的时间——30 分钟的会话延长到 3 小时，中断后无缝恢复，不再重新来过。搭配 caveman 使用，你的 token 消耗从"烧钱模式"变成"精打细算模式"。
