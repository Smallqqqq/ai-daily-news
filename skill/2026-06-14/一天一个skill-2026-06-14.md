# prompt-refine-skill — 你的每句话，模型自己先润色一遍

  你用 Claude 时写 prompt 是一种风格，用 GPT 时又是另一种风格，切来切去还得记住每个模型喜欢什么样的提示词格式。prompt-refine-skill 自动检测当前运行的是哪个模型，在回答之前先按该模型的官方最佳实践把你原始的请求重新组织一遍——整个过程对你是透明的，你只看到最终更好的回答。

---

### 它解决什么问题

  不同 AI 模型的提示策略差异很大：Claude 偏好 XML 结构化标签，GPT 喜欢 Markdown 分段指令，Gemini 有自己的一套规范。每次切换模型，你都得在心里转换句式，否则同样的需求，换个模型可能输出质量骤降。prompt-refine-skill 接管了这件事——它以 Agent Skill 的形式装在对话工具里，激活后自动识别宿主模型，把用户的请求静默改写为该模型最优的提示格式，再让模型回答。

### 核心原理

  prompt-refine-skill 的设计哲学很简单：**按宿主模型路由，不按任务路由**。如果当前是 Claude 在回答，就用 Anthropic 的策略；如果是 GPT 在回答，就用 OpenAI 的策略。一个编码任务不会把 Claude 切成 GPT 风格，一个写作任务也不会把 GPT 切成 Claude 的 XML 风格。

  技术实现上非常轻量——没有运行时依赖，没有 App Server，没有额外的优化器调用。Skill 文件本身极短，加上选中的策略文件，在上下文中只占很少的 token。它内置了 14 套模型策略（Claude、GPT、Gemini、DeepSeek、Llama、Mistral、Qwen、Grok、Perplexity、Kimi、Cohere、Nova、Phi、通用回退），每套策略都提炼自对应厂商的官方提示工程指南。

  它还是上下文感知的：后续追问会继承之前的意图、约束和术语偏好，但最新的用户指令始终优先。

### 安装配置

```bash
# Claude Code（推荐）
git clone https://github.com/Li-Bailiang/prompt-refine-skill.git .claude/skills/prompt-refine

# 或使用 degit（不含 .git 目录，更干净）
npx degit Li-Bailiang/prompt-refine-skill .claude/skills/prompt-refine

# Cursor / Codex / Gemini CLI 等
git clone https://github.com/Li-Bailiang/prompt-refine-skill.git .agents/skills/prompt-refine
```

激活后，在对话中输入 `/prompt-refine` 即可开启。可用控制命令：

```text
/refine verbose    # 显示改写前后的对比（调试用）
/refine off        # 关闭改写（当前对话剩余部分）
/prompt-refine     # 重新激活（上下文压缩后或新会话）
```

### 使用指南

#### 场景一：模糊需求的自动补全

  你随手丢一句"帮我分析一下这个市场"，什么都没说清楚。prompt-refine-skill 自动识别到需求不完整，在内部把提示重写为带有角色定义、约束条件、输出格式的结构化指令——Claude 拿到的是 XML 标签格式的分析框架，GPT 拿到的是 Markdown 分段的任务说明。你看到的最终回答不再是敷衍的套话，而是有假设前提、有竞争格局、有战略含义的专业分析。

#### 场景二：精确约束的保护

  你提了一个很具体的要求："写一个 5 条 npm 发版检查清单，每条不超过 8 个字"。prompt-refine-skill 判断这是一个约束清晰的需求，内部的改写只加了最小干预——保留你的原始约束，添加输出格式合约和自我质量检查。最终回答严格遵循你的字数限制，不会"自作主张"地展开成长篇大论。

#### 场景三：多模型切换的零摩擦体验

  你在 Claude Code 里写代码，切换到 Codex CLI 里做代码审查，又在 Cursor 里写文档。prompt-refine-skill 装一次，三个环境都生效——每个环境自动用当前模型的策略优化你的 prompt，你不需要记住三种不同的提示写法，也不需要每次切工具时调整自己的表述习惯。

### 进阶技巧

  **中英文混合场景**：prompt-refine-skill 在 60 条中文 prompt 上的盲测胜率是 72.9%（与英文 75.0% 接近）。中文用户不需要刻意写英文 prompt——用自己的语言表达，Skill 会按模型的策略优化结构，同时保留你的语言和意图。

  **组合技：prompt-refine + verbose 调试**：刚装上的时候，建议开启 `/refine verbose` 跑几轮对话，观察改写前后的差异。你会发现很多惊喜——比如原来 Claude 把你的模糊需求补全得这么完整，而 GPT 把你的精确约束保护得这么严格。熟悉后关掉 verbose，享受静默优化。

### 常见踩坑

  **长对话中 Skill 可能失活**：prompt-refine-skill 的优化效果依赖于 Skill 内容保持在上下文中。如果你的对话很长、发生了上下文压缩，Skill 可能会暂时失效。解决方案：重新输入 `/prompt-refine` 激活即可。

  **对已经很清晰的 prompt 无明显提升**：这是设计如此。prompt-refine-skill 不是在每个请求上都强行改写——如果你的 prompt 已经结构完整、约束明确，Skill 会做最小干预甚至不干预。它不是把好 prompt 变更好的工具，而是把烂 prompt 变好的工具。
