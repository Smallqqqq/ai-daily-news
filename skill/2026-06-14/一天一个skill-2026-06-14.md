# caveman — 让 Agent 闭嘴省钱，技术精度一点都不少

  安装 caveman 之后，你的 AI Agent 输出 token 直接砍掉 65%-75%。不是让你忍着看简略回答，而是把所有"我理解你的问题""让我帮你看看""希望这对你有帮助"之类的废话全删了，只留下纯技术干货。代码块、URL、文件路径、命令——一个字不少。2 个月 72000+ star，这不是噱头，是真金白银的省钱机器。

### 它解决什么问题

  Claude Code（以及 Codex、Gemini 等）在进行代码解释、Bug 分析、PR Review 时，输出里夹杂大量礼貌用语、过渡句、解释性铺垫。这些 token 你花钱买，读完浪费时间，对解决问题毫无帮助。caveman 在 Agent 输出层面做了一件事：把所有自然语言的冗余全部剥离，只保留技术结论。根据真实 API 测试数据，10 个常见任务平均节省 **65%** 输出 token，其中 React 错误边界实现和 re-render 解释场景节省高达 **87%**。

  更关键的是，这不仅仅是省 token 钱。2026 年 3 月一篇论文（"Brevity Constraints Reverse Performance Hierarchies in Language Models"）发现，约束大模型简短回复后，某些基准上的准确率反而提升了 26 个点。少说话不一定是偷懒，有时候是更准。

### 核心原理

  caveman 的工作机制分五层，不是简单地在 prompt 里加一句"说简短点"。

  **第一层：Skill 注入。** 安装后，SKILL.md 被写入 Agent 的技能目录。Agent 启动时加载其 description 和 name，当你说"caveman mode"或输入 `/caveman` 时，完整规则集被注入上下文，告诉 Agent：删除填充词、保留实质内容、使用片段式表达。

  **第二层：Hook 自动激活。** 对 Claude Code，安装脚本额外写入一个 SessionStart hook，每次会话开始时写一个标记文件。Agent 读到标记，从第一条消息起就用 caveman 风格说话——不需要你手动触发 `/caveman`。

  **第三层：压缩等级。** 四个等级可选：`lite`（去掉礼貌用语，保留语法）、`full`（默认，去掉冠词、只留关键片段）、`ultra`（电报式极限压缩）、`wenyan`（文言文模式，比白话更短）。

  **第四层：输入压缩。** `/caveman-compress` 子技能把你的 CLAUDE.md、项目笔记等记忆文件重写成 caveman 风格，每次会话启动时的上下文就小了约 46%。代码块、URL、路径、日期精确保留，只压缩自然语言描述。

  **第五层：统计面板。** `/caveman-stats` 读取 Claude Code 的会话日志，统计实际 token 使用量和终身节省量，换算成美元，显示在状态栏上。

  **一个关键点**：caveman 只压缩输出 token，不影响模型的 thinking/reasoning token。大脑还是那么大，只是嘴巴变小了。

### 安装配置

```bash
# macOS / Linux / WSL / Git Bash
curl -fsSL https://raw.githubusercontent.com/JuliusBrussee/caveman/main/install.sh | bash

# Windows (PowerShell)
irm https://raw.githubusercontent.com/JuliusBrussee/caveman/main/install.ps1 | iex

# 或通过 npx 按技能安装
npx skills add JuliusBrussee/caveman
```

  约 30 秒完成。需要 Node ≥ 18。安装脚本自动检测你装了哪些 Agent（Claude Code、Codex、Gemini、Cursor、Windsurf 等 30+），逐个安装。没装的 Agent 自动跳过。可以安全重复运行。

  启用：输入 `/caveman` 或者说 "talk like caveman"。停止：说 "normal mode"。

### 使用指南

#### 场景一：日常编码对话

  **触发条件：** 在 Claude Code 中调试代码、询问技术问题、让 Agent 解释某个概念。

  **操作：** 输入 `/caveman` 启用压缩模式，然后正常提问。

  **效果：**
  ```
  你：这个 React 组件为什么一直重新渲染？

  正常 Claude（69 tokens）：
  "The reason your React component is re-rendering is likely because you're
  creating a new object reference on each render cycle. When you pass an inline
  object as a prop, React's shallow comparison sees it as a different object every
  time, which triggers a re-render. I'd recommend using useMemo to memoize the
  object."

  caveman（19 tokens）：
  "New object ref each render. Inline object prop = new ref = re-render.
  Wrap in `useMemo`."
  ```

#### 场景二：一键压缩记忆文件

  **触发条件：** 你的 CLAUDE.md 或项目笔记越来越长，每次会话都要消耗大量输入 token。

  **操作：** 输入 `/caveman-compress CLAUDE.md`。

  **效果：** 真实数据——平均压缩 46%。一个 1122 token 的项目 CLAUDE.md 压缩到 636 token。代码块、路径、URL 原样保留。自动备份 `.original.md` 方便回退。

#### 场景三：精简 Commit 和 PR Review

  **触发条件：** 写完代码准备提交，或者收到 PR 需要 review。

  **操作：**
  - 提交：`/caveman-commit`，自动生成 ≤50 字符的 Conventional Commit 消息
  - Review：`/caveman-review`，生成单行评论，如 `L42: 🔴 bug: user null. Add guard.`

  **效果：** Commit 消息从"添加了用户认证功能，包括登录、注册和密码重置"变成 `feat(auth): add login, register, reset password`。PR 评论从一段文字变成一行定位。

#### 场景四：跨语言压缩

  **触发条件：** 你用中文、日语、葡萄牙语等任何语言和 Agent 交流。

  **操作：** caveman 只压缩风格，不改变语言。你用中文提问，Agent 用中文 caveman 风格回复。

  **效果：**
  ```
  正常："这个问题的根本原因是你的认证中间件没有正确验证 token 过期时间。
         我建议你检查一下时间比较逻辑。"

  caveman："认证中间件 token 过期判断用 `<` 而非 `<=`。改掉。"
  ```

#### 场景五：MCP 工具描述压缩

  **触发条件：** 你的 Agent 连接了大量 MCP 服务器，工具描述占用大量上下文。

  **操作：** 安装 `caveman-shrink` npm 包作为 MCP 中间件，自动压缩所有 MCP 工具描述。

  ```bash
  npm install -g caveman-shrink
  # 配置为 MCP 中间件包装器
  ```

  **效果：** MCP 工具描述被压缩，Agent 上下文更干净，长会话中表现更稳定。

### 进阶技巧

  **组合拳：caveman + prompt caching。** caveman 削减输出，Claude 的 prompt caching 削减输入。缓存命中后输入 token 成本降至原来的 10%。两项叠加，单次问答总成本可降至原来的 1/3 以下。

  **永久生效：** 对 Claude Code、Codex、Gemini CLI，caveman 安装后默认开启自动激活（built-in 支持）。对 Cursor、Windsurf、Cline 等，使用 `--with-init` 标志写入常驻规则文件，每次会话自动生效，不需要手动触发。

  **caveman 生态全家桶：** 同一作者还提供了 caveman-code（整个终端 Agent 都是压缩版，比 Codex 省 2× token）、cavemem（跨 Agent 记忆系统）、cavekit（spec 驱动的构建循环）、cavegemma（用 caveman 语料微调的 Gemma 4 31B 模型）。五个工具遵循同一哲学：Agent 做更多事，用更少 token。

### 常见踩坑

  **坑 1：深度工作流中 caveman 可能过度压缩。** caveman 最适合单轮回答场景（代码解释、Bug 修复建议、简短 Review）。对于需要 Agent 输出大量上下文的多步骤工作流（如复杂架构设计、长篇文档撰写），过度压缩会丢失必要的推理过程。建议：长文档撰写时暂时 `normal mode`，写完切回 caveman。

  **坑 2：输出节省不代表总成本等比例下降。** caveman 只压缩输出 token，不压缩输入和 thinking token。如果你的使用模式是大量工具调用（Agent 读取很多文件再回答），输出 token 占总成本的比重其实不大。一篇实际测试指出，Agent 做了 8 次工具调用消耗 15000 输入 token 后，caveman 省 100 输出 token，总节省不到 1%。caveman 最划算的场景是**人工对话多、工具调用少**的会话。

  **坑 3：某些 Agent 需要手动触发。** 虽然 Claude Code 支持 Hook 自动激活，但部分 Agent 需要每次会话手动 `/caveman`。安装后检查你的 Agent 的 features matrix（见 INSTALL.md），确认是否开启了自动激活。

| 技能 | 安装方式 | 核心特点 | 适用场景 | 热度 |
|------|---------|---------|---------|------|
| **caveman**（今日推荐）| `curl 一键安装` | 输出 token 压缩 65-75%，四级压缩，跨 30+ Agent | 日常编码对话、PR Review | ⭐ 72k |

  caveman 的优势在于它不是泛泛的"说简短点"，而是有四级精确控制、有基准测试数据、有自动激活机制、有输入压缩配套。更重要的是，它是为 AI Agent 量身定做的——压缩的是 agent 的啰嗦风格，而不是让你去写 prompt。
