# SkillAnything — 给一个目标，还你一个生产级 AI Agent Skill

  AI Agent 的技能生态正在以周为单位膨胀。每天都有新的 SKILL.md 被推到 GitHub——操作 Playwright 的、写 commit message 的、管 PR review 循环的。但有一个问题没人解决：如果你用的工具不在热门列表里怎么办？如果你想让 Claude Code 精通 `jq`、精通你们内部的那个 CLI、精通公司自研的 API 网关——你只能自己手写一份 SKILL.md。SkillAnything 把这件事自动化了：告诉它目标是什么，它给你生成一个经过评估和优化的跨平台技能包。445 stars，MIT 协议，2026 年 4 月发布。

### 它解决什么问题

  手写一份 SKILL.md 听起来不难——一个 Markdown 文件，写清楚什么时候触发、做什么事。但实际上，写一份**能稳定触发、质量过关**的 skill 至少有五个坑：

  1. **描述写不准**。`description` 字段决定了 Claude 什么时候加载这个技能。写得太大触发太频繁（浪费 token），写得太窄永远不触发（白装了）。
  2. **不知道要给 AI 什么上下文**。一份好的 SKILL.md 需要告诉 AI：这个工具怎么用、参数怎么传、常见错误怎么处理。这些知识你脑子里有，但你不会一次性全写下来。
  3. **没有评估机制**。你写的 skill 到底好不好用？能不能在正确的场景下激活？没人帮你测。
  4. **跨平台适配**。Claude Code 支持 hooks 和 tool restrictions，Codex 用 `openai.yaml` 做元数据，OpenClaw 有自己的配置格式。给每个平台各写一份？没人有那个时间。
  5. **维护**。工具升级了、API 改了、最佳实践变了——你的 skill 还停留在半年前的版本。

  SkillAnything 用一个 7 阶段自动化流水线把这五个坑全填了。它不是一个"帮你写文案"的 prompt 模板，而是一个**会执行代码、会跑评估、会打包多平台**的元技能。

### 核心原理

  SkillAnything 把自己叫"元技能"——一个生成技能的技能。它的核心是一个 Python 驱动的 7 阶段流水线：

  ```
  目标（jq / Stripe API / 你的工作流）
    ↓
  [分析] → [设计] → [实现] → [测试] → [评估] → [优化] → [打包]
    ↓
  跨平台 skill 包（Claude Code / Codex / OpenClaw / 通用格式）
  ```

  **阶段 1：分析（Analyze）**。自动检测目标类型——是 CLI 工具就解析 `--help` 输出，是 REST API 就抓 OpenAPI/Swagger 规范，是 Python 库就读 docstring，是工作流就分析步骤依赖。输出一份 `analysis.json`，列出所有能力点。

  **阶段 2：设计（Design）**。把分析结果映射到 skill 架构——用什么模式（tool-augmentation、workflow-orchestrator、reference-lookup），需要几个脚本、几个 reference 文件。输出 `architecture.json`。

  **阶段 3：实现（Implement）**。根据架构生成完整的 skill 目录：`SKILL.md`（含 description、触发条件、工作流指令）、脚本文件（如需要）、参考文档（如需要）。这不是模板填空——Claude Code 在阶段 3 充当"skill 作者"的角色，根据分析结果实际编写内容。

  **阶段 4-5：测试与评估**。自动生成测试用例和触发查询（比如"帮我用 jq 提取 JSON 中的 name 字段"），然后在有 skill 和无 skill 两种条件下分别跑，对比结果。评分维度包括：触发准确率（该触发时是否触发）、输出质量（有无 skill 的质量差异）、token 效率。

  **阶段 6：优化（Optimize）**。如果触发准确率低于阈值，SkillAnything 进入优化循环（最多 5 轮）：微调 description → 重新跑评估 → 对比结果 → 留下更好的版本。这个阶段解决的是"skill 写完了但触发不稳定"的常见痛点。

  **阶段 7：打包（Package）**。把优化后的 skill 打包为多平台格式——Claude Code 的 `~/.claude/skills/` 结构、Codex 的 `openai.yaml` 伴生文件、OpenClaw 的 `settings.json` 配置，以及一个平台无关的 `.skill.zip` 通用格式。

  整个流水线有两种运行模式：**全自动模式**（`auto_mode: true`，一口气跑完七个阶段）和**交互模式**（每个阶段结束后停下来让你审阅确认）。

### 安装配置

  SkillAnything 本身是一个 Agent Skill，安装到 Claude Code 只需一条命令：

```bash
git clone https://github.com/AgentSkillOS/SkillAnything.git ~/.claude/skills/skill-anything
```

  前置依赖：Python ≥ 3.9。不需要额外安装 pip 包——SkillAnything 的脚本只依赖 Python 标准库。Claude Code 的 agent 调用负责实际的内容生成和评估。

  安装后重启 Claude Code 会话，SkillAnything 会被自动发现。直接在对话中说"给 ffmpeg 生成一个 skill"即可触发。

  如果你要在 Claude Code 以外使用，也可以单独跑某个阶段：

```bash
# 只分析目标
python -m scripts.analyze_target --target "jq" --output analysis.json

# 只看架构设计
python -m scripts.design_skill --analysis analysis.json --output architecture.json
```

  可选的 `config.yaml` 配置：

```yaml
pipeline:
  auto_mode: true           # true=全自动, false=每阶段人工确认
  skip_eval: false          # true=跳过评估优化，快速出原型

platforms:
  enabled: [claude-code, openclaw, codex, generic]
  primary: claude-code

eval:
  max_optimization_iterations: 5   # 优化循环最多跑几轮
  runs_per_query: 3                # 每个查询跑几次取平均
```

### 使用指南

#### 场景一：给团队的 CLI 工具生成专属 skill

  你们团队维护了一个叫 `deploy-svc` 的内部 CLI，负责把微服务部署到 K8s。它有 12 个子命令、40 多个参数，新人根本记不住。你希望 Claude Code 能帮同事正确使用它。

  直接在 Claude Code 中说：

  ```
  给 deploy-svc 这个 CLI 生成一个 skill，它部署微服务到 K8s
  ```

  SkillAnything 检测到 CLI 类型，解析 `deploy-svc --help` 的输出，提取所有子命令和参数。生成的 SKILL.md 会包含：触发条件（部署相关关键词）、每个子命令的用法、参数说明、常见的排障步骤。同事以后只需要说"帮我把 user-service 部署到 staging"，Claude Code 就会加载这个 skill，给出正确的 `deploy-svc` 命令。

#### 场景二：把 REST API 变成 AI 可调用的技能

  你的后端系统有一个内部 API 网关，文档是用 OpenAPI 3.0 写的。你想让 Claude Code 在开发时能直接查询这个 API 的结构、参数、返回格式。

  ```
  给 http://api-gateway.internal/openapi.json 生成一个 skill，名字叫 internal-gateway
  ```

  SkillAnything 抓取 OpenAPI spec，提取所有 endpoint、请求参数、响应 schema，生成一个以 reference-lookup 模式为主的 skill。以后你问"用户列表接口的查询参数有哪些"，Claude Code 直接从这个 skill 的参考文档里找到答案，不用你自己翻文档或问同事。

#### 场景三：把多步骤工作流固化为可复用的 skill

  你有一个每月执行的数据处理流程：从 PostgreSQL 导出上月数据 → 用 Pandas 清洗 → 上传到 S3 → 触发 Athena 查询。每次做都怕漏步骤。

  ```
  把这个流程变成 skill：从 Postgres 导数据、Pandas 清洗、上传 S3、触发 Athena
  ```

  SkillAnything 识别出 4 个步骤的工作流模式，给每个步骤生成详细指令（连接字符串怎么传、清洗规则是什么、S3 路径格式），步骤之间有错误处理逻辑——某一步失败了，后续步骤自动跳过并提示你检查。生成的 skill 是一个可复用的工作流模板，下个月直接说"跑月度数据处理"就行。

#### 场景四：快速原型，跳过评估

  你只是想知道"给 Redis CLI 做 skill 大概什么结构"，不需要评估和优化：

  ```yaml
  # config.yaml
  pipeline:
    skip_eval: true
  ```

  SkillAnything 跳过阶段 4-6，直接生成初始版本的 skill。5 分钟内产出一个能用的原型，后续再手动迭代。

### 进阶技巧

  **组合使用 SkillAnything 和 gh skill**。SkillAnything 生成的 skill 包可以直接作为 GitHub 仓库发布，然后团队用 `gh skill install` 一键安装。工作流是：SkillAnything 生成 → push 到 GitHub → 队友 `gh skill install`。从"一个点子"到"全团队在用"只需 20 分钟。

  **批量生成 skills**。如果你有一个工具链（比如 `jq` + `httpie` + `gh`），可以一口气全生成：

  ```
  给 jq、httpie、gh 三个 CLI 分别生成 skill
  ```

  SkillAnything 为每个目标独立跑全流程。产出的三个 skill 互不冲突，可以并存。

  **自定义优化循环**。如果你觉得默认的 5 轮优化不够（或者太多），改 `config.yaml` 里的 `max_optimization_iterations`。基于实际测试，大部分 skill 在 3 轮以内触发准确率就能达到 85% 以上，5 轮是给那些"触发条件特别微妙"的 skill 留的余量。

### 常见踩坑

  **不是所有目标都能自动检测**。目前 SkillAnything 支持 CLI 工具、REST API（OpenAPI/Swagger）、Python 库、工作流描述、Web 服务文档。GraphQL endpoint、gRPC 服务、Windows 专有工具目前不在自动检测范围内。碰到这些情况，手动描述目标也能走完流程——SkillAnything 会降级为"按描述生成"模式。

  **评估阶段依赖 Claude API**。阶段 5 的 benchmark 需要调用 Claude API 做 A/B 对比，会消耗 token。如果你的 API 额度紧张，建议把 `skip_eval` 设为 `true`，先跑出原型再手动检查质量。

  **Python 脚本路径**。SkillAnything 的入口是 `~/.claude/skills/skill-anything/`，但 Python 脚本在子目录 `scripts/` 里。如果手动跑单个阶段，命令是 `python -m scripts.analyze_target`，需要在 skill 根目录下执行。

| 方案 | 安装方式 | 核心特点 | 适用场景 | 热度 |
|------|---------|---------|---------|------|
| **SkillAnything**（今日推荐）| `git clone` 一条命令 | 7 阶段自动流水线、多目标类型、跨平台打包、内置评估优化 | 为任意工具/API/工作流生成生产级 skill | ⭐ 445 |
| Anthropic skill-creator | 官方插件市场 | 交互式 Q&A 引导，Claude 官方出品 | 有明确想法需要对话式细化的场景 | ⭐ 官方 |
| FrancyJGLisboa/agent-skill-creator | `npx skills add` | 17 平台、security scan、PDF/URL 输入 | 跨 17 平台的复杂 skill 生成 | ⭐ 1.6K |

  SkillAnything 的差异化在于"一键全自动"。Anthropic 官方的 skill-creator 是交互式 Q&A——你需要和 Claude 来回对话细化需求，适合"我有明确想法但需要 AI 帮我打磨"的场景。SkillAnything 反过来——你扔一个目标给它，它自己分析、设计、实现、评估、优化、打包，你全程只需要发一条消息。如果手写 SKILL.md 像自己做菜，官方 skill-creator 像跟着大厨边问边做，SkillAnything 就是点了个外卖——但它是现做的。

## 今日小结

  SkillAnything 适合所有想让 AI 编程助手"学会"特定工具的开发者——不管那个工具是 jq、httpie、ffmpeg、Stripe API，还是你们团队内部没人想写文档的 CLI。安装一条 git clone，之后把"给 XX 生成一个 skill"当一句话的事。它不是一个更好的 skill 编辑器，它把"写 skill"变成了"声明目标"。对于那些在 2026 年想要跟上 AI Agent 技能生态的团队，这个元技能提供的不是一张技能表，而是一条技能产线。
