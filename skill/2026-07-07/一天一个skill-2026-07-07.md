# Google agents-cli — 一键把你的 AI 编程助手，变成 AI Agent 构建专家

  你今天让 Claude Code 写了个 Web 应用，明天你可能想让 Claude Code 帮你构建一个 AI Agent——能调用工具、有评估体系、能部署到生产环境的那种。但 Claude Code 不懂 Google Cloud 的 Agent Development Kit，不知道 eval 怎么跑，不清楚部署到 Cloud Run 需要哪几个步骤。Google agents-cli 把 Google 内部构建 AI Agent 的完整方法论，打包成了 7 个 SKILL.md 文件和一套 CLI 工具，一条命令装进你的编程助手。4,574 stars，Apache 2.0，7 月 1 日刚发布 v1.0.0 正式版。

### 它解决什么问题

  2026 年每个人都在用 AI 编程助手写代码。但如果你想更进一步——让 AI 帮你构建一个 AI Agent——事情立刻复杂十倍。

  一个能上生产的 AI Agent 需要：选择合适的模型、设计工具调用逻辑、写评估用例、配置部署基础设施、接入可观测性、注册到企业平台。每一步都有独立的工具链和 API。Google Cloud 这边，ADK（Agent Development Kit）负责 agent 逻辑，Cloud Run 或 GKE 负责部署，Cloud Trace 负责监控，Terraform 负责基础设施，Gemini Enterprise 负责企业注册。把这些串起来的难度，约等于又做了一份全职工作。

  agents-cli 把这摊碎片拼回一条流水线。它的定位很精准：一个**给编程助手用的专家层**。装完以后，你对着 Claude Code 说"帮我构建一个压缩文本的 caveman agent"，它会自动走完 scaffold → build → eval → deploy → observe 的完整流程。不需要你手动翻文档、记命令、配 YAML。

### 核心原理

  agents-cli 分两层：

  **上层：7 个 SKILL.md 技能文件**。安装时写入编程助手的技能目录。每个文件按"渐进式披露"设计——日常只加载 metadata（约 100 tokens），任务匹配时才加载完整内容。7 个技能覆盖完整生命周期：

  | 技能 | 教编程助手什么 |
  |------|-------------|
  | `google-agents-cli-workflow` | 开发全流程、代码保留规则、模型选择指引。始终激活 |
  | `google-agents-cli-adk-code` | ADK Python API：agent 类型、工具定义、编排模式、回调、状态管理 |
  | `google-agents-cli-scaffold` | `create`、`enhance`、`upgrade` 三个脚手架命令 |
  | `google-agents-cli-eval` | 评估方法论：指标、数据集、LLM-as-judge、自适应评分标准 |
  | `google-agents-cli-deploy` | 部署到 Agent Runtime / Cloud Run / GKE + CI/CD + 密钥管理 |
  | `google-agents-cli-publish` | Gemini Enterprise 注册 |
  | `google-agents-cli-observability` | Cloud Trace、日志、BigQuery Agent Analytics、第三方集成 |

  **下层：CLI 命令**。`agents-cli` 本身是一个 Python CLI（78% Python + Terraform + Go），可以直接在终端里跑，不依赖编程助手。核心命令：

  ```
  agents-cli setup          # 安装 CLI + 技能到编程助手
  agents-cli scaffold       # 创建/增强/升级 agent 项目
  agents-cli eval generate  # 跑 agent 推理，产出 trace
  agents-cli eval grade     # 按指标评分
  agents-cli eval optimize  # GEPA 算法自动优化 prompt
  agents-cli deploy         # 一键部署
  agents-cli publish        # 注册到 Gemini Enterprise
  ```

  **5 阶段评估飞轮**（eval pipeline）。这是 agents-cli 最有技术含量的部分，也是大多数 agent 工具直接跳过的一环：

  1. `eval dataset synthesize` — 用 LLM 模拟用户，从 agent 的指令和工具定义自动生成多轮测试场景。不需要人手写测试用例
  2. `eval generate` — 跑 agent 推理，捕获完整 trace 含工具调用
  3. `eval grade` — 用内置或自定义指标评分
  4. `eval analyze` — 聚类失败模式，告诉你 agent 在哪类问题上**系统性**表现不好
  5. `eval optimize` — GEPA（Genetic Evolutionary Prompt Augmentation）算法，基于失败分析自动改写 agent 的系统指令，改完自动重跑评估验证效果

  第五步是区分"能部署 agent"和"敢在生产上用 agent"的分水岭。一个编程助手能在五分钟内帮你搭出一个 agent；但只有集成的评估系统能告诉你这个 agent 是否真的可靠。

  **本地开发不需要 Google Cloud 账号**。用 Google AI Studio 的 API key 就能在本地完成 scaffold、run、eval 全流程。只有部署到生产才需要 Google Cloud。

### 安装配置

  前提：Python 3.11+、uv 包管理器、Node.js。

  ```bash
  # 一键安装 CLI + 7 个技能到所有检测到的编程助手
  uvx google-agents-cli setup

  # 或只装技能，让编程助手自己处理后续
  npx skills add google/agents-cli
  ```

  安装后重启 Claude Code（或其他编程助手），技能自动被发现。不需要额外配置文件。验证方式：在 Claude Code 中输入 `/help`，确认 google-agents-cli 相关技能出现在列表中。

  支持平台：macOS、Linux、Windows（WSL 2）。

### 使用指南

  #### 场景一：从零构建一个 AI Agent，全程不需要翻文档

  你想做一个"caveman 风格"的文本压缩 agent——输入一段啰嗦的文字，输出电报式的精简版。

  对着 Claude Code 说：

  ```
  用 agents-cli 构建一个 caveman agent，把冗长文本压缩成简短的技术摘要
  ```

  Claude Code 自动激活 workflow + scaffold 技能：
  - 先问几个澄清问题（部署目标？安全约束？）
  - 把需求写成 `.agents-cli-spec.md`
  - 执行 `agents-cli scaffold create caveman-agent --prototype --yes`
  - 编辑 `app/agent.py`，用 ADK 的 `Agent` 类定义 agent 逻辑
  - 跑 `agents-cli run "请帮我理解部署选项"` 冒烟测试
  - 输出：`Deploy options: Agent Runtime, Cloud Run, GKE. Pick one. Ship.`

  五分钟内从一句话到一个能跑的 agent。你不需要知道 ADK 的 API 长什么样——workflow 技能里写好了。

  #### 场景二：评估 agent 质量，不是"看起来还行"而是"有数据支撑"

  你刚搭好的 agent 在几个测试用例上表现不错，但你不知道它在边界情况下的表现，也不知道它是否对某些类型的输入系统性失败。

  对着 Claude Code 说：

  ```
  给这个 caveman agent 写评估用例，跑完告诉我有哪些问题
  ```

  Claude Code 激活 eval 技能，走完完整评估流程：
  - `eval dataset synthesize` — 自动生成压缩质量、技术术语保留、caveman 语气一致性等多个维度的测试用例
  - `eval generate` — 跑 agent 推理，捕获每次工具调用的完整 trace
  - `eval grade` — 按指标打分
  - `eval analyze` — 聚类失败模式，比如"对问候语的回复过于礼貌，不像 caveman"
  - 如果某类问题需要修复，说一句"让回复更粗鲁一些"，Claude 改完指令后重新评估

  这套流程解决了 agent 开发中最被忽视的问题：你改了一个 prompt，它是真的变好了，还是只是换了一种方式失败？eval 数据给你答案。

  #### 场景三：一键部署到 Google Cloud，附带可观测性

  你的 agent 在本地跑得很好，现在要上线。

  ```
  把这个 agent 部署到 Cloud Run，配好监控
  ```

  Claude Code 激活 deploy + observability 技能：
  - `agents-cli scaffold enhance --deployment-target cloud_run` — 添加部署基础设施
  - `agents-cli deploy` — 一键部署
  - `agents-cli infra single-project` — 配置服务账号、GCS bucket、BigQuery 数据集
  - 部署完成后，Cloud Trace 默认开启。打开 Google Cloud Console 的 Trace explorer，发几个请求就能看到每个 LLM 调用和工具执行的 span

  从本地原型到生产环境，中间不需要你手写一行 Terraform 或 YAML。

  #### 场景四：用 eval optimize 让 agent 自动进化

  你的 agent 在 50 个评估用例上的准确率是 72%。改了几轮 prompt 后涨到了 78%，然后就卡住了。你不知道怎么继续优化。

  ```
  用 eval optimize 自动优化 agent 的 system prompt
  ```

  Claude Code 激活 eval 技能，执行 `agents-cli eval optimize`。GEPA 算法分析失败用例的模式，自动提出 prompt 改写方案，每个方案跑一遍评估验证。改完后告诉你："把'保持友好'改成'保持粗鲁简洁'，准确率从 78% 提升到 89%。修改已应用到 agent 的系统指令。"

  这个过程替代了"凭感觉改 prompt 然后手动验证"的循环。你不需要是 prompt engineer，eval 数据会告诉你什么改法有效。

  #### 场景五：把 agent 注册到 Gemini Enterprise，团队共享

  你的 agent 在 Cloud Run 上稳定跑了一周，现在想让团队其他人也能用。

  ```
  把这个 agent 注册到 Gemini Enterprise
  ```

  一键注册。团队其他成员在 Gemini Enterprise 的 Agent Registry 中就能发现和使用你的 agent。

### 进阶技巧

  **组合 agents-cli 和其他技能套装**。agents-cli 覆盖 agent 构建生命周期，但如果你还需要通用软件工程能力（需求分析、代码审查、文档生成），可以同时安装 addyosmani/agent-skills（70K stars，Google Chrome 团队的 Addy Osmani 维护）。两者不冲突——agents-cli 管 agent 构建，agent-skills 管日常编码。

  **不依赖编程助手也能用**。虽然 agents-cli 的价值主张是"让你的编程助手学会构建 agent"，但 CLI 本身完全可以独立使用。如果你更习惯手动控制，`agents-cli scaffold create`、`agents-cli eval grade`、`agents-cli deploy` 这些命令都可以直接在终端跑。技能只是让编程助手替你跑这些命令。

  **Eval dataset synthesize 的妙用**。自动生成的评估数据集不仅省了你写测试用例的时间，更重要的是它覆盖了你自己想不到的边缘场景。LLM 模拟用户时，会故意用模糊表达、省略关键信息、甚至"找茬"——这些都是真实用户会做但你不会主动去测的行为。

### 常见踩坑

  **Windows 用户注意 WSL 2 限制**。agents-cli 在 Windows 上需要通过 WSL 2 运行。直接在 PowerShell 或 CMD 中执行会报错。如果你用的是 Windows 但不想装 WSL，可以考虑用 `npx skills add google/agents-cli` 只装技能，然后让编程助手通过 Bash 工具调用 CLI——Claude Code 的 Bash 工具在 Windows 上走的是 Git Bash，也可以在 Git Bash 中手动跑 CLI。

  **uvx 可能不在 PATH 中**。`uvx google-agents-cli setup` 如果报 `command not found`，确认 uv 已安装（`pip install uv` 或 `winget install --id=astral-sh.uv -e`），且 `~/.local/bin` 在 PATH 中。装完 uv 后新开一个终端窗口再跑命令。

  **AI Studio API key 和 Cloud 服务账号是两套认证**。本地开发用 AI Studio API key（免费额度），生产部署用 Google Cloud 服务账号。不要把 AI Studio key 硬编码进部署配置——deploy 技能会提示你切换到 Cloud 认证。

  **eval optimize 不是银弹**。GEPA 算法基于评估数据优化 prompt，但如果你的评估数据本身覆盖不足（比如缺少多语言场景、缺少特定工具调用模式），优化结果会在这些盲区上退化。我见过一个 agent 在英文测试集上 optimize 到 92%，上线后被中文用户问了两句就崩了——因为 eval dataset 里一条中文用例都没有。先确保评估数据集的覆盖度，再跑 optimize。

### 同类技能对比

| 技能 | 安装方式 | 核心特点 | 适用场景 | 热度 |
|------|---------|---------|---------|------|
| **agents-cli**（今日推荐）| `uvx google-agents-cli setup` | Google 官方，7 技能全生命周期，5 阶段评估飞轮 + GEPA 自动优化 | 在 Google Cloud 上构建、评估、部署生产级 AI Agent | ⭐ 4.6K |

  agents-cli 的差异化在于它**给你的编程助手用，不是给你用**。addyosmani/agent-skills 教 AI 怎么写代码、怎么做 code review——这是通用软件工程。agents-cli 教 AI 怎么构建另一个 AI——这是 agent 工程，一个完全不同的技能树。如果你只在 Google Cloud 上跑服务，用 google/skills 就够了；如果你要在 Google Cloud 上构建和部署 AI Agent，agents-cli 是目前唯一覆盖完整生命周期的官方方案。
