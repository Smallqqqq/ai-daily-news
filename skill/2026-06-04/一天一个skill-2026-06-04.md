# guizang-social-card-skill — 让 AI Agent 帮你做小红书图文和公众号封面

  写一篇公众号文章需要配封面、做一套小红书图文需要排版设计、把产品测评变成 9 张图文卡片需要版式感和美学判断——这些本该是设计师的活，现在你的 AI Agent 也能干了。今天这个 Skill 内置两套完整的视觉系统（电子杂志风 + 瑞士国际主义），28 个版式骨架，10 套主题色预设，把你的文章、文案、截图、产品笔记直接变成可以发的小红书 3:4 组图和公众号 21:9 + 1:1 封面对。

  [guizang-social-card-skill](https://github.com/op7418/guizang-social-card-skill) 是归藏系列技能的第二作——之前出过爆款 PPT Skill，这次瞄准的是静态社交图文。5 月 27 日发布，不到 10 天斩获 2600+ stars，社区反响远超预期。

### 它解决什么问题

  你用 AI Agent 写文章、做内容已经很快了，但轮到配图这一步就卡住了——要么打开 Canva 手动排版，要么找设计师帮忙，要么用模板套一个看起来差不多的。这个 Skill 把你的 Claude Code / Codex 变成一个懂版式、会配色的视觉编辑，输出不是文字建议，是直接可发的高质量 PNG 图片。

  什么场景需要它？

  - 写了一篇长文要发公众号，需要 21:9 头图 + 1:1 分享卡，要求视觉一致
  - 做了一个产品测评想发小红书，需要 5-9 张 3:4 图文卡片
  - 写了一份游戏攻略想做成轮播图文，但自己没有截图素材
  - 年终总结数据想用可视化的方式呈现，但不想学设计工具

### 核心原理

  这个 Skill 不做 AI 生图——它做的是**版式编排**。核心流程是一条 7 步管线：

  1. **Intake** — 抓取目标平台、内容素材、用户图片。没图时一次性给用户 A/B/C 三选（自己拍/AI 生图/网络取图），不反复追问
  2. **故事提取** — 把原文拆成页面计划：封面钩子 + 每页一个核心观点，5-9 张为宜
  3. **风格选择** — Editorial 杂志风（衬线体 + 墨流背景 + 纸纹质感）或 Swiss 瑞士风（网格 + 发丝线 + 高饱和锚点色）
  4. **版式匹配** — 从 28 个版式骨架中按内容结构挑选（M01-M16 编辑杂志 / S01-S12 瑞士数据），不允许自创版式
  5. **模板复制** — 从种子 HTML 模板开始，不改核心 CSS 系统，只填内容和换版式
  6. **构建渲染** — 单文件 HTML + Playwright 截图出 PNG，不需要前端构建链
  7. **交付审查** — 先给用户看图，再按需跑 validator 自动校验

  关键设计决策：**只用 HTML + CSS 排版，不用 AI 生图做主体**。这避免了 AI 生图的不可控性——文字位置、字号、网格对齐都是确定性的。图片部分走用户提供 → AI 补图 → 网络取图（Unsplash/Pexels/Flickr/Wallhaven）的优先级链。

### 使用指南

#### 场景一：长文章 → 小红书 3:4 图文组图

  你写了一篇关于 AI Agent 工作流的深度文章，想发小红书。对话直接说：

```text
帮我基于这篇文章做一套瑞士风小红书图文，5 张，IKB 蓝。
```

  Agent 会自动：抽核心观点 → 选 Swiss 风格 + IKB 克莱因蓝主题 → 为每张卡片匹配版式骨架（封面用 S01 Hero Statement、数据页用 S09 KPI Tower）→ 复制种子模板 → 填入内容 → `node render.mjs` 出图 → 展示 PNG 给你看。

#### 场景二：公众号封面对（21:9 + 1:1）

  写完公众号文章后需要配封面：

```text
帮我基于这篇文章做公众号封面对：21:9 头图 + 1:1 分享卡，视觉保持一致。
```

  Skill 会在同一个 HTML 文件里同时排版两个画板，确保视觉连贯。21:9 放完整标题 + 配图，1:1 自动从长标题提取短标题，大字号居中，默认不加图。

#### 场景三：截图教程 / 产品说明

  你有一堆产品截图需要做成教程图文：

```text
我有 6 张产品截图，帮我做一套电子杂志风的小红书教程图文。
```

  Skill 会用 `.frame-shot` + `.device-browser` 工具类给截图加壳，优先排版为 Swiss 网格底。截图保持原样不拉伸，文字围绕截图编排而非覆盖。

#### 场景四：旅行 / 生活方式分享

  你有几张露营照片，想做一套满图风格的小红书：

```text
我有 3 张露营照片，帮我做一套全图风格的小红书图文。
```

  Skill 默认走 Editorial + Midnight Ink 或 Dune 主题，封面用 M16 Image-Led Cover 满铺主图，文字叠在安全区（自动做遮罩 + 人脸避让），正文页走 M05 大图 Wells + 旁注。

### 进阶技巧

  **两套视觉系统混搭策略**：虽然 Skill 强调不要混用，但在同一账号的不同内容类型间切换风格是合理的——干货教程用 Swiss + IKB 蓝（数据感、可信度），生活方式内容用 Editorial + Dune（氛围感、叙事性）。这样你的账号既有辨识度又不会单调。

  **4 横带密度检查**：小红书 3:4 画布高度 1440px，Skill 要求内容覆盖 ≥75% 画布高度。渲染后让 Agent 跑 `node validate-social-deck.mjs`，它会用 Playwright 真实渲染测量 DOM 位置，检测溢出、字号超标、底部碰撞、4 横带空白——不是静态扫描，是真实 DOM 测量。

### 常见踩坑

  **Swiss 风格下标题字重过重**：Swiss 的核心规则是"越大越细"。如果你看到 80px 的标题用了 font-weight: 700，validator 会报 R6 警告。正确做法是用 `.h-hero` / `.h-statement` 预定义类名，它们已经内置了正确的字重配对。

  **3:4 卡片内容偏上/偏下**：常见原因是直接套用杂志对开页的留白逻辑。社交卡是逐张独立刷的，没有对开页吸收留白。用 `flex column` + `margin-top: auto` 把 footer 推到底部，内容区域用 `flex: 1` 撑满，不要用 `position: absolute` 定位 footer。

  **图片裁切人脸**：这不是 bug，是默认的 `object-position: center` 在 3:4 裁切比例下必然翻车。Skill 要求在每个 `<img>` 上写 inline `object-position`，基于 Read tool 识别的面部位置——如果是半身人物照，一般是 `center 30%`（上部留天空，主体在中央偏上）。
