# ssh-skill — 让 Claude Code 直接操控远程服务器

  还在 terminal 和 Claude Code 之间来回切换查日志、传文件、重启服务？ssh-skill 把 SSH 操作内嵌为 AI Agent 的原生能力——Claude Code 自动识别你的运维意图，直接登录服务器执行命令、上传下载文件、甚至打通 SSH 隧道把远程数据库端口映射到本地。

---

![[skill/2026-06-12/cover.png]]

### 它解决什么问题

  Claude Code 在本地写代码很强，但一旦涉及远程服务器操作——查 Nginx 日志、上传部署包、数据库迁移——你就得切换回终端手动敲 SSH 命令。每次切换都打断 AI 协作流，而且手动操作容易出错。ssh-skill 把标准的 OpenSSH 配置转化为 Agent 可调用的能力，让 Claude Code 像操作本地文件一样操作远程服务器。

### 核心原理

  ssh-skill 是一套 Python 脚本集，基于 Paramiko 和系统 SSH 客户端，包装为 Claude Code 的 Agent Skill（SKILL.md）。它在标准的 `~/.ssh/config` 之上加了一层元数据（环境标签、服务器描述、自定义标签），并提供五个核心能力：

  - **守护进程长连接**：首次连接启动后台守护进程，后续命令复用连接，连续命令场景性能提升 3.75 倍
  - **智能大文件传输**：≤80MB 走原生 SCP 极速传输，>80MB 自动切换 Paramiko SFTP 显示实时进度
  - **服务器间直传**：数据在服务器之间直接传输，不经过本地，零本地带宽消耗
  - **SSH 隧道**：把远程数据库、内部站点、Docker 容器端口映射到本地 localhost
  - **批量并发操作**：对多台服务器按环境/标签过滤后并发执行命令

### 安装配置

```bash
# 1. 安装依赖
pip install paramiko

# 2. 克隆 skill 到 Claude Code 技能目录
git clone https://github.com/badseal/ssh-skill.git ~/.claude/skills/ssh-skill/

# 3. 确保 ~/.ssh/config 已配置好目标服务器
# 4. 重启 Claude Code，Skill 自动加载
```

SSH 配置示例（标准 OpenSSH 格式，附加元数据注释）：

```ssh-config
# ===== prod-web-01 =====
# description: 生产环境 Web 服务器
# environment: production
# tags: web,nginx,production
# location: 阿里云-北京
Host prod-web-01
    HostName 192.168.1.100
    User root
    IdentityFile ~/.ssh/id_rsa
    Port 22

# ===== internal-server =====
# description: 内网应用服务器（通过跳板机）
# environment: production
Host internal-server
    HostName 10.0.1.100
    User appuser
    ProxyJump bastion
```

### 使用指南

#### 场景一：日常运维——一句话查状态、看日志

  过去你要开一个新 terminal，`ssh prod-web-01`，然后 `systemctl status nginx`，再 `tail -f /var/log/nginx/error.log`。现在在 Claude Code 里直接说就行：

```
用户：在 prod-web-01 上检查 Nginx 状态，看看最近 50 条错误日志
```

  AI 自动调用 `ssh_execute.py` 连接服务器执行命令，返回结果直接展示在对话中。不需要离开 Claude Code，不需要记 IP 和路径。

#### 场景二：大文件部署——自动显示进度，不怕超时

  部署一个 500MB 的应用包到生产服务器，传统 SCP 没有进度条，超时 30 秒就断。ssh-skill 智能判断：大于 80MB 自动走 SFTP，实时显示百分比、速度、预计剩余时间：

```bash
# AI 自动调用
python ~/.claude/skills/ssh-skill/scripts/ssh_upload.py \
  prod-web-01 ./app-v2.0.tar.gz /opt/apps/

# 输出：
# 上传进度: 45.2% (2.1 MB/s) ETA: 102.3s
```

  断点续传也支持——网络断了重连后加上 `--resume` 接着传，不用从头开始。

#### 场景三：数据库内网访问——SSH 隧道一键打通

  生产数据库出于安全不开放外网端口（比如 MySQL 3306），以前你得手动建立 SSH 隧道。现在一句话：

```
用户：把 dev-001 服务器中的 MySQL 容器映射到本地
```

  ssh-skill 自动创建后台守护进程，把远程容器内的 3306 端口映射到 `localhost:3306`。之后你本地的任何工具（Navicat、DBeaver、代码里的数据库连接）都能直接访问，就像数据库跑在本地一样。

#### 场景四：数据迁移——服务器间直传，不占本地带宽

  从旧服务器迁移数据库到新服务器，几百 GB 的数据如果先下载到本地再上传，耗时翻倍还占带宽。ssh-skill 的服务器间直传模式让数据直接在源和目标之间流动：

```bash
# 自动模式——AI 一句话触发
用户：把 old-server 的 /data/database.sql 迁移到 new-server 的 /data/

# 大文件用直连模式（数据不经过本地）
# 支持 rsync 增量同步
```

### 进阶技巧

  **批量并发运维**：对所有 production 环境的 Web 服务器同时重启 Nginx，`--parallel` 并发执行，`--max-workers` 控制并发数：

```bash
ssh_cluster.py "systemctl restart nginx" --environment production --parallel --max-workers 10
```

  **组合 Claude Code 工作流**：把 ssh-skill 和其他 Skill 联动。比如 `/devloop` 在自动部署验证阶段，通过 ssh-skill `curl` 验证线上 URL 是否返回了新功能标记；或者代码审查发现潜在问题时，直接上服务器查实时日志辅助判断。

### 常见踩坑

  **Windows 路径问题**：在 Git Bash 或 MSYS2 环境下，文件路径可能被自动转换。上传文件时加 `MSYS_NO_PATHCONV=1` 前缀禁用路径转换：

```bash
MSYS_NO_PATHCONV=1 python ~/.claude/skills/ssh-skill/scripts/ssh_upload.py prod-web-01 ./app.tar.gz /tmp/
```

  **守护进程不释放**：长连接守护进程空闲 30 分钟自动退出。如果端口被占用，手动杀掉：`pkill -f ssh_daemon.py`。

  **跳板机认证链**：确保本地能 `ssh bastion` 直接登录，且跳板机能解析内网主机名。建议用 `IdentityFile` 明确指定密钥路径，避免 Agent 在多个密钥间猜测。

## 今日小结

  ssh-skill 适合所有需要管理远程服务器的开发者——无论是个人项目的 VPS、团队开发环境、还是生产集群。安装只需 `pip install paramiko` 加复制目录两步，5 分钟配置好 `~/.ssh/config` 就能让 Claude Code 接管所有 SSH 操作。配置一次，之后每次运维操作都省掉切窗口、记命令、手动传文件的重复劳动。如果你是后端开发者或 DevOps，今天装上就能立刻感受到"不用离开 Claude Code"的流畅感。
