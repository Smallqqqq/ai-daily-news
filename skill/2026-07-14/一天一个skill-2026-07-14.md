# caveman — 让 AI 学会像原始人一样说话，Token 省下 65%

  AI 编程助手的一句"Sure! I'd be happy to help you with that. Let me take a look at the issue..."十个单词里六个不携带信息。caveman 是 GitHub 上最火的 Claude Code 技能之一（88,992 stars），用一条极简规则解决了这个问题：让 AI 像原始人一样说话——意思不变，废话全砍。平均省下 65% 的输出 token，且代码、命令、错误信息逐字节保留。

  这个技能来自 Julius Brussee（游戏引擎之父 Arjan Brussee 的儿子），40 位贡献者维护，v1.9.1 刚在 7 月 3 日发布。它不仅是一个 Claude Code 技能，还支持 Codex、Gemini CLI、Cursor、Windsurf、Cline、Copilot 等 30+ 个 AI 编码工具。

### 它解决什么问题

  用 AI 编程的人迟早会撞上同一个问题：**Token 太贵了，而且大部分是废话。**

  一个典型的 AI 回复长这样：

  > "Sure! I'd be happy to help you with that. The issue you're experiencing is most likely caused by your authentication middleware not properly validating the token expiry. Let me take a look and suggest a fix."

  同样的信息，caveman 模式下的回复：

  > "Bug in auth middleware. Token expiry check use `<` not `<=`. Fix:"

  意思完全一致，代码片段一字不差，但 tokens 从 69 压到 19。在一个 AI 辅助编程持续数小时的场景里，这种节省会滚雪球——每次 agent 回话都省 60-70%，一天下来就是几美元的差距。

  更关键的是，2026 年 3 月一篇论文（*Brevity Constraints Reverse Performance Hierarchies in Language Models*）测试了 31 个模型后发现：**约束大模型给出简短回答，在某些基准测试上准确率反而提升了约 26 个百分点。** 少说话不仅省钱，有时还更正确。

### 核心原理

  caveman 的核心只是一条系统提示词规则：

  > "You are a caveman. Speak like one. Keep code, commands, and error messages byte-for-byte exact. Cut all filler words, politeness, hedging, and lead-in sentences."

  这条规则被封装成一个符合 Agent Skills 标准的 `SKILL.md` 文件。安装后，Claude Code 在每次对话中自动加载这个技能，所有输出都走这条压缩规则。

  关键设计细节：

  - **代码和命令逐字节保留**。压缩只作用于自然语言部分，代码块、终端命令、错误日志完全不动。你不会因为开了 caveman 而丢失任何技术信息。
  - **六档压缩等级**，`lite` / `full` / `ultra` / `wenyan`，随时切换。`lite` 只砍掉礼貌语和开场白，`full`（默认）进一步压缩解释性文字，`ultra` 把回复压成电报体，`wenyan` 用文言文——中文语境下信息密度最高。
  - **语言保持**。用英文提问，caveman 用英文回。葡语、西语、法语同理。它压缩的是**风格**不是语言。`wenyan` 是唯一例外——文言文的信息密度天然高于现代白话，故意设计成这样。
  - **caveman-compress 连输入 token 一起省**。`/caveman-compress` 命令可以把 Claude Code 的记忆文件（`CLAUDE.md`、`MEMORY.md` 等）压缩成 caveman 风格，不丢代码、不丢路径、不丢 URL。实测能把 `CLAUDE.md` 从 706 tokens 压到 285 tokens（省 59.6%），而这些文件是**每轮对话都会加载的**——省输入 token 的效果是乘数级的。

  技能本身有约 1-1.5k tokens 的输入开销（系统提示词注入），所以对于非常简短的对话（一两轮就结束），开了可能净亏。官方文档 [`docs/HONEST-NUMBERS.md`](https://github.com/JuliusBrussee/caveman/blob/main/docs/HONEST-NUMBERS.md) 对此非常坦诚——它把什么情况下赚、什么情况下亏、怎么自己实测都写清楚了。

### 安装配置

  **一行命令，自动检测所有已安装的 AI 编码工具，一次性全部安装：**

  ```bash
  # macOS · Linux · WSL · Git Bash
  curl -fsSL https://raw.githubusercontent.com/JuliusBrussee/caveman/main/install.sh | bash
  ```

  ```powershell
  # Windows · PowerShell 5.1+
  irm https://raw.githubusercontent.com/JuliusBrussee/caveman/main/install.ps1 | iex
  ```

  约 30 秒完成。需要 Node ≥ 18。已安装的 agent 自动覆盖，没装的跳过。

  **单个 agent 安装：**

  ```bash
  # Claude Code 插件
  claude plugin marketplace add JuliusBrussee/caveman && claude plugin install caveman@caveman

  # Gemini CLI 扩展
  gemini extensions install https://github.com/JuliusBrussee/caveman

  # Cursor / Windsurf / Cline / Codex 等 30+ agent（通过 skills 注册表）
  npx skills add JuliusBrussee/caveman -a cursor
  ```

  安装后 Claude Code 的状态栏会显示 `[CAVEMAN] ⛏ 12.4k`——这是你的终身 token 节省量，每次跑 `/caveman-stats` 时更新。

### 使用指南

#### 场景一：日常开发——每次回复都自动压缩

  安装后无需任何操作。Claude Code 启动时自动加载 caveman 技能，所有回复默认用 `full` 级别压缩。你正常写代码、问问题、修 bug，回复自动变短。

  效果对比例子：

  > **正常模式**（69 tokens）："The reason your React component is re-rendering is likely because you're creating a new object reference on each render cycle. When you pass an inline object as a prop, React's shallow comparison sees it as a different object every time, which triggers a re-render. I'd recommend using useMemo to memoize the object."

  > **caveman 模式**（19 tokens）："New ref each render. Wrap object in `useMemo`."

  技术信息完全等价。省下的 50 个 token 全是"polite filler"。

#### 场景二：长任务中的 token 预算控制

  跑一个多小时的 agent 任务——比如"帮我重构整个认证模块"——agent 会在几十轮对话里产生大量输出。每轮省 60-70%，40 轮下来可能就是几万 tokens 的差距。在 Anthropic API 定价下，这意味着从 $0.50 的任务成本降到 $0.18。

  如果你在用 Claude Code 的 `/loop` 做持续任务，配合 caveman 效果尤其显著——因为循环中的每一轮都在省钱。

#### 场景三：压缩项目记忆文件——输入 token 也省

  这是 caveman 最被低估的功能。`/caveman-compress` 命令会读取你的 `CLAUDE.md` 或任何 markdown 记忆文件，重写成 caveman 风格，保存原内容中的所有代码、路径和 URL：

  ```bash
  # 在 Claude Code 中输入
  /caveman-compress CLAUDE.md
  ```

  实测数据：

  | 文件 | 原始 tokens | 压缩后 | 节省 |
  |------|-----------|--------|------|
  | claude-md-preferences.md | 706 | 285 | 59.6% |
  | project-notes.md | 1145 | 535 | 53.3% |
  | team-conventions.md | 892 | 348 | 61.0% |

  这些文件每轮对话都会加载到上下文中。压缩一个 `CLAUDE.md`，相当于**每次对话都省了 400+ 个输入 token**。一个工作日 50 轮对话就是 20,000 tokens。

#### 场景四：caveman-commit 和 caveman-review

  caveman 附带两个子命令，把压缩逻辑延伸到 Git 工作流：

  - `/caveman-commit`：生成 ≤50 字符的 Conventional Commit 消息，"why"优先于"what"
  - `/caveman-review`：PR 评论压缩成一行——`L42: 🔴 bug: user null. Add guard.`

  这对于用 `/commit` 或 `/code-review` 内置技能的用户来说，相当于给这些技能加了一层"输出压缩器"。

### 进阶技巧

  **组合技：caveman + caveman-shrink + cavecrew。** caveman 的生态有三个组件：（1）`caveman` 技能本身——压缩 agent 的直接输出；（2）`caveman-shrink`——MCP 中间件，包装任何 MCP 服务器的工具描述，把工具说明也压缩成 caveman 风格，省的是输入 token；（3）`cavecrew-*`——三个 caveman 风格子 agent（investigator、builder、reviewer），比原生子 agent 省约 60% 的输出 token。三个组件一起用，从主对话到子 agent 到 MCP 工具描述全链路压缩。

  **文言文模式的隐藏价值。** `wenyan` 级别不是噱头——中文世界的开发者用文言文模式处理中文代码注释和文档时，信息密度远超现代汉语。一段 50 字的白话说教，文言文 15 字表达干净。如果你的 `CLAUDE.md` 是用中文写的，用 `wenyan` + `caveman-compress` 能压出更高的节省比。

  **不值得用 caveman 的场景。** 官方文档明确列出了不适用场景：已经非常简短的回复（比如 `git status` 输出）、需要详细解释的教程类对话、与非技术背景的合作者交流。在这些场景里，1-1.5k 的技能开销反而让总 token 上升。

### 常见踩坑

  **Token 节省率不是 65% 全程生效。** 65% 是 10 个 benchmark 任务的输出 token 平均值（范围 22%-87%）。实际使用中，如果你的对话本身就简短精炼，节省率会低于 65%。官方在 `HONEST-NUMBERS.md` 中明确写了一个公式帮你预估：`session_savings = (output_tokens × 0.65) - (turns × 1500)`。当 `turns` 很少、`output_tokens` 也很少时，可能净亏。

  **caveman 模式下的回复可能显得"没礼貌"。** 对于习惯了 AI 说"当然！我很乐意帮你"的用户，突然看到"Bug here. Fix:"会觉得生硬。这不是 bug，是设计目标。如果你在给别人演示 AI 编程，建议临时切回 `lite` 模式或直接说"normal mode"关闭。

  **wenyan 模式对英文技术内容不友好。** 文言文擅长压缩中文，但英文函数名、变量名、技术术语在文言文语境里很别扭。如果你的代码库主要是英文，`ultra` 模式通常比 `wenyan` 更实用。

### 同类技能对比

| 技能 | 安装方式 | 核心特点 | 适用场景 | 热度 |
|------|---------|---------|---------|------|
| **caveman**（今日推荐）| `curl` 一键脚本 | 输出 token 压缩 65%，六档等级，记忆文件压缩，多 agent 支持 | 所有 AI 编程场景的 token 成本控制 | ⭐ 89k |
| token-budget skill | `git clone` 手动配置 | 设定 token 上限，超出后自动总结 | 严格预算控制 | ⭐ ~50 |
| concise-mode（Claude Code 内置）| 无安装 | 下拉框选择简洁/详细模式 | 临时切换 | 内置 |

  caveman 通过改变写作风格来压缩输出，而不是限制长度。限制长度会导致 AI 半句话没说完就被截断，caveman 确保每个回复都是完整的思想表达，只是没有废话。加上 `caveman-compress` 能永久压缩记忆文件的输入 tokens，收益是长期的、复利式的。

## 今日小结

  caveman 可能是你安装过的"最不正经"的 AI 技能——让 Claude 学原始人说话。但它也可能是回报率最高的技能之一。30 秒安装，零配置，之后你的每一次 AI 编程对话都在省钱。不是靠限制输出长度（那会截断代码），而是靠砍掉所有对技术内容没有贡献的礼貌语、开场白、重复解释。

  如果你每天的 AI 编程对话超过 20 轮，caveman 每个月的 token 节省量大概够你免费多跑一个大任务的。适合所有用 AI 编程且关心 token 成本的开发者。不适合的场景也很明确：教学演示、非技术沟通、极短对话。
