# gh skill — 一行命令，统一管理 30+ AI Agent 的技能

  2026 年 4 月，GitHub 官方在 CLI v2.90.0 中悄悄上线了 `gh skill`——一个用于发现、安装、管理和发布 AI Agent 技能的扩展命令。它解决的问题很简单也很棘手：你同时在用 Claude Code、Copilot、Cursor、Codex CLI，每个工具的技能安装路径和管理方式各不相同，团队里每人一把配置，代码规范写了却没人在用。`gh skill` 把这摊碎片拼回一个来源。

### 它解决什么问题

  AI 编程助手的碎片化已经到了影响生产力的程度。你上午用 Claude Code 写后端，下午切 Copilot 调前端界面，晚上用 Codex 跑批量重构。每个助手都有自己的技能目录、自己的安装命令、自己的更新机制。

  更糟的是团队层面：你给 Claude Code 写的代码审查规范，Copilot 用户看不到；你沉淀的 commit 约定，Cursor 同事读不到。技能（skills）是 AI 时代的知识资产，但它没有一个包管理器——直到 `gh skill` 出现。

  `gh skill` 做三件事：统一安装入口（一个命令装到所有 agent）、版本锁定（tag/commit SHA 钉死版本）、出处追踪（每份技能文件记录来源 repo 和 tree SHA）。它不是另一个技能市场，而是技能生态的包管理器——类似 npm 之于 JavaScript，pip 之于 Python。

### 核心原理

  `gh skill` 是 GitHub CLI 的扩展命令，底层依赖两个机制：

  **自动路由到各 agent 目录**。安装时，`gh skill` 读取技能包的 `skill.yaml` 清单，根据 `--agent` 参数（或自动检测已安装的 agent），把 `SKILL.md` 文件写入对应位置：
  - Claude Code → `~/.claude/skills/`
  - Codex CLI → `~/.codex/skills/`
  - Cursor → `.cursor/skills/`
  - Copilot → VS Code 配置目录
  - Gemini CLI → `~/.gemini/skills/`

  **Git 原生的供应链安全**。每份技能在安装时，会在 `SKILL.md` 的 frontmatter 中写入来源仓库、git ref 和 tree SHA。这意味着一份技能无论被复制、移动、重命名多少次，你都能追溯到它的来源，也能检测到上游的实质内容变更（不是版本号 bump，而是 tree SHA 对比）。技能包作者可以启用 immutable releases，发布后的版本连管理员都无法篡改。

  `gh skill` 也自带安全阀：安装前用 `gh skill preview` 查看技能内容，防止 prompt injection 和恶意脚本。GitHub 明确表示不会审核技能内容——这和 npm/PyPI 一样，用户自行判断。

### 安装配置

  前提：GitHub CLI ≥ v2.90.0。

```bash
# 升级 gh CLI（macOS）
brew upgrade gh

# 升级 gh CLI（Windows）
winget upgrade GitHub.cli

# 安装 gh skill 扩展
gh extension install github-actions/gh-skills

# 验证安装
gh skill --help
```

  三步搞定，不需要额外的运行时或配置文件。

### 使用指南

#### 场景一：快速安装一个技能——"让 AI 按团队的 commit 规范写提交信息"

  你在 GitHub 上找到了一个 `git-commit` 技能包，想安装到所有 agent：

```bash
# 浏览仓库中有哪些技能
gh skill install owner/git-workflow-skills

# 直接安装指定技能到 Claude Code
gh skill install owner/git-workflow-skills git-commit --agent claude-code

# 安装到所有已检测到的 agent
gh skill install owner/git-workflow-skills git-commit --all
```

  技能会自动出现在每个 agent 的技能列表中。重启 agent 或新开会话即可生效。

#### 场景二：锁定版本，防止意外升级——"上次那个技能自动更新后把代码格式搞乱了"

  经历过一次静默升级踩坑后，你会想锁死版本：

```bash
# 锁定到发布标签
gh skill install owner/repo code-review --pin v1.2.0

# 锁定到特定 commit（最大程度可复现）
gh skill install owner/repo code-review --pin abc123def
```

  锁定后的技能在 `gh skill update --all` 时会被自动跳过。只有你手动解除锁定或重新指定版本才会变更。

#### 场景三：团队统一技能配置——"新同事入职，一键装好所有项目技能"

  维护一个团队技能仓库，用 `skill.yaml` 列出所有项目需要的技能：

```yaml
# skill.yaml
name: team-coding-standards
version: 2.1.0
description: "公司统一编码规范"
agents:
  - claude-code
  - copilot
  - cursor
  - codex
skills:
  - path: skills/commit-conventions.md
  - path: skills/code-review-checklist.md
  - path: skills/api-design-standards.md
  - path: skills/testing-requirements.md
```

  新同事入职只需一行：

```bash
gh skill install your-org/team-coding-standards --all
```

  四项技能同时安装到所有 agent，团队知识资产秒级同步。

#### 场景四：发现社区技能——"有没有好用的 MCP 调试技能？"

```bash
# 关键词搜索
gh skill search mcp-debugging

# 浏览特定仓库
gh skill install github/awesome-copilot
```

  交互式选择界面会列出仓库中所有可用技能，选好后自动安装。

### 进阶技巧

  **用 `gh skill preview` 审查陌生技能**。任意技能安装前，先预览内容：

```bash
gh skill preview owner/repo skill-name
```

  这会直接输出 `SKILL.md` 的完整内容到终端。重点检查：是否有隐藏的 `curl | bash` 调用、是否读取敏感环境变量、是否写入系统目录。

  **把技能出处信息当作文档**。安装后的技能 frontmatter 会包含来源信息。打开 `~/.claude/skills/xxx/SKILL.md` 可以看到 `_gh_skill_repo`、`_gh_skill_ref`、`_gh_skill_tree_sha` 等字段。当你需要向同事解释"这个技能从哪来的、什么版本"，不用翻聊天记录，文件自带答案。

### 常见踩坑

  **某些 agent 不完全支持复杂指令**。Copilot 对条件逻辑和多步骤工作流的支持弱于 Claude Code。解决方案：把共享技能写成简洁的平铺指令，Claude Code 专有的高级配置（hooks、工具限制）单独放在 `agent-overrides/` 目录下，安装时只针对 Claude Code。

  **gh skill 需要 GitHub 仓库的读取权限**。如果你安装的仓库是私有的，确保 `gh auth status` 显示已登录且对该仓库有读取权限。公开仓库无此限制。

  **技能仓库的结构必须符合规范**。根目录需要有 `skill.yaml` 清单。如果你的技能是用 `npx skills` CLI 安装的，迁移到 `gh skill` 需要补一个 `skill.yaml`。步骤：在仓库根目录新建 `skill.yaml`，填写 `name`、`version`、`skills` 列表（每项指向 Markdown 文件路径），然后 `gh skill publish --fix` 自动修正元数据。

### 同类技能对比

| 方案 | 安装方式 | 核心特点 | 适用场景 | 热度 |
|------|---------|---------|---------|------|
| **gh skill**（今日推荐）| `gh extension install` | Git 原生版本管理、供应链安全、跨 30+ agent | 团队统一技能管理 | ⭐ GitHub 官方 |
| `npx skills` (Vercel) | `npx skills add` | 最早的生态 CLI，skills.sh 排行榜 | 个人快速安装 | ⭐ 26K+ installs |
| `asm` (agent-skill-manager) | `npm install -g asm` | TUI 交互界面、安全审计、多源索引 | 需要审计和批量管理的团队 | ⭐ 新项目 |
| 手动复制 | `cp -r` | 零依赖 | 单个技能的一次性安装 | — |

  `gh skill` 的差异化在于"Git 原生的供应链安全"。`npx skills` 更早更成熟，skills.sh 排行榜很实用，但它没有版本锁定和出处追踪——你装了一个技能，三周后更新了，你不知道改了什么。`gh skill` 用 tree SHA 对比解决了这个问题。如果团队已经有了 `gh` CLI 的使用习惯，切换到 `gh skill` 的认知成本极低。

## 今日小结

  `gh skill` 适合已经把 GitHub CLI 融入日常的开发者，以及需要管理多个 AI 编程助手的团队。安装耗时 2 分钟（升级 gh + 安装扩展），之后发现技能、安装技能、锁定版本、更新技能全部统一为 `gh skill` 子命令。它不生产技能，但让技能的传播和管理从手工拷贝变成了一条命令。如果你的团队已经在用 CLAUDE.md 加 SKILL.md 沉淀规范，下一个自然步骤就是把它变成可分发、可追溯的技能包。
